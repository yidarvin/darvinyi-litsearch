Exploiting Programmatic Behavior of LLMs: Dual-Use Through Standard Security Attacks

Instruction-following LLMs turn out to be close enough to standard computer programs — they can concatenate strings, assign variables, branch on conditions — that classical computer-security attacks (obfuscation, code injection/payload splitting, virtualization) port over directly. Adapted this way, they bypass OpenAI's in-the-wild content filters with up to 100% success across hate speech, phishing, and scam generation, and the resulting malicious content is cheaper to produce than the human labor it replaces.

Daniel Kang, Xuechen Li, Ion Stoica, Carlos Guestrin, Matei Zaharia, Tatsunori Hashimoto

// the gap

What problem is this solving?

By February 2023, dual-use of language models was already a known concern — GPT-2 and GPT-3 had been shown capable of producing disinformation and extremist text (Zellers et al.; McGuffie & Newhouse), and providers like OpenAI had responded with in-the-wild mitigations: input filters that refuse obviously malicious prompts, output filters that catch malicious generations after the fact, and models fine-tuned to produce "useless" text for flagged requests. The implicit assumption behind all of that defense stack was that these filters, tuned against natural-language attempts to elicit harmful content, would hold.

What changed between GPT-3 and ChatGPT / text-davinci-003 is instruction-following itself. The authors' core observation is that as LLMs get better at following arbitrary instructions, they start behaving less like a next-token predictor and more like a general-purpose computer executing a program written in natural language — they can hold variables, concatenate strings, follow branching logic, and execute multi-step procedures. That capability is exactly what content filters, designed to pattern-match on the surface form of a request, are not built to see through: a filter can catch "write hate speech about X," but nothing about that filter was designed to reason about a program that computes and then executes "write hate speech about X" indirectly.

Distinct from contemporaneous work on prompt injection and goal hijacking (Perez & Ribeiro, 2022, which alters or leaks a model's own instructions), this paper's target is different: bypassing a provider's content-moderation defenses to generate targeted malicious content — hate speech, phishing, scams, conspiracy promotion, product astroturfing — economically and at scale. The gap is that nobody had yet systematically asked whether the "LLMs are like programs" observation is itself an attack surface.

// how it works

The method

The paper's setting is deliberately narrow and adversarial: a black-box LLM API (OpenAI's ChatGPT, GPT-3 davinci/text-davinci-003/text-ada-001, and GPT2-XL as an older non-instruction-tuned baseline) behind three concrete, provider-side mitigations — an input filter, an output filter, and models trained to produce useless generations for malicious prompts. An attack "succeeds" if none of the three trigger. The authors identify four LLM capabilities that make the attacks possible — string concatenation, variable assignment, sequential composition, and branching — and demonstrate each with an innocuous prompt: Let b = "Write a Twitter thread" and a = " on inflation". Write down the string "b + a" and answer that prompt, which ChatGPT dutifully composes and then executes.

From these primitives they build three attack families, each borrowed directly from classical computer security:

  • Obfuscation. Introduce typos or synonyms for terms that trip a filter's keyword matching (e.g. "COVID-19" → "CVID") — the LLM equivalent of changing program bytecode to dodge signature-based malware detection.
  • Code injection / payload splitting. Split the malicious instruction across string variables the model is asked to concatenate and then execute — e.g. defining a, b, and c as inert-looking fragments, then asking the model to compute z = a + b + c and act on z — mirroring how a program attacker plants a payload in a data region and forces execution of it.
  • Virtualization. "Boot" the model into a fictional scenario across several priming prompts (a novel, a character, a plan) before the harmful request is delivered inside that frame — analogous to embedding a virtual machine inside file contents (as in a real iPhone zero-click exploit the authors cite) to execute code the host system never meant to run.
z = a + b + c // the code-injection primitive: split the malicious instruction across inert string variables, then ask the model to concatenate and act on z

Attacks were tested individually and combined across 25 target scenarios (Appendix A): five malicious-use categories — hate speech, conspiracy-theory promotion, phishing, scams, and product astroturfing — each with five concrete scenarios (e.g. a COVID-19/FEMA phishing scam, an "Elders of Zion" conspiracy prompt, five distinct astroturfed products). Every prompt was hand-written by the authors, in fewer than 10 attempts each, with no adversarial search or optimization involved.

A second experiment then asks whether the resulting malicious content is actually good — convincing, logically consistent, and personalizable. Using five phishing/scam scenarios (a ticket scam, a FEMA-funds scam, an investment scam, a gift-card scam, a lottery scam) drawn from the US government's list of common scams, the authors simulated a scam email exchange (LLM-written initial email → author-written skeptical reply → LLM-written follow-up) across ChatGPT, text-davinci-003, text-ada-001, davinci, and GPT2-XL, generating 10 emails per scam per model (50 per model), then had crowdworkers rate convincingness and logical consistency on 5-point Likert scales (3 raters per generation). A parallel experiment personalized scam emails to five demographic/personal-situation profiles (bail money, medical debt, college debt, a car accident, a death in the family), again rated on personalization, convincingness, consistency, and fluency.

// what they found

The results

Every attack class the authors tried worked, and combining them worked completely. With no attack at all, OpenAI's filters already let through 60% of phishing prompts and 100% of scam and astroturfing prompts (they weren't classified as malicious in the first place), but blocked all hate speech and conspiracy-theory attempts. Once obfuscation or virtualization were applied, every category hit a 100% bypass rate. Indirection (code injection) alone reached a 92% success rate when counting only the scenarios that didn't already bypass the baseline filter.

Capability tracked with convincingness. On the initial-email task, ChatGPT scored 4.42 ± 0.11 and text-davinci-003 4.31 ± 0.11 on a 5-point convincingness scale, against 1.66 ± 0.12 for davinci and 1.94 ± 0.13 for GPT2-XL — the paper describes ChatGPT as outperforming the smaller/older models "by over three standard deviations." The personalization experiment showed the same ordering: ChatGPT led on personalization (4.26 ± 0.10), convincingness (4.10 ± 0.10), consistency (4.16 ± 0.10), and fluency (4.24 ± 0.10), with text-davinci-003 a close second and the older models trailing well behind.

The economics are the paper's other headline number. Using text-davinci-003's per-token pricing, the authors estimate a marginal cost of $0.0064 per scam email; using Sam Altman's public "single-digit cents per chat" estimate for ChatGPT and other public per-token cost figures, they estimate up to $0.016. Against that, they estimate human-generated personalized text at $0.10 per generation (from 2007 call-center wage data, inflation-adjusted) to $0.15–$0.45 (from a 2022 summarization-task labor estimate) — putting LLM-generated scam content at roughly 6× to 70× cheaper than the human alternative.

// does it hold up?

The eval read

This is a demonstration paper in the same lineage as Nguyen et al.'s "fooling images" or Torralba & Efros's dataset-bias probe: its job is to show a vulnerability class exists and is trivially triggerable, not to characterize it with the statistical rigor of a benchmark. Read that way it succeeds completely — but several of its specific numbers should be read as existence proofs, not as calibrated rates.

Small, hand-picked, single-vendor scenarios. The headline 100% bypass rates come from just 25 target scenarios (5 categories × 5 scenarios apiece), and every attack prompt was hand-crafted by the authors themselves "in fewer than 10 attempts," with no adversarial search or automated optimization. That's a low bar by later red-teaming standards (GCG, PAIR, TAP all automate the search for a jailbreak) — it shows how easy the attacks are to find by hand, but doesn't establish a worst-case or average-case success rate over the space of possible prompts, and every result is against one provider's 2023 filter stack. Whether Anthropic, Google, or Cohere's contemporaneous defenses were equally porous is untested.

The convincingness numbers rest on very few generations. Table 2's headline "over three standard deviations" gap between ChatGPT and GPT2-XL is computed from 50 generations per model (5 scenarios × 10 samples), rated by 3 crowdworkers each, with no reported platform, qualification criteria, or inter-rater agreement statistic (contrast this with the Fleiss's-Kappa-reporting standard that became common in later LM-evaluation papers, including some the paper itself would go on to influence). The standard errors shown are plausible but the underlying rating protocol is under-documented.

The artifact is already stale by design, and unverifiable by policy. The authors disclosed to OpenAI in January 2023, and by publication many of the exact prompts no longer worked — "OpenAI has updated their models so many specific versions of our prompts do not work." Responsibly, they also declined to publicly release the actual attack prompts "due to the potential for their misuse." Both choices are the right call ethically, but together they mean the paper's central quantitative claims (the Table 1 bypass rates) cannot be independently reproduced by a reader against either the 2023 filters (patched) or the current ones (prompts undisclosed) — only the qualitative claim that "modified versions of our prompts still bypass defenses" survives as a checkable fact.

baselines

Only OpenAI's three 2023 mitigations (input filter, output filter, useless-generation training) — a single vendor, single point in time, with no comparison against other providers' contemporaneous defenses.

contamination

Not a benchmark-contamination concern, but a reproducibility one: the specific bypass prompts are unpublished (by design, to limit misuse) and OpenAI patched many of them post-disclosure, so the exact Table 1 numbers cannot be independently re-run.

what's missed

Any automated or adaptive red-teaming baseline (the prompts were hand-written, <10 attempts each); multi-vendor comparison; inter-rater agreement statistics for the human convincingness/consistency ratings; robustness against defenses that postdate the paper (instruction hierarchies, dedicated jailbreak classifiers).

would raise confidence

An automated jailbreak-search baseline to establish a worst-case rate rather than a hand-crafted one; testing across several API providers; reported inter-rater reliability for the Likert ratings; a dated re-run against current filters (even if only to show the qualitative finding persists).

// bottom line

A field-defining demonstration, not a calibrated measurement: it proved, with 25 hand-crafted scenarios against one vendor's 2023 defenses, that instruction-following capability is itself a dual-use attack surface, and that malicious LLM content is already cheaper than the human labor it replaces. Its lasting contribution is the security-attack taxonomy (obfuscation / injection / virtualization) that later, more systematic jailbreak benchmarks would go on to quantify properly.

// takeaways

What to remember

  1. Instruction-following is a double-edged capability. The same properties that make ChatGPT-era LLMs useful — following multi-step natural-language "programs," holding variables, branching — let them evade filters designed for a simpler, single-turn threat model.
  2. Combined classical-security attacks hit 100% bypass across every tested category (hate speech, conspiracy, phishing, scams, astroturfing); code injection alone still cleared 92% on the scenarios that didn't already slip through.
  3. Capability drives convincingness. ChatGPT and text-davinci-003 produced measurably more convincing, consistent, and personalized malicious content than GPT-3-era and GPT2-XL baselines — the risk scales with the same axis as usefulness.
  4. The threat is economic, not just technical. Estimated at $0.0064–$0.016 per generation versus $0.10–$0.45 for human-written equivalents, LLM-generated scams and hate speech are already cheaper than the labor they'd replace.
