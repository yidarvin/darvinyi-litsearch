Counterfactual Multi-Agent Policy Gradients

A centralized critic paired with a counterfactual baseline — marginalizing out one agent's action while holding the rest fixed — to solve multi-agent credit assignment in cooperative deep RL.

// the gap

What problem is this solving?

By 2017, deep RL had strong single-agent recipes, but cooperative multi-agent systems break an assumption those recipes depend on twice over. First, when every agent's policy is changing simultaneously, the environment is perpetually non-stationary from any single agent's point of view. Second — and this is the problem COMA is actually built to fix — cooperative agents typically share one global reward, so a joint action that wins the game produces exactly the same reward signal for the agent that made the decisive play and for an agent that did nothing useful. This is multi-agent credit assignment (Chang, Ho, and Kaelbling 2003): with only a team-level reward to learn from, an individual agent's gradient reflects the whole team's outcome rather than its own marginal contribution, and that noise compounds as more agents explore simultaneously into the same shared signal. The standard fix — difference rewards (Wolpert and Tumer 2002), which shape each agent's learning signal by comparing the real global reward to what would have happened had that agent taken some default action instead — requires re-simulating the environment for every agent, every step, which is workable in a fully controllable simulator but flatly infeasible in an environment like StarCraft, where there is no way to freeze the world, swap out one unit's action, and replay. Independent per-agent learners sidestep the simulation cost but throw away credit assignment entirely: each agent's own critic sees only its own local reward and local history, with no way to reason about what any other agent did. Compounding this, the existing StarCraft micromanagement approaches with any degree of centralization — Usunier et al. (2016)'s GMEZO and Peng et al. (2017)'s bidirectionally-coordinated nets — sidestepped the harder version of the problem by giving a single controller the full game state and the engine's built-in attack-move macro-actions, which auto-path a unit into firing range. That is a meaningfully easier task than learning genuinely decentralized control — each unit conditioning only on its own local observations at execution time — from raw move/attack/stop commands. What was missing was a method that could use the fact that training happens in a simulator (extra state, cross-agent communication) to solve credit assignment directly, without needing extra simulations, while still producing policies that only need local information once deployed.

// how it works

The method

COMA is centralized training with decentralized execution, built from three ideas that layer on top of each other. First, a single centralized critic Q(s, τ, u) is learned during training only — it conditions on the true global state s, the joint action-observation histories τ, and the full joint action u of every agent — while each agent's actor πa(ua|τa) still conditions only on its own action-observation history τa, exactly as an independent actor-critic agent would (Figure 1a); only the actor is needed at execution time, so nothing about deployment changes. A naive way to use this critic would be to follow the plain TD-error gradient, g = ∇log π(uat|τat)(r + γV(st+1) − V(st)) — but this is exactly as uninformative about individual contribution as the shared reward it's built from: every agent receives the same global TD error regardless of what it, specifically, did, and as more agents explore simultaneously that gradient gets noisier for everyone. So second, COMA replaces the shared TD-error baseline with a counterfactual baseline computed separately for each agent: for agent a, take the critic's Q-value for the joint action actually taken, and subtract the expectation of that same Q-value under agent a's own policy, holding every other agent's action fixed at whatever it actually did.

Aa(s, τ, u) = Q(s, τ, u) − Σu′ᵃ πa(u′a|τa) Q(s, τ, (u−a, u′a)) // Eq. 4 — agent a's counterfactual advantage: the actual joint action's Q-value, minus what the critic expects if only a's own action were resampled from its policy while everyone else's action u⁻ᵃ stays exactly as it was

This is precisely the difference-rewards idea — reward agent a only for the part of the outcome its own action is responsible for, not the whole team's — but computed by querying the learned critic instead of re-simulating the world with a hand-chosen default action, and because a policy-gradient baseline's expected contribution to the gradient is always zero regardless of what it depends on, COMA avoids the recursive self-consistency problem that earlier value-based attempts at this same idea (the "aristocrat utility") ran into. Third — and this is what keeps the counterfactual term computationally cheap rather than requiring |U| separate critic evaluations per agent per timestep — the critic is architected so that the other agents' actions u−a are part of its input, and it outputs a Q-value for every one of agent a's |U| possible actions in a single forward pass (Figure 1c). The sum in Eq. 4 then costs one extra dot-product against the actor's own action distribution, not |U| separate network queries — and because the same critic is reused for every agent, all agents' advantages can be computed in one batched pass. The resulting COMA policy gradient (Lemma 1) sums each agent's log-probability gradient weighted by its own counterfactual advantage, gk = Eπ[Σa ∇log πa(ua|τa)Aa(s, τ, u)], and converges to a locally optimal joint policy under the same assumptions as single-agent actor-critic convergence (Sutton et al. 1999) — though see the critique below for a caveat on that proof's history.

// what they found

The results

COMA is evaluated on decentralized StarCraft unit micromanagement — a deliberately harder version of the standard StarCraft RL setup. Instead of a full game state and the engine's built-in attack-move macro-actions, each unit gets only a restricted field of view equal to its weapon's firing range and must issue raw move/attack/stop/noop commands itself, introducing genuine partial observability and removing the auto-pathing crutch. Across four combat scenarios — 3 marines (3m), 5 marines (5m), 5 wraiths (5w), and 2 dragoons + 3 zealots (2d_3z) — averaged over 35 independent training runs and the final 1000 evaluation episodes, COMA's mean win rate beats every decentralized baseline on every map: 87±3% on 3m (vs. 83±3% for the strongest ablation, central-V, and 56±6% / 47±3% for the independent IAC-Q / IAC-V baselines), 81±5% on 5m (vs. 71±9% central-QV), 82±3% on 5w (vs. 76±1% central-QV), and 47±5% on the hardest, most heterogeneous map, 2d_3z (vs. 39±5% central-QV, and only 19±21% / 27±9% for IAC-Q / IAC-V — a confidence interval so wide on 2d_3z that it's barely informative). The central-QV ablation is the sharpest test of the counterfactual baseline specifically: it keeps the same centralized critic but replaces Eq. 4's counterfactual term with a plain learned Q−V advantage, and COMA beats it on every map in both training speed and final performance — the paper's direct evidence that the counterfactual baseline, not just centralization, is doing the work.

// does it hold up?

The eval read

COMA and the contemporaneous MADDPG (Lowe et al. 2017) are the two 2017 answers to "give agents a centralized critic during training" — and COMA's own related-work section draws the contrast directly: "(Lowe et al. 2017) concurrently propose a multi-agent policy-gradient algorithm using centralised critics. Their approach does not address multi-agent credit assignment. Unlike our work, it learns a separate centralised critic for each agent and is applied to competitive environments with continuous action spaces." That's an accurate read of the trade, but it's asymmetric in a way worth naming: one shared critic vs. one critic per agent is the crux. COMA's single critic is reused across every agent (only the other agents' actions and this agent's own history change what's queried), which is parameter-efficient and comes with a purpose-built credit-assignment term MADDPG's paper openly admits it lacks. But a single shared critic implicitly assumes there's one team reward to centralize over — exactly why COMA can't handle MADDPG's competitive or mixed-reward settings without redesign, while MADDPG's per-agent critics handle adversarial rewards natively at the cost of N separate critic networks whose combined parameter count and training cost scale with the number of agents. Neither paper benchmarks against the other — the contrast is entirely narrative, on two different environments (StarCraft micromanagement vs. continuous-space particle worlds), so "COMA solves credit assignment MADDPG doesn't" and "MADDPG handles competition COMA can't" are both true and neither is empirically demonstrated head-to-head. On the scaling question specifically, COMA's authors did test it: "We experimented with critic architectures that are factored at the agent level... we found that the bottleneck for scalability was not the centralisation of the critic, but rather the difficulty of multi-agent exploration. Hence, we defer further investigation of factored COMA critics to future work." That's an honest finding, but it means the factored-critic numbers were never published, and the single shared critic's own behavior past five agents is untested here — the same open scaling question MADDPG's paper carries about its per-agent critics, deferred by both 2017 papers to work neither delivers in this evaluation. Separately, the DQN/GMEZO comparison in Table 1 isn't a controlled experiment — those baselines get the full state and macro-actions COMA's setting is explicitly designed to deny its own agents — and the one scenario with heterogeneous unit types, 2d_3z, is exactly where COMA's mean performance (47%) loses outright to a hand-coded heuristic (63%), the weakest point in the "competitive with centralized state-of-the-art" headline. Finally, a footnote worth knowing: the arXiv record for this paper carries an errata section — the original proof of Lemma 1 (COMA's convergence guarantee) had an error that went uncorrected until outside researchers (Frans Oliehoek, Chris Amato) flagged it years later, and a line of follow-up work (Lyu et al., 2021–2024) has since studied whether centralized critics like COMA's actually converge to good solutions in general — meaning the theoretical guarantee wasn't as settled at publication as the original AAAI paper's Lemma 1 implied.

baselines

IAC-V/IAC-Q and central-V/central-QV form a genuinely clean ablation ladder — each isolates exactly one of COMA's three design ideas (centralization, learning Q vs. V, and the counterfactual baseline specifically) — but the "competitive with state-of-the-art centralized controllers" claim leans on Usunier et al.'s GMEZO/DQN numbers from a strictly easier problem (full field of view, built-in attack-move macros), not a controlled comparison; MADDPG, the paper's own named contemporaneous rival, is discussed but never benchmarked against directly.

contamination

Not applicable in the train/test-leakage sense — this is from-scratch RL training, not a static test set — but the four scenarios (3m, 5m, 5w, 2d_3z) were chosen by the same lab that also designed the harder decentralized field-of-view variant of the benchmark, so there's no independent check that the difficulty tuning wasn't shaped around what COMA happens to do well.

what's missed

Scaling past five agents — the authors tried a factored per-agent critic, attributed the scalability bottleneck to exploration rather than the critic, but never published those numbers; any competitive or mixed-reward setting, since a single shared critic presumes one team reward to centralize over; and the one scenario with mixed unit types and ranges (2d_3z), where COMA's mean (47%) loses outright to the hand-coded heuristic (63%).

would raise confidence

A shared-benchmark, head-to-head run against MADDPG; published results for the deferred factored/agent-level critic variant at agent counts well past five; an actual continuous-action or competitive-reward experiment testing the "easily extended" claim for Eq. 4; and note that Lemma 1's original convergence proof had an error uncorrected for years, only fixed after outside researchers flagged it — a re-verified proof at publication time, rather than years later, would have mattered.

// bottom line

COMA is the cooperative-only mirror image of MADDPG's problem: the same centralized-training/decentralized-execution shell, but where MADDPG gives each agent its own critic to handle arbitrary — including adversarial — rewards, COMA shares one critic across all agents and spends the savings on an explicit counterfactual credit-assignment term MADDPG's own paper doesn't have. Both design choices are principled, and both papers defer the identical open question — does the recipe survive contact with far more than five agents? — to future work neither delivers here, a question nearly a decade of follow-up work (including a proof correction as late as 2024) suggests was harder than either 2017 paper's evaluation could show.

// takeaways

What to remember

  1. The counterfactual baseline — Aa(s,τ,u) = Q(s,τ,u) minus the expectation of Q over agent a's own action distribution with everyone else held fixed — turns the classical difference-rewards idea (compare the outcome to what would've happened under a default action) into something computable from one learned critic in a single forward pass, instead of re-simulating the environment once per agent — the trick that makes credit assignment tractable in a game (StarCraft) where re-simulating counterfactuals isn't possible at all.
  2. One shared critic, not one per agent: the central-QV ablation shows the counterfactual baseline specifically (not just centralizing the critic) is responsible for the gains — COMA beats central-QV on every map, in both training speed and final win rate — but the single-critic design presumes one team reward to centralize over, which is exactly why COMA can't handle MADDPG's competitive settings without redesign, and exactly why the two methods were never run head-to-head.
  3. The "competitive with fully-centralized state-of-the-art" claim is scenario-dependent: COMA's best decentralized runs match or approach GMEZO/DQN with full field-of-view on 3 of 4 maps, but on the one map with heterogeneous unit types (2d_3z), even a hand-coded heuristic beats COMA's mean performance — a reminder that one table's average win rate can hide the scenario where a method's advantage disappears.
