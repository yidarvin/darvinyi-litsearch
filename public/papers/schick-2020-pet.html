Exploiting Cloze-Questions for Few-Shot Text Classification and Natural Language Inference

Reformulate a task as a cloze question, finetune a separate small model per pattern on as few as 10 labeled examples, then distill the ensemble's soft labels on a pile of unlabeled data into one classifier. This paper — posted months before GPT-3 — pushes AG's News accuracy from 25.0% to 87.5% and Yahoo Questions from 10.1% to 63.8%, the pre-FLAN prompt-based finetuning recipe (PET) that GPT-3, FLAN, and the few-shot-calibration literature all cite directly.

// the gap

What problem is this solving?

Standard supervised finetuning is data-hungry: with only a handful of labeled examples, it often can't beat chance. The paper's own numbers make this concrete — with just 10 labeled examples (|T|=10), plain supervised RoBERTa-large finetuning reaches only 21.1% on Yelp Reviews (5-way), 25.0% on AG's News (4-way, barely above the 25% random-guess floor), 10.1% on Yahoo Questions (10-way, at the 10% chance floor), and 34.2%/34.1% on MNLI (3-way, near the 33% floor). Meanwhile, a separate line of work (Radford et al., 2019; Puri and Catanzaro, 2019) had shown that giving a pretrained language model a natural-language "task description" — appending a hint and reading off its next-token prediction — can solve some tasks with zero labeled examples, but this zero-shot approach clearly underperforms a properly supervised model once real training data exists.

The paper motivates the gap with a simple example: given two labeled texts ("This was the best pizza I've ever had" → l; "You can get better sushi for half the price" → l′) and a third unlabeled one ("Pizza was average. Not worth the price."), the correct label is genuinely ambiguous from the examples alone — plausible justifications exist for both l and l′. But if you're told the task is "does the text say anything about price," the third example trivially resolves to l′. Nobody had combined this task-description intuition with actual few-shot supervised training on classification and NLI tasks before — zero-shot prompting and few-shot finetuning had been developed as separate techniques, each with a different failure mode, and PET's aim is to combine them so the weaknesses cancel out.

// how it works

The method

Pattern-Exploiting Training (PET) defines a pattern P: a function that turns an input x into a phrase containing exactly one mask token (a cloze question), and a verbalizer v: an injective mapping from each task label to a single vocabulary word. Together (P, v) form a pattern-verbalizer pair (PVP). For recognizing whether two sentences agree or contradict, a PVP might use the pattern a? ___, b. with verbalizer v(contradict)="Yes", v(agree)="No" — so "Mia likes pie" / "Mia hates pie" becomes the cloze question "Mia likes pie? ___, Mia hates pie." and the model's job is just to decide whether "Yes" or "No" is the more likely fill.

Given a PVP, the unnormalized score for label l is the masked-LM score M(v(l) | P(x)) for its verbalizer word at the mask position; softmax over labels gives a probability, and cross-entropy against the true label on the small training set T is the finetuning loss for that PVP's model. Because T is tiny, catastrophic forgetting is a real risk, so PET adds an auxiliary masked-language-modeling loss (computed on the unlabeled set D, never asking the model to predict the actual mask slot) alongside the classification loss:

L = (1 − α) · LCE + α · LMLM // α = 10⁻⁴ throughout, chosen once in preliminary experiments (L_MLM is naturally much larger than L_CE)

PET's full three-step procedure (Figure 1) is: (1) define a set of PVPs that intuitively make sense for the task, and finetune a separate pretrained LM per PVP on T — cheap, since T is small, even for many PVPs (each config is trained 3× with different seeds). (2) Use the ensemble of finetuned models to soft-label a large unlabeled set D: combine each model's unnormalized class scores with weights w(p) (either uniform, or the PVP's own pre-finetuning training-set accuracy — the "weighted" variant), then apply a temperature-T=2 softmax (following Hinton et al.'s 2015 knowledge-distillation recipe) to get soft targets. (3) Finetune a fresh model C with a standard classification head on this soft-labeled set TC — this is the final classifier the method ships.

Distilling every PVP into a single classifier means the PVPs can no longer correct each other's mistakes after the fact, and some patterns perform much worse than others — so the soft-labeled set can contain a lot of noise. iPET (iterative PET) addresses this by training k generations of models instead of one: starting from the initial ensemble M0, each generation's models are relabeled by a random subset of λ=25% of the previous generation's other models, growing the effective training set by a factor d=5 each round (while preserving T's original label ratio, and preferring examples the ensemble is more confident about) until every model has seen at least 1,000 examples. The final generation's ensemble then produces TC and trains C exactly as in vanilla PET.

Experiments use RoBERTa (large) for four English tasks — Yelp Review Full, AG's News, Yahoo Questions (all from Zhang et al., 2015), and MNLI — and XLM-R (base) for x-stance, a German/French/Italian stance-detection dataset, to test whether the recipe transfers beyond English. Each task gets 2–6 hand-written patterns; Yelp's four patterns include P₁(a) = It was ___. a with a shared verbalizer mapping ratings 1–5 to "terrible"/"bad"/"okay"/"good"/"great". Hyperparameters (learning rate 1·10⁻⁵, batch size 16, max sequence length 256) were fixed from prior work rather than tuned on a held-out dev set, honoring the paper's own few-shot assumption of no large validation set.

// what they found

The results

With |T|=10 — the headline few-shot regime — PET lifts RoBERTa-large's accuracy from a near-chance supervised baseline to 52.9% (Yelp), 87.5% (AG's News), 63.8% (Yahoo), and 41.8%/41.5% (MNLI m/mm); iPET pushes further to 57.6%, 89.3%, 70.7%, and 43.2%/45.7%. Even the fully zero-shot version of iPET — no labeled examples at all, just the ensemble of untrained-then-bootstrapped PVPs — reaches 56.7% / 87.5% / 70.7% / 53.6–54.2%, which on AG's News actually exceeds standard supervised training with 1,000 labeled examples (86.9%). As training-set size grows the gap narrows but doesn't vanish: at |T|=1000, PET still improves over supervised training on Yelp (64.8 vs 63.0), Yahoo (72.7 vs 70.5) and MNLI (85.3/85.5 vs 73.1/74.8), and is roughly tied on AG's News (86.9 vs 86.9).

Against dedicated semi-supervised SOTA baselines UDA (Xie et al., 2020) and MixText (Chen et al., 2020) — both of which require back-translation machine translation models trained on thousands of parallel sentences — PET and iPET win clearly using RoBERTa-base (a smaller backbone, chosen because MixText is architecture-specific to a 12-layer transformer): at |T|=10, UDA reaches 27.3/72.6/36.7/34.7 and MixText 20.4/81.1/20.6/32.9 on Yelp/AG's/Yahoo/MNLI, versus PET's 48.8/84.1/59.0/39.5 and iPET's 52.9/87.5/67.0/42.1 — and notably, the paper tuned UDA's and MixText's hyperparameters directly on the test set (no dev set was used for them either), which if anything favors the baselines.

On x-stance (German/French/Italian, XLM-R base), PET's advantage holds even with much larger training sets: at |T|=1000, supervised training gets 43.3/49.5/41.0 (De/Fr/It) versus PET's 66.4/68.7/64.7, and even training on the entire available German+French data jointly (45,640 examples total), PET still edges out supervised training — 78.8/80.6/77.2 vs 77.6/79.1/75.9.

// does it hold up?

The eval read

The instinctive comparison to reach for with a 2020 few-shot paper — "how does this stack up against GPT-3's in-context learning?" — doesn't actually apply here. This paper (arXiv v1: January 2020) predates GPT-3 (May 2020) entirely, and even its January 2021 EACL camera-ready revision never mentions GPT-3 once. Its actual baselines are zero-shot pattern scoring (Radford et al., 2019-style, no training) and two semi-supervised SOTA methods, UDA and MixText — not pure prompting at any scale. The now-common narrative that "PET beats GPT-3 with a fraction of the parameters" belongs to Schick and Schütze's own NAACL 2021 follow-up, It's Not Just Size That Matters, which explicitly benchmarks PET against GPT-3 on SuperGLUE — a different paper with a different arXiv ID, easy to conflate with this one since both are "PET."

Is the semi-supervised setup actually comparable to "few-shot learning" as later popularized? PET is not the no-gradient, no-extra-data setting GPT-3 popularized — it requires full gradient-based finetuning of several separate models per task (one per pattern, ×3 seeds, ×k generations for iPET) plus a large pool of unlabeled in-domain data D to distill onto. The paper is honest about calling itself "semi-supervised," but the headline "52.9% with 10 examples" framing invites a reader to compare it directly to prompting methods that get neither the extra unlabeled corpus nor the compute for multiple finetuning runs. The authors do run one control for this: Figure 5 further-pretrains RoBERTa on Yelp's own unlabeled review text (an unrelated way of using the same extra data) and shows the resulting supervised+PT baseline still clearly trails PET at every training-set size — evidence PET's edge isn't simply "access to more unlabeled text." But that ablation is run on one dataset (Yelp) "as language model pretraining is expensive in terms of GPU usage" (the paper's own words) — it's not repeated for AG's News, Yahoo, or MNLI, so how much of PET's gain on those three tasks is attributable to D itself versus the pattern-based finetuning trick is untested.

Backbone and tuning asymmetries between the two results tables. Table 1's headline supervised-vs-PET comparison uses RoBERTa-large throughout, but Table 2's comparison against UDA and MixText switches to RoBERTa-base — a reasonable accommodation (MixText's architecture is tied to a 12-layer transformer) that the paper states plainly, but it means the "beats SOTA" numbers and the "beats supervised" numbers aren't on the same backbone, so the two margins shouldn't be read as directly stacking. In UDA/MixText's favor, though: their hyperparameters were tuned directly on the test set while PET used no dev-set tuning at all — a rare case of a paper handicapping its own method's baselines correctly rather than sandbagging them.

The method still leans on hand-designed patterns, and that cost is invisible in the headline numbers. Table 4 shows that at |T|=10, individual PVPs vary hugely: the worst single pattern per task ("min") scores 39.6/82.1/50.2/36.4 on Yelp/AG's/Yahoo/MNLI, while the best ("max") scores 52.4/85.0/63.6/40.2 — a 13-point swing on Yelp alone. PET's ensemble-and-distillation step compensates for this variance, but only because the authors supplied several reasonable patterns per task up front; a practitioner without Schick and Schütze's task intuition, or applying PET to a task where good cloze patterns aren't obvious, has no guarantee of landing anywhere near these numbers. The paper doesn't quantify how sensitive results are to pattern quality in general, only to which of its own hand-picked set happens to do best or worst.

baselines

Reasonable and mostly good-faith: zero-shot pattern scoring, standard supervised finetuning, and two real semi-supervised SOTA methods (UDA, MixText) whose hyperparameters were tuned directly on the test set (favoring them, not PET). The backbone silently changes from RoBERTa-large (Table 1) to RoBERTa-base (Table 2), so the two headline margins aren't directly comparable to each other.

contamination

Not addressed — Yelp Review Full, AG's News, Yahoo Questions, and MNLI are all older, widely circulated academic datasets that RoBERTa/XLM-R's web-scale pretraining corpora plausibly contain fragments of. This predates today's contamination-checking norms, and it affects PET and its supervised baseline symmetrically, but the absolute accuracy numbers shouldn't be read as clean generalization scores.

what's missed

The in-domain-pretraining control isolating "extra unlabeled data" from "pattern-based training" (Figure 5) is run on Yelp only, not the other three English tasks; pattern/verbalizer sets are hand-designed per task with no systematic study of how much expertise that requires; only 3 random seeds per configuration; no comparison at all to in-context prompting at any model scale (impossible at the time — GPT-3 postdates this paper).

would raise confidence

Extend the in-domain-pretraining ablation to AG's News, Yahoo, and MNLI; report a systematic pattern-quality sensitivity study rather than just best/worst of the hand-picked set; and a contamination check given how old and well-circulated these four datasets are.

// bottom line

A genuinely important prompt-based finetuning precursor — the pattern/verbalizer vocabulary and the ensemble-then-distill recipe it introduces are still visible in later instruction-tuning work — and its own numbers (near-chance supervised baselines lifted to 60–90% with 10 labeled examples) are real and well-documented. But it is a semi-supervised method that needs both hand-written patterns and a large unlabeled corpus, evaluated against zero-shot scoring and two SOTA semi-supervised baselines — not the pure in-context learning its "few-shot" framing evokes in hindsight. Read the GPT-3 comparison into the sequel paper, not this one.

// takeaways

What to remember

  1. PET reformulates a task as a cloze question (a pattern + verbalizer pair), finetunes a separate small model per pattern on as few as 10 labeled examples, then distills the ensemble's soft labels on unlabeled data into one final classifier — lifting near-chance supervised baselines (10.1% on Yahoo, 25.0% on AG's News at |T|=10) to 63.8% and 87.5%.
  2. It's a semi-supervised, gradient-based method — not pure in-context prompting. It predates GPT-3 by months and never mentions it; the popular "PET beats GPT-3 with far fewer parameters" story belongs to Schick and Schütze's NAACL 2021 sequel, It's Not Just Size That Matters, evaluated on SuperGLUE.
  3. It's a foundational prompt-based finetuning precursor: GPT-3's own paper, FLAN (wei-2021-flan), and the "Calibrate Before Use" line of few-shot evaluation work all cite it directly, and its pattern/verbalizer framing plus ensemble-and-distill recipe carry into later instruction-tuning methods.
