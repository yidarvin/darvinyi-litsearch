Teaching Language Models To Gather Information Proactively

Most assistants, handed an under-specified prompt, answer anyway. This paper reframes the missing context as something to go get — and reward-finetunes a 7B model to ask the clarifying questions that pull genuinely hidden requirements out of the user before it writes.

Tenghao Huang, Sihao Chen, Muhao Chen, Jonathan May, Longqi Yang, Mengting Wan, Pei Zhou · Microsoft / USC / UC Davis

// the gap

What problem is this solving?

Users increasingly treat LLMs as collaborators rather than oracles — Anthropic's education telemetry (Handa et al., 2025) found that more than half of student interactions across disciplines are multi-turn, back-and-forth exchanges rather than one-shot prompts. But the prompts people bring to those exchanges are lazy: incomplete, under-specified, and full of information asymmetry. The user knows the domain conventions, the format they actually want, and the constraints that go unsaid; the model sees only a fraction of that and is expected to fill in the rest.

Off-the-shelf models handle this badly in a specific way. When they do ask a clarifying question, they tend to ask about what is already on the table — restating the visible context ("what learning outcomes do you want?") — or they fall back on generic queries that don't move the conversation forward. That is the descendant of classic slot-filling and ambiguity resolution, where the job is to disambiguate something already present in the input. The skill this paper targets is different and harder: proactively gathering information that is missing — unstated domain expertise, fine-grained requirements, the format nobody mentioned — especially in open-ended domains like social science or business where crucial details are routinely left implicit and no single correct answer exists.

Why hasn't this been trained directly? Two obstacles. First, data: high-quality collaborative dialogue is scarce, organic logs are noisy and proprietary, and crowdsourcing nuanced domain exchanges is expensive. Second, and more fundamental, reward: the qualities that make a question "good" — helpfulness, novelty, contextual complementarity — are subjective and resist standard reward signals. Prior RL-for-LLM work leaned on verifiable domains (math, code) with dense or easily-checked rewards; open-ended writing offers only sparse, uninformative supervision.

// how it works

The method

Formally, the user holds explicit information E (stated goals, facts, constraints) and implicit information I (unstated assumptions, domain conventions, fine-grained requirements). The ideal output y* = f(E, I) depends on both, but the model only sees E. So it must build its own estimate Î of the missing pieces through dialogue and draft ŷ = f(E, Î), minimizing the gap to y*.

Manufacturing ambiguity from expert tasks

The training data is built by masking, not crowdsourcing. The authors adapt DOLOMITES (Malaviya et al., 2024) — 519 long-form, domain-specific writing templates elicited from 266 professionals across 25 fields (medicine, law, civil engineering, …). Each instance is a quadruple ⟨o, p, i, s⟩: task objective, task procedure (domain steps), input section (facts/data), and output specification (required format and subsections). The split is the trick: the model is shown only the explicit half E = ⟨o, i⟩ — what a lazy user would actually type — while the implicit half I = ⟨p, s⟩, the procedure and the output spec, is hidden. To produce a good answer, the model has no choice but to ask for it.

A partially-observable game with an LLM user

Each episode is framed as a POMDP. The assistant sees only E; a second LLM, the user oracle, sees both E and I. Over up to n ≤ 5 turns the assistant asks a clarification question qt, the oracle answers from the hidden information, and the running dialogue Dt becomes the assistant's estimate Î. After the turn budget runs out or the assistant issues STOP, it produces the final draft. A fixed vanilla GPT-4o is the writer at this draft phase — so the model being trained is purely the question-asker.

The core innovation: an evidence-sentence reward

Scoring the final 500-token essay against a non-exhaustive reference gives a sparse, noisy reward — useless early in training when good questions are rare. Instead, the paper rewards each question directly, by whether its answer touches genuinely hidden content. Split the implicit information into sentences with hidden-field indices H. For a question qt, prompt the user oracle to return the set of sentence indices A(qt) it would cite to answer it. The reward is binary:

rt(qt) = 1  if  A(qt) ∩ H ≠ ∅,   else 0 // a question is rewarded iff answering it requires citing a masked field — i.e. it pulls in information not already in E

This sidesteps span-level annotation entirely: the oracle's own citation behavior is the label. That dense-ish, per-turn signal trains the policy with PPO in an actor–critic setup (Schulman et al., 2017), via the verl framework. Concretely: Qwen-2.5-7B, 3 epochs, 8×A100, a 5-turn budget, batch size 256 episodes (minibatch 16), PPO clip 0.2, separate learning rates (actor 2×10⁻⁵, critic 1×10⁻⁴), gradient-norm clip 1.0.

Evaluation

The hidden output specification is distilled into a checklist of items {c1, …, cm}. A frozen LLM judge scores each item MATCH(cj, ŷ) ∈ {0,1}, and the writing score is the fraction satisfied — no credit for omitted or mis-formatted items. So the final metric measures how much of the masked output spec the model managed to recover and deliver.

// what they found

The results

The reward-finetuned questioner wins. Qwen-2.5-RFT scores 0.65, the top of every comparison and the +18% relative gain over o3-mini that the paper headlines (significant at p < 0.05; o3-mini sits at ≈0.55 read off the Fig. 4 bar chart — the paper prints only the relative figure and RFT's 0.65). Every method here pairs a questioner with the same fixed GPT-4o writer, so the gap is attributable to question quality: a 7B model asking better questions beats o3-mini asking its own.

SFT didn't just fail to help — it hurt. Both supervised variants fine-tuned on synthetic multi-turn conversations (GPT-4o-SFT 0.47, Qwen-2.5-SFT 0.46) scored below their own base models (0.51, 0.50). Imitating model-generated question traces taught the wrong thing; only outcome-based reward optimization moved the needle. The strongest non-RL baseline, GPT-4o-Indirect — trained with DPO on real human brainstorming turns mined from QMSUM meeting transcripts — reached a competitive 0.57, confirming that exposure to genuine human clarification helps, but still trailed RFT.

Where the gains concentrate

Per domain (Table 1 in the paper), RFT scores 0.83 in social science, 0.53 in technology, 0.71 in humanities. Against the direct baseline that is +0.37 in social science and +0.31 in humanities — the open-ended fields where the "right" answer is underdetermined and depends most on eliciting unstated intent — versus a smaller lift on the more procedural technology tasks. Proactive questioning pays off exactly where context is most implicit.

Two more analyses round it out. On dialogue length, Qwen-RFT keeps improving up to its 5-turn peak and sustains it, while GPT-4o and GPT-4o-SFT plateau after 3 turns — the reward-trained model actually uses extra turns productively. And human evaluation (3 graduate annotators, 30 side-by-side items, against o3-mini) preferred RFT-Qwen's questions 62% of the time (18% tie, 20% loss) and its outlines 50% (28% tie, 22% loss). The abstract's "42% and 28%" are these as net margins (wins minus losses), not raw win rates.

// does it hold up?

The eval read

The cleverness is real — the evidence-sentence reward is a genuinely nice way to wring a dense, per-turn signal out of an open-ended task without span annotation. But the headline number lives inside a tightly closed loop, and the showcase comparison leans on a quietly load-bearing design choice.

The deepest issue is target overlap between reward and metric. The reward fires when a question elicits evidence from the masked fields I = ⟨p, s⟩; the final metric is checklist coverage of the masked output spec s. The model is trained to surface exactly the content it is later graded on recovering. That is not circular in a fatal way — recovering hidden requirements is the task — but it means 0.65 measures "how well RFT reconstructs the specific masked spec," a narrower claim than "writes better." And the entire pipeline is LLM-in-the-loop: the user oracle, the reward labeler, and the judge are all language models, with the oracle answering faithfully from full ground-truth I. Real users are noisy, uncertain, and sometimes wrong; a perfectly cooperative simulated user likely overstates real-world gains.

The "7B beats o3-mini" framing also deserves an asterisk. Because a fixed GPT-4o does all the drafting, the 7B model is only the question-asker — a fair isolation of question quality, but the end-to-end output quality rides on GPT-4o's pen, not Qwen's. Finally, the reward is coarse: +1 for touching any hidden field, with no notion of which missing piece matters most, no question ordering, and no penalty for redundant or spammy questions — a binary signal that can't distinguish an efficient, well-targeted clarification from a scattershot one.

baselines

Current and varied — o3-mini, GPT-4o, two SFT variants, and a DPO model trained on real meeting transcripts — all evaluated as questioners feeding the same fixed GPT-4o writer. A clean isolation of question quality, but it means the 7B "win" is a questioner result, with GPT-4o doing the writing.

contamination

DOLOMITES is public (2024), so base models may have seen the templates; the masked-reconstruction setup blunts memorization since the graded content is hidden, but no within-benchmark train/eval template split is described.

what's missed

A binary "touched a hidden field" reward ignores question importance, ordering, and redundancy, and never penalizes over-asking. The oracle answers more faithfully than any real user; everything is graded by LLMs on a single benchmark.

would raise confidence

A second benchmark; noisier or human users; a reward that ranks and penalizes questions rather than thresholding; decoupling the questioner from the GPT-4o writer; and a larger human study against more opponents reporting raw win rates.

// bottom line

A well-isolated proof that you can reward-finetune a small model to ask better questions, carried by a clever sparse-to-dense reward. But the result sits inside one benchmark where the reward, the simulated user, and the grader all point at the same masked fields, and the marquee "7B beats o3-mini" rides on a constant GPT-4o writer. Promising recipe; the generalization is still owed.

// takeaways

What to remember

  1. Proactive information gathering ≠ clarification. The target isn't resolving stated ambiguity (slot-filling — "what did you mean by X?") but eliciting unstated, missing requirements ("have you considered Y?"). It matters most in open-ended domains where the right answer is underdetermined.
  2. The evidence-sentence reward is the trick. Rather than grade a 500-token essay against a thin reference, reward each question by whether the user oracle would cite a hidden field to answer it — a binary, annotation-free signal that turns a sparse open-ended objective into a dense per-turn one.
  3. RFT > SFT, and SFT actively hurt. Fine-tuning on synthetic conversation traces scored below the base models; only PPO on outcome-based reward improved question quality. Imitation taught the wrong behavior here.
  4. Watch the closed loop. The reward, the simulated user, the judge, and even the final writer are all LLMs, and the reward target overlaps the eval target — so read 0.65 as "recovers the masked spec well," and treat the small-model-beats-o3-mini line as a question-quality claim, not end-to-end superiority.
