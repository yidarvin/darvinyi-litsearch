Instruction-Following Evaluation for Large Language Models

Drop the human judge and the LLM judge. IFEval scores instruction-following only on verifiable instructions — "write at least 400 words," "respond in JSON," "mention the keyword AI at least 3 times" — that a short deterministic program can check. 541 prompts, 25 instruction types, fully reproducible.

What problem is this solving?

Following natural-language instructions is the headline capability of an instruction-tuned LLM — and yet, when this paper was written, there was no cheap, standardized, reproducible way to measure it. The obstacle is that human language is ambiguous. Tell a model to "write with a funny tone" or "give detailed reasoning but don't over-explain," and there is no objective ground truth for whether it complied; two annotators will disagree.

The authors lay out the three families of evaluation in use, each with a structural flaw:

  • Human evaluation (as in InstructGPT, MT-Bench, Alpaca) is expensive, slow, and not objectively reproducible — different annotators, different verdicts.
  • Model-based / LLM-as-judge evaluation is only as good as the judge: it inherits the evaluator model's blind spots and biases, and a contemporaneous result — "Large language models are not fair evaluators" — showed judges can be swayed by position and verbosity.
  • Quantitative benchmarks are standardized and scalable, but the ones that existed mostly scored task correctness (does the answer pass the unit test, is the math right), not whether the model obeyed the formatting and constraint instructions wrapped around the task.

IFEval's wedge is to give up generality in exchange for objectivity. Instead of trying to grade every instruction, it restricts the test set to verifiable instructions — directives whose compliance a simple, interpretable, deterministic program can check. "Write at least 400 words" is verifiable: count the words. "Be funny" is not. By confining the benchmark to the verifiable slice, the score becomes automatic, judge-free, and byte-for-byte reproducible — at the cost of saying nothing about the unverifiable instructions that also matter.

// how it works

The method

The unit of the benchmark is the verifiable instruction: an atomic directive that a deterministic function can label as followed or not. The authors hand-build 25 instruction types organized into 9 groups — Keywords (include / forbid / count words and letters), Language (respond entirely in language X), Length Constraints (N words / sentences / paragraphs), Detectable Content (add a postscript, include N bracketed placeholders), Detectable Format (exactly N bullets, a <<title>>, JSON output, N highlighted sections), Combination, Change Case, Start/End, and Punctuation. Each type carries many variants, both in its parameters ("450–500 words" vs. "350–400 words") and in its phrasing ("write 450 to 500 words" vs. "your response must contain 450 to 500 words"), so a model can't pattern-match a single template.

From these, they assemble 541 prompts, each bundling one or more verifiable instructions onto an otherwise ordinary writing task (Figure 1). A model's response is fed to a per-instruction verifier program, and compliance is the boolean:

is_followed(resp, inst) = True  if the program confirms inst in resp
False otherwise // Eq. 1 — the "strict" criterion: a deterministic check, no judge model

Strict vs. loose: taming false negatives

A naive string check is brittle. Asked to "end your email with: P.S. I do like the cake," a model might write "P.S. I do like the cake" — correct, but the markdown bold (**) breaks an exact match and the verifier wrongly scores it as a failure. To absorb these formatting artifacts the authors add a loose criterion: transform the response a handful of ways and count it as compliant if any transform passes.

is_followedloose(resp, inst) = ⋁t  is_followed( transformt(resp), inst ) // Eq. 2 — followed if it passes under any of t = 1 … 8 transforms

There are three base transforms — (1) strip markdown emphasis (*, **), (2) drop the first line (to skip intros like "Sure, here it is:"), (3) drop the last line (to skip outros like "Hope it helps.") — plus every pairing, all three together, and the identity transform: 8 in total. Loose accuracy is deliberately a complement to strict, not a replacement: the authors note it trades false negatives for false positives (dropping the first line can let a response that violates a word count slip through), so strict is a lower bound on true compliance and loose an upper bound.

Two granularities of scoring

Each criterion is reported at two resolutions, giving the benchmark's four headline numbers:

  • Prompt-level accuracy — the fraction of prompts for which all verifiable instructions are satisfied. One miss zeroes the whole prompt; this is the harsh, all-or-nothing view.
  • Instruction-level accuracy — the fraction of individual instructions satisfied, partial credit allowed.

Crossing {prompt, instruction} × {strict, loose} yields prompt-strict, instruction-strict, prompt-loose, and instruction-loose. The prompts and the full verifier code are released open-source, so anyone can reproduce a model's four numbers exactly.

// what they found

The results

The paper reports two reference models scraped through their APIs: GPT-4 (responses from November 2023) and PaLM 2 Small (August 2023). The headline is that even the strongest model of the day did not reliably follow simple, mechanically-checkable instructions: GPT-4 satisfied every instruction in a prompt only 76.89% of the time (prompt-level strict) — it dropped roughly one prompt in four. PaLM 2 S managed 43.07%. The two are not a fair head-to-head (very different parameter counts and snapshot dates), so read these as calibration anchors, not a leaderboard.

Model Prompt strict Inst strict Prompt loose Inst loose
GPT-4 (Nov 2023)76.89%83.57%79.30%85.37%
PaLM 2 S (Aug 2023)43.07%55.76%46.95%59.11%

Three patterns fall out of the numbers:

  • Prompt-level is much harsher than instruction-level. GPT-4 follows 83.57% of individual instructions but only 76.89% of prompts in full — the gap is the price of the all-or-nothing rule, which compounds across the multiple instructions per prompt.
  • Loose lifts scores only a little. GPT-4 goes 76.89% → 79.30% (prompt) and 83.57% → 85.37% (instruction) — a few points. So most "failures" are genuine non-compliance, not just bold-text or intro-line artifacts; the formatting fudge accounts for only ~2–3 points.
  • Difficulty is uneven across instruction types. Both models are near-perfect on language (respond entirely in language X), ~0.96 each. The pain is in combination, length constraints, and punctuation, where even GPT-4 sags into the 0.66–0.74 range.
// does it hold up?

The eval read

IFEval's lasting contribution is methodological, not empirical: a way to score instruction-following with zero judges and perfect reproducibility. That design is also exactly where the caveats live. "Verifiable" is a synonym for "narrow." By construction the benchmark only tests instructions a regex or a counter can check — casing, word counts, keyword presence, output formats. The instructions that actually make instruction-following hard in deployment — tone, helpfulness, faithfulness, "don't over-explain," priority among conflicting asks — are deliberately excluded. A high IFEval score certifies that a model can count to 400 words and emit valid JSON; it does not certify that the model is a good instruction-follower in the open-ended sense the title implies.

The two-model evaluation (GPT-4, PaLM 2 S), which the paper itself flags as "not directly comparable," means the absolute numbers are anchors rather than a ranking — there are no open-weight models, no within-family size sweep, and no analysis of why models fail a given type. And because the verifiers are deterministic programs, the benchmark is inherently gameable: a model — or a fine-tuner optimizing against it — can learn to satisfy the literal checker (always pad to the word count, always emit the markdown divider) without getting better at following intent. That matters because IFEval has since become a standard training target, not just an eval, which is where format-overfitting turns into measured-but-fake progress.

baselines

Only two models, GPT-4 (Nov 2023) and PaLM 2 S (Aug 2023), and the paper concedes they're "not directly comparable" (different sizes, different snapshots). Useful as calibration anchors; not a leaderboard, and no open-weight or same-family comparisons.

contamination

All 541 prompts, the 25 instruction types, and the verifier code are public on GitHub since 2023, and IFEval now sits in many post-training mixes. There is no held-out split and no refresh mechanism, so for any model trained after late 2023 the score partly reflects exposure to the test itself.

what's missed

Everything unverifiable — tone, helpfulness, faithfulness, instruction conflicts, multi-turn carry-over. Instructions are atomic and English-centric; the all-or-nothing prompt-level metric is high-variance on only 541 items. Loose scoring fixes false negatives but admits false positives, so true compliance sits somewhere between strict and loose.

would raise confidence

More and comparable models (same family, open weights) with variance / confidence intervals; a private held-out or periodically-refreshed prompt set to fight contamination; multilingual and compositional instructions; and an explicit check for format-overfitting versus genuine instruction-following.

// bottom line

IFEval nailed one thing and was honest about its scope: judge-free, deterministic, perfectly reproducible scoring of the slice of instruction-following that's mechanically checkable. Treat it as a floor, not a ceiling — a low score is damning, but a high score (now easy to saturate and partly contaminated) is necessary, not sufficient, evidence that a model follows instructions. Its real legacy is the "verifiable instruction" idea, which seeded a wave of rubric- and checklist-based evaluation and reward design.

// takeaways

What to remember

  1. Verifiable instructions are the whole trick. Restrict the test set to directives a deterministic program can check ("≥400 words," "respond in JSON," "mention AI 3×"), and you get instruction-following scores with no human and no LLM judge — cheap, automatic, byte-for-byte reproducible.
  2. Four numbers, two axes. {prompt-level, instruction-level} × {strict, loose}. Loose runs 8 response transforms (strip markdown, drop first/last line + combinations) and passes if any does, killing formatting false negatives; true compliance lies between strict (lower bound) and loose (upper bound).
  3. Even GPT-4 failed ~1 prompt in 4. 76.89% prompt-level strict for GPT-4 (Nov 2023), 43.07% for PaLM 2 S — mechanically-simple constraints (length, combination, punctuation) were surprisingly hard, while "respond in language X" was near-solved.
  4. Read it as a floor. The verifiers are gameable, the 541 prompts are public and now in training mixes, and the benchmark skips every unverifiable instruction that matters in practice. A low IFEval score is damning; a high one is necessary but not sufficient.
