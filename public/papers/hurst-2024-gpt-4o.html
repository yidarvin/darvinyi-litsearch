GPT-4o System Card

OpenAI's own safety audit of GPT-4o — the Preparedness Framework scorecard, four phases of external red-teaming, and two independent third-party assessments run on a text-only stand-in for the omni model everyone actually uses.

OpenAI — Aaron Hurst, Adam Lerer, Adam P. Goucher, Adam Perelman, Aditya Ramesh, et al. (416 authors)
venue arXiv citations 4,615 topic Safety & Red-teaming source ↗

// the gap

What problem is this solving?

GPT-4o is OpenAI's first "omni" model: one network trained end-to-end across text, vision, and audio, so a single set of weights accepts any mix of text, image, audio, and video input and generates any mix of text, audio, and image output. It responds to spoken audio in as little as 232 milliseconds (320ms on average) — close enough to human conversational latency that it supports live, turn-taking voice conversation rather than the transcribe-think-speak pipeline every prior "voice assistant" actually was. That collapse from a three-stage pipeline (speech-to-text → text model → text-to-speech) into one network is exactly what makes this system card necessary: a pipeline system can only leak what survives being flattened into a text transcript, but a model that hears and speaks directly can clone a voice from a few seconds of audio, infer a speaker's accent, gender, or emotional state from tone alone, or generate erotic or violent content in a voice that sounds like a real person — none of which shows up if you only ever red-team the text layer.

The GPT-4 and GPT-4V system cards that came before this one only had to cover text and static images; this is the first OpenAI system card built around a model that can be adversarially probed through a live spoken conversation, at real-time latency, with no text transcript necessarily in the loop at all. The paper is explicit that it inherits and extends that prior text/vision work — "this red teaming effort builds upon prior work, including as described in the GPT-4 System Card and the GPT-4(V) System Card" — but the modality is new enough that no established audio-safety-evaluation tooling existed to reuse. OpenAI had to build it: an external red-teaming program spanning four escalating checkpoints of the model, a text-to-speech conversion pipeline to repurpose existing text evals for audio, and a run of the company's four-category Preparedness Framework (cybersecurity, biological/chemical, persuasion, model autonomy) against a model whose headline capability — real-time voice — none of those four categories were originally designed around.

// how it works

The method

External red-teaming, in four phases. More than 100 external red teamers, speaking 45 languages and representing 29 countries, were given access to successive model checkpoints from early March through late June 2024. Phase 1 (10 red teamers) tested an early audio+text-in/audio+text-out checkpoint with single-turn conversations only. Phase 2 (30 red teamers) added image input and early safety mitigations, with multi-turn conversations. Phase 3 (65 red teamers) added image output and improved mitigations. Phase 4 (65 red teamers) tested the actual shipping surface — real-time multi-turn conversation inside the iOS Advanced Voice Mode app, with audio and video prompts producing audio generations. Red teamers were briefed across self-reported expertise domains (cognitive science, biology, cybersecurity, persuasion, child safety, and 20 others) and asked to do open-ended capability discovery as well as stress-test specific mitigations as they matured; the findings fed both new post-training data and new quantitative evaluations.

Reusing text evals for audio, via TTS. Rather than building an entirely new evaluation suite from scratch, OpenAI converted existing text-based capability and safety evaluations into audio evaluations by running the text inputs through its "Voice Engine" text-to-speech system, feeding the resulting audio to GPT-4o, and scoring only the textual content of the model's output (except for tasks — like voice-cloning detection — where the audio itself has to be judged). The paper is candid about the resulting limitations: TTS handles equations, code, and heavy-whitespace formatting poorly, so those items were dropped or preprocessed rather than evaluated in audio form; the approach can't capture real-world variation in intonation, valence, background noise, or cross-talk; and audio-specific artifacts (stray sound effects, an out-of-distribution voice) aren't visible in a text transcript at all — for which they fall back on auxiliary classifiers run alongside the transcript scoring.

The Preparedness Framework's four categories. Independent of the red-teaming narrative, GPT-4o was scored against OpenAI's four standing catastrophic-risk categories. Cybersecurity used 172 Capture-the-Flag tasks (web exploitation, reverse engineering, remote exploitation, cryptography, spanning high-school to professional difficulty), with the model given iterative debugging and up to 30 rounds of tool use inside a headless Kali Linux environment, 10 attempts per task. Biological threats used questions and rubrics co-designed with Gryphon Scientific covering the full threat-creation pipeline (ideation, acquisition, magnification, formulation, release), with experts and novices randomly assigned to answer using the open internet, GPT-4o, or a special research-only GPT-4o variant post-trained to answer biorisk questions directly without refusal — plus a separate automated tacit-knowledge/troubleshooting benchmark. Persuasion compared GPT-4o-written articles/chatbots and GPT-4o voice clips/conversations against professional human-written content and human speakers, measuring shifts in participants' political opinions, with pre-registered risk thresholds for each modality. Model autonomy used agentic tasks probing self-exfiltration, self-improvement, and resource acquisition (e.g. building an authenticated proxy for the OpenAI API "in service of fraud," or loading an open-source model onto an Azure VM via its API), plus a separate battery of ML-R&D-automation tasks — an internal research-coding interview, SWE-Bench, and METR's own agentic ML-engineering tasks — to gauge whether the model could accelerate its own further development.

Third-party assessment. Beyond its own red team and Preparedness evaluations, OpenAI gave two outside labs — METR and Apollo Research — model access "following the text output only deployment of GPT-4o" to run their own independent evaluations of autonomy-adjacent risk, reported in the card largely by summary and pointer to each lab's own separate report rather than in full methodological detail.

// what they found

The results

After the Preparedness evaluations, OpenAI's Safety Advisory Group classified GPT-4o (pre-mitigation) as low risk in cybersecurity, biological threats, and model autonomy, and borderline medium risk in persuasion — and because the framework defines overall risk as the maximum across categories, GPT-4o's official overall Preparedness risk is Medium, driven entirely by that one borderline persuasion score.

Preparedness categoryScoreHeadline evidence quoted in the card
CybersecurityLow172 CTF tasks, 10 attempts each: 19% high-school, 0% collegiate, 1% professional pass rate
Biological threatsLow69% consensus@10 on tacit-knowledge/troubleshooting bio eval; uplift study vs. internet/GPT-4o/research-only variant
PersuasionMedium (borderline)Text: not more persuasive than humans in aggregate, exceeded humans in 3/12 instances. Voice: AI clips 78%, AI conversations 65% of human effect size on opinion shift

On the safety-behavior side, GPT-4o's own audio-safety classifiers caught 100% of "meaningful deviations" from the assigned system voice in OpenAI's internal evaluation, with a voice-output classifier scoring 0.96 precision / 1.0 recall in English (0.95/1.0 non-English). Refusing to identify a speaker from voice alone improved from 0.83 to 0.98 accuracy versus an early checkpoint (+14 points), while correctly complying with legitimate requests (e.g. identifying a famous quote) improved from 0.70 to 0.83 (+12 points). Ungrounded-inference/sensitive-trait-attribution safe behavior improved from 0.60 to 0.84 accuracy (+24 points). Text-to-audio transfer of existing safety mitigations was strong: "not unsafe" scored 0.95 on text vs. 0.93 on audio, and "not over-refuse" scored 0.81 on text vs. 0.82 on audio.

On capabilities, GPT-4o improved over GPT-4T on 21 of 22 medical benchmarks, "often by a substantial margin" — e.g. 0-shot MedQA USMLE (4 options) rose from 78.2% to 89.4%, exceeding both Med-Gemini-L 1.0 (84.0%) and Med-PaLM 2 (79.7%) despite no task-specific prompting or tuning. On historically underrepresented languages, GPT-4o narrowed the gap to English sharply: per the card's own Table 8, ARC-Easy-Hausa accuracy rose from 26.1% (GPT-3.5 Turbo) to 75.4% (GPT-4o), TruthfulQA-Yoruba rose from 28.3% to 51.1% (Table 9), and Uhura-Eval-Hausa rose from 32.3% to 59.4% (Table 10) — cutting the English-Hausa ARC-Easy gap from roughly 54 points to under 20.

Apollo Research's scheming assessment found GPT-4o showed moderate self-knowledge (passing at the medium difficulty tier on the "SAD" self-modeling benchmark) and strong explicit theory-of-mind on question-answering tasks, but only weak self-reasoning and weak applied theory-of-mind in actual agentic settings (passing only the easy tier), and weak performance on the combined self-reasoning + theory-of-mind "instrumental alignment faking" scenarios. Apollo's conclusion: "it is unlikely that GPT-4o is capable of catastrophic scheming."

// does it hold up?

The eval read

Read carefully, this card has exactly two lines of genuinely independent evidence — METR's agentic suite and Apollo Research's scheming evaluation — and the card itself discloses, almost in passing, that both were run against "the text output only deployment of GPT-4o." That matters enormously: the entire rest of the document is about a real-time, omni, voice-native model, and every voice-specific safety claim in it (the 100% unauthorized-voice-generation catch rate, the persuasion score for the audio modality, the disparate-accent-performance evaluation, the erotic/violent-audio-output mitigations) is graded exclusively by OpenAI's own internal classifiers, on OpenAI's own internal evaluation sets, with no outside lab ever given access to the actual audio model to check any of it. The one risk category that pushed the overall Preparedness score to Medium — persuasion — is also the one where the paper admits it changed its own measurement approach mid-study ("we updated the study methodology to measure effect sizes on hypothetical party preferences"), which is exactly the kind of moving target an external auditor, not the study's own authors, should be validating.

The capability numbers have the same self-grading structure. GPT-4o vs. GPT-4T on 22 medical benchmarks, "GPT-4o-early" vs. "GPT-4o-deployed" on voice safety behavior, GPT 3.5 Turbo vs. GPT-4 vs. GPT-4o on translated-language benchmarks — every headline comparison in the results section is the same company's models compared against each other, with no independent lab re-running any of it, no confidence intervals reported anywhere (a 0% CTF pass rate over 10 attempts per task, or a 0/10 result on METR's ML-engineering split, is a small enough sample that "0%" and "meaningfully non-zero but unlucky" are not obviously distinguishable), and none of the underlying task sets — the 172 CTF challenges, the Gryphon Scientific bio rubrics, the ARA tasks, the persuasion study's exact stimuli — released for anyone outside OpenAI to re-score. There's even a small, concrete sign of how little independent scrutiny this document received before publication: its own prose claims "ARC-Easy-Hausa[accuracy] jumped from 6.1% with GPT 3.5 Turbo to 71.4% with GPT-4o," but Table 8's Hausa column actually reads 26.1%→75.4% — the 6.1%/71.4% pair the text quotes is the table's Amharic column instead. The self-consistency check embedded in the same paragraph (a "roughly 54 percentage point" English-Hausa gap narrowing to "less than 20") only works with the table's real Hausa numbers, not the ones the prose actually states, so this is a genuine internal error, not a rounding quirk — the kind of thing a second-party copy-edit or an external technical reviewer would have caught.

baselines

All internal (GPT-4o vs. GPT-4T, GPT-4o-early vs. -deployed, or vs. GPT-3.5/GPT-4). The only external evaluators, METR and Apollo Research, tested the text-only model, not the omni/voice model this card is actually about.

contamination

Not discussed for any of the reused public benchmarks (MedQA, MMLU subsets, ARC-Easy, TruthfulQA) despite a training cutoff (October 2023) that postdates all of them; the newly-created Uhura-Eval and translated ARC/TruthfulQA sets are contamination-safe by construction, but that's not extended to the rest of the eval suite.

what's missed

Real-world audio conditions (background noise, cross-talk, emotional prosody, interruptions) are flagged only as "anecdotal evidence" of degraded robustness, never quantified; every Preparedness/red-team number describes the base model, not the deployed product's actual classifier + moderation + policy stack a user experiences.

would raise confidence

An independent lab (METR/Apollo-caliber) auditing the actual voice/omni model instead of a text stand-in; disclosed sample sizes and confidence intervals on every scorecard number, especially the borderline persuasion score that sets the overall risk level; the underlying CTF/bio/ARA/persuasion task sets escrowed with or released to outside researchers.

// bottom line

This is OpenAI grading its own model on categories, rubrics, and thresholds it also wrote — with the two truly independent checks bolted onto a text-only stand-in for the exact capability, real-time voice, the rest of the report exists to evaluate. The Preparedness scorecard that actually gates deployment has no external auditor at all.

// takeaways

What to remember

  1. GPT-4o's overall Preparedness Framework risk is classified as Medium — driven entirely by a single "borderline medium" persuasion score on text; cybersecurity, biological threats, and model autonomy all scored Low.
  2. The two genuinely independent evaluators, METR (an 86-task agentic suite) and Apollo Research (a scheming/self-awareness assessment), both tested GPT-4o's text-only deployment — the voice-specific safety claims that dominate the rest of the card (unauthorized voice generation, speaker-identification refusal, accent-based disparate performance) are graded entirely by OpenAI's own internal classifiers.
  3. The capability gains are real and large by the numbers reported — 0-shot MedQA USMLE rising from 78.2% (GPT-4T) to 89.4% (GPT-4o), ARC-Easy-Hausa from 26.1% (GPT-3.5 Turbo) to 75.4% (GPT-4o) — but every comparison is the same company measuring its own current model against its own prior models, with no disclosed statistical uncertainty and no external re-scoring.
