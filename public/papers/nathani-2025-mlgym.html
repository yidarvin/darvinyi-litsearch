MLGym: A New Framework and Benchmark for Advancing AI Research Agents

The first Gym environment for AI-research tasks — 13 open-ended ML problems, a metric borrowed from AutoML to compare models fairly across them, and five frontier LLMs that reliably tune hyperparameters but invent nothing new.

Nathani, Madaan, Roberts, … Raileanu · UCSB · UCL · GenAI & FAIR at Meta

// the gap

What problem is this solving?

By early 2025 there were plenty of agent benchmarks, but almost none of them measured doing AI research. SWE-bench and its descendants test whether an agent can resolve GitHub issues; MLAgentBench scores a handful of single-domain ML tasks; MLE-bench frames Kaggle competitions. Each is narrow, each fixes its own harness, and — the detail MLGym fixates on — none of them is a Gym. They expose a benchmark you can score against, not an environment you can train against. That matters because the obvious way to make agents better at research is reinforcement learning, and RL needs a standard step()/reset() interface with reproducible state, rewards, and resets.

There is a second, subtler gap: how do you even score an agent across heterogeneous research tasks? One task reports accuracy, another BLEU, another validation loss, another average reward in a game. Naively averaging those numbers is meaningless, and ranking is worse — it disproportionately punishes a method that solves a task well but ties with another that also solved it. MLGym wants a single, defensible aggregate that respects each task's native metric.

So Meta builds both halves at once: MLGym, a Gymnasium-style framework where ML research tasks become RL environments and any model or agent scaffold can be dropped in; and MLGym-Bench, a suite of 13 open-ended tasks spanning computer vision, NLP, RL, and game theory. The explicit scope is modest and honest — they place it at Level 1 of a six-level capability ladder (see below), i.e. improving a non-SOTA baseline, not making novel discoveries. The question is how close today's frontier models get even to that floor.

// the six-level ladder

L0 Reproduction · L1 Baseline Improvement · L2 SOTA Achievement · L3 Novel Scientific Contribution · L4 Groundbreaking Contribution · L5 Long-Term Research Agenda. MLGym-Bench targets L1: given a working but un-tuned baseline, can the agent measurably improve it? Everything above L2 — coming up with new methods, architectures, or research agendas — is left as the unmeasured frontier.

// how it works

The method

MLGym wires three boxes together. An Agent (any LLM behind a scaffold) holds the task description, tool documentation, and prompts. A Gymnasium Environment mediates every interaction. And a Computer — a real shell with tools and a file system holding the task's data, code, and requirements — is where work actually happens. The agent emits an action, the environment forwards it to the computer, and feedback flows back. Because the wrapper is a genuine Gym, the whole loop is also an RL environment: state, action, reward, reset. That is the framework's headline claim — the first Gym interface for machine-learning research tasks.

The action space is built on SWE-Agent's Agent-Computer Interface — search, file navigation, a line-numbered file viewer, and a linting editor — extended for research. MLGym adds a permission system (try to edit a read-only evaluation script and you get a feedback string, not a silent corruption), a literature-search + PDF-parsing tool, and a Memory Module that lets the agent write and re-read "research logs" so it can carry findings across a long run. A single run is capped at 50 steps, each action must finish within 1800 seconds, and at the end the last codebase state is auto-submitted.

Scoring across incommensurable tasks

To compare models fairly when every task has its own metric, MLGym adopts performance profiles (Dolan & Moré, 2002) and the AUP score (area under the profile) from the AutoML Decathlon. For task t and model m, you compute a performance ratio against the best model on that task, then ask: across all tasks, for what fraction does the model land within a factor 10τ of the best (the ratio is log-scaled, and inverted for higher-is-better metrics like accuracy)? Integrating that profile over τ gives one number per model — the AUP.

rt,m = ℓt,mminm ℓt,m  ρm(τ) = |{ t : log10 rt,m ≤ τ }||T|  AUPm = ∫ ρm(τ) dτ // performance ratio vs. the best model on task t (inverted to max/ℓ for higher-is-better metrics) → fraction of tasks within a factor 10τ of the best → area under that curve. Higher AUP = better.

Two flavors are reported. Best Submission@4 uses the agent's final submitted solution across 4 seeds — rewarding the discipline of remembering to fall back to your best intermediate result. Best Attempt@4 uses the best validated solution it produced at any point — the capability ceiling, regardless of whether the agent had the sense to submit it. MLGym-Bench's 13 tasks include CIFAR-10, Fashion-MNIST, MS-COCO captioning, MNLI, language modeling on FineWeb, Atari Breakout, MountainCar, Meta Maze, 3-SAT, house-price regression, and three game-theory tasks (Battle of the Sexes, Prisoner's Dilemma, Blotto).

// what they found

The results

Five frontier models were run: Claude-3.5-Sonnet, Llama-3.1-405B-Instruct, GPT-4o, OpenAI o1-preview, and Gemini-1.5-Pro. On aggregate, o1-preview is the best model, with a Best Attempt AUP@4 of 1.150 and Best Submission AUP@4 of 1.176 — but Claude-3.5-Sonnet (1.142 / 1.135) and Gemini-1.5-Pro (1.140 / 1.125) sit right behind it. The whole field is compressed into a narrow band; the strongest and weakest aggregate scores differ by less than 0.18.

On the raw per-task numbers, the agents genuinely beat the provided baselines — often by a lot. On CIFAR-10, accuracy climbs from a baseline of 0.497 to 0.895 (Claude); on Fashion-MNIST, from 0.783 to 0.945 (Claude); on MNLI, from 0.525 to 0.838 (Gemini); on language modeling, validation loss drops from 4.673 to 3.966 (o1). But read the paper's own abstract back: these gains "usually" come from finding better hyperparameters, not from new hypotheses, algorithms, or architectures. The agents are competent optimizers, not researchers. And they are brittle — Llama-3.1-405B fails to produce any valid solution for the Language Modeling task, and GPT-4o does the same on Breakout (both marked ∞).

The behavioral data is where the "optimizer, not researcher" story gets concrete. Across all runs, Edit is ~50% of all actions (3,708 calls) and the agents cycle relentlessly through edit → view → run-Python → validate. Search is used essentially never — only 11 calls, ~1% of actions. The tools for exploring the literature and the codebase exist; the models barely touch them. They don't go looking for ideas — they tweak the code in front of them and re-run it.

// does it hold up?

The eval read

MLGym is admirably self-aware — its headline finding is itself a critique of the agents — but the benchmark's construction deserves the same scrutiny. The most important caveat is one the paper states plainly and then risks under-weighting in framing: this measures Level 1, baseline improvement, and almost nothing above it. "Advancing AI Research Agents" is the title, but the instrument can only see hyperparameter tuning on a non-SOTA starting point. The conclusion that models "do not generate novel hypotheses" is partly a finding and partly a property of a benchmark that never asks them to, scores them on a single scalar if they tried, and caps every run at 50 steps.

That single-scalar reduction is the second issue. "Open-ended AI research" collapses to optimizing one number per task — accuracy, BLEU, reward, loss. MS-COCO captioning is scored by BLEU, a metric widely known to correlate poorly with caption quality; the game-theory tasks reduce to average reward. A leaderboard metric rewards leaderboard behavior, which is exactly the edit-and-rerun loop the action analysis exposes — so the harness and the metric may be producing the very narrowness the paper laments, not just observing it.

Third, contamination. Every task is a famous public benchmark on a famous public dataset — CIFAR-10, Fashion-MNIST, MNLI, Breakout. Frontier models have surely seen these datasets and good architectures for them in pre-training. The 0.497→0.895 jump on CIFAR-10 may be less "research" than recall of a known recipe — and Figure 11 shows the mechanism literally: the agent pulls a 93.51%-accurate ResNet config from memory and re-implements it. Finally, AUP is relative: it normalizes against the best model in this particular pool, so a score of 1.0 means "≈ GPT-4o," not "good," and the compressed 1.00–1.18 spread makes real capability gaps look smaller than they are — and would shift if you added or dropped a model.

baselines

Each task ships a deliberately non-SOTA baseline, which is fair for an L1 "can you improve it?" test — but means strong absolute scores reflect a low bar, not frontier performance. The model pool (o1-preview, Claude-3.5, Gemini-1.5, GPT-4o, Llama-3.1) is a Feb-2025 snapshot already overtaken by newer reasoning models.

contamination

Severe risk: all 13 tasks use canonical public datasets/benchmarks. Gains plausibly reflect memorized architectures and recipes rather than research — the Memory-Module trajectory shows recall of a known 93.51% config in action.

what's missed

A single scalar per task can't see ideation, experimental design, analysis, or genuine novelty — the L2–L5 capabilities the title evokes. AUP's relative normalization compresses model differences and depends on which models are in the pool. The 50-step / 1800s caps and edit-heavy harness may bound exploration.

would raise confidence

Held-out or freshly-constructed tasks resistant to contamination; multi-metric or human evaluation of solution novelty; longer step/compute budgets to test whether "no new ideas" is a ceiling or a budget artifact; and re-running with current reasoning models.

// bottom line

MLGym is a genuinely useful piece of infrastructure — the first Gym for ML-research agents, plus a principled cross-task metric — and its central empirical message is sturdy and well-evidenced: in early 2025, frontier agents tune, they don't invent. Just read the title's ambition against the instrument's reach: it convincingly measures the floor (Level 1 baseline improvement), and the more exciting claims about advancing AI research remain a frontier the benchmark frames but cannot yet score.

// takeaways

What to remember

  1. First Gym for AI-research agents. MLGym wraps 13 open-ended ML tasks (CV, NLP, RL, game theory) in a Gymnasium step()/reset() interface, so agents can be both evaluated and RL-trained — built atop SWE-Agent's tools plus permissions, literature search, and a memory module.
  2. A fair cross-task metric. Performance profiles + the AUP score (from AutoML) give one comparable number across tasks with incommensurable metrics. o1-preview leads (Best Attempt AUP@4 = 1.150), with Claude-3.5 and Gemini-1.5 close behind — and Gemini ~9× cheaper than o1 at ~99% of its score.
  3. Agents optimize, they don't research. Models beat baselines (CIFAR-10 0.497→0.895, Fashion-MNIST 0.783→0.945) but by tuning, not inventing — Search is 11 of thousands of actions, the work is ~50% Edit, and the benchmark explicitly tops out at Level 1 of its own six-level ladder.
