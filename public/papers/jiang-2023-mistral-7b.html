Mistral 7B

A 7-billion-parameter open model that beat Llama 2 13B on every benchmark tested — and Llama 1 34B on math, code, and reasoning — built around sliding-window attention and a rolling cache for cheap inference. A clean benchmark sheet that says almost nothing about the data behind it.

// the gap

What problem is this solving?

The default recipe for a better language model was a bigger one. But scale raises three costs that don't move together: training compute, model memory, and — the one you pay on every single request in production — inference latency. By late 2023 the strongest openly-released models were the LLaMA family, and the practical bar for "good but deployable" sat at the Llama 2 13B / 34B tier. The 7B class (Llama 2 7B) was cheap to serve but trailed 13B by a wide margin on reasoning, math, and code.

Mistral 7B targets a sharp, practical question: can a 7B model — small enough to run on a single commodity GPU — actually match or beat a 13B, so practitioners stop paying the larger-model inference tax for capabilities a well-built 7B can deliver? And can the architecture itself keep attention cost from exploding as context grows? Vanilla self-attention is quadratic in sequence length and its key/value cache grows linearly with every token generated, so long sequences are expensive at serving time regardless of parameter count. The paper's bet is that careful architecture plus careful training can collapse the quality gap to the 13B tier while cutting the inference bill.

// how it works

The method

Mistral 7B is a decoder-only transformer (dim 4096, 32 layers, 32 attention heads of head_dim 128, hidden_dim 14336, vocab 32000) that departs from the LLaMA recipe in two inference-oriented ways, plus the cache machinery that makes them pay off.

Grouped-query attention (GQA). The 32 query heads share just 8 key/value heads (n_kv_heads = 8). Since the KV cache is the dominant memory cost during generation, this shrinks it ~4× versus full multi-head attention — which lets you run larger batches and get higher throughput, at negligible quality cost.

Sliding-window attention (SWA). Rather than letting every token attend to the entire history, each token attends only to the W = 4096 most recent tokens at each layer, capping per-layer attention cost. The subtlety is that stacked layers compound the window: a token's hidden state at layer k has already absorbed information from W tokens back at every layer beneath it, so its effective receptive field grows roughly linearly with depth. After k layers a token can be influenced by tokens up to ≈ k × W positions away — even though no single attention operation ever looks past 4096.

effective span after k layers  ≈  k · W  =  32 × 4096  ≈  131,072 tokens // W = 4096 window over 32 layers ⇒ a ~131K theoretical attention span, while each layer's attention stays bounded to a 4096-token window

Rolling buffer cache. Because attention never reaches past W, the KV cache needs only W slots. The key/value for position i is written to slot i mod W, overwriting the now-irrelevant entry. On a 32K-token sequence this cuts cache memory ~8× with no impact on quality. Pre-fill and chunking: since the prompt is known in advance, its cache is pre-filled; a long prompt is split into chunks of size W, each attending to the rolling cache plus itself. With modifications to FlashAttention and xFormers, the result is a 2× speedup over a vanilla-attention baseline at a 16K sequence length. The released base model has a context length of 8192.

// what they found

The results

Mistral re-ran every benchmark through its own evaluation pipeline — so the LLaMA numbers below are Mistral's re-measurements, not the originals — to keep the comparison apples-to-apples. Mistral 7B beats Llama 2 13B on all measured tasks, and beats Llama 1 34B on math, code, and reasoning.

On MMLU it scores 60.1% against Llama 2 13B's 55.6% (and Llama 2 7B's 44.4%). The widest gaps are in math: MATH 13.1% vs 6.0%, and GSM8K 52.2% vs 34.3% — more than double Llama 2 13B on MATH. On code, HumanEval 30.5% (vs 18.9%) nearly matches the code-specialist Code-Llama 7B (31.1%), and MBPP 47.5% trails Code-Llama 7B's 52.5% by only a few points — i.e. near-specialist coding without sacrificing general performance. Commonsense and reasoning follow the same pattern (HellaSwag 81.3, WinoGrande 75.3, PIQA 83.0, ARC-c 55.5 — all above Llama 2 13B).

"Equivalent model size." Mapping Mistral's scores onto the Llama 2 size curve, it matches a Llama 2 of ~23B on MMLU (3.3× its size), ~38B on reasoning (5.4×), and ~21B on comprehension (3×) — but only ~13B on world knowledge (1.9×), since a 7B's limited parameter count caps how many facts it can store.

Instruction model. Mistral 7B – Instruct, fine-tuned only on publicly available Hugging Face instruction datasets ("no proprietary data or training tricks"), scores MT-Bench 6.84 ± 0.07 and Chatbot Arena ELO 1031 — beating every 7B chat model (Llama 2 7B Chat 6.27, Vicuna 7B 6.17) and edging Llama 2 13B Chat (6.65), i.e. comparable to 13B chat models. In an uncontrolled human vote on llmboxing.com (as of Oct 6, 2023), its outputs were preferred 5020 times vs 4143 for Llama 2 13B Chat.

Guardrails. With a recommended safety system prompt, the instruct model declined all 175 unsafe test prompts; a self-reflection content moderator hit 99.4% precision / 95.6% recall on a self-curated adversarial set. The safety prompt costs some utility — MT-Bench drops to 6.58 with the Mistral system prompt vs 6.84 with none.

// does it hold up?

The eval read

The good practice is real: Mistral re-ran every benchmark in a single pipeline, so the headline comparison doesn't suffer from the cross-paper eval inconsistency that makes most LLM leaderboards incomparable. But the comparison set is exclusively the LLaMA family — no Falcon, MPT, Qwen, or other strong 7–13B open contemporaries appear — so "best open 13B / 34B" is really "best LLaMA." The "beats 34B" line is carried by Llama 1 34B, an older and weaker 34B that stands in for the never-released Llama 2 34B (honestly footnoted, but it flatters the claim).

The deeper gap is scientific. The paper discloses nothing about pretraining: no data sources, no mixture, no token count, no compute. So its marquee conclusion — that a 7B "compresses knowledge more than previously thought" — is demonstrated but not explained. We can't tell whether the gains come from architecture (GQA/SWA) or simply from more, cleaner, or differently-distributed training data, because there are no ablations isolating the two. And with the data a black box, benchmark contamination into MMLU/GSM8K/HumanEval can't be ruled out — or even probed. The architecture story, meanwhile, is about inference cost, not quality, so it doesn't account for the accuracy jumps at all.

Finally, the signature SWA capability is never exercised. The advertised ~131K theoretical span is a layering argument; the released context length is 8192 and every benchmark here is a short-context task. There is no long-context evaluation (needle-in-a-haystack, long-document QA) showing SWA delivers usable long-range performance rather than just cheaper attention. The instruction and safety sections are explicitly "simple and preliminary": MT-Bench / Arena are noisy, the human vote is an uncontrolled web poll, and the 100%-refusal and 99.4%/95.6% moderation numbers come from small, self-built prompt sets, not standardized red-team benchmarks.

baselines

Only the LLaMA family (Llama 1/2, Code-Llama), all re-scored in Mistral's own pipeline — fair internally, but no non-LLaMA 7–13B contemporaries. "Beats 34B" uses the older Llama 1 34B as a stand-in for the unreleased Llama 2 34B.

contamination

Untested and untestable from outside: training data is fully undisclosed, so leakage into MMLU / GSM8K / HumanEval can be neither ruled out nor probed.

what's missed

Why a 7B reaches 3× quality is unexplained — no data, token-count, compute, or GQA/SWA-vs-data ablation. The "131K span" is asserted, never measured: context is 8192 and no long-context eval is run.

would raise confidence

Pretraining-data + contamination analysis; non-LLaMA baselines; long-context benchmarks that actually use SWA; ablations isolating architecture from data; standardized safety / red-team evaluation.

// bottom line

A landmark release paper: an Apache-2.0 7B that beat Llama 2 13B across the board and reset expectations for what small models can do, packaged with genuinely useful inference engineering (GQA + sliding-window attention + rolling cache). As science it's thin — the quality gains are reported, not explained, because the data and training are a black box and contamination is untested. Read it as a strong, clean-benchmarked product launch, not as evidence about why a 7B can punch at 3× its weight.

// takeaways

What to remember

  1. A well-built 7B can beat a 13B: Mistral 7B tops Llama 2 13B on every benchmark and Llama 1 34B on math/code/reasoning (MMLU 60.1, MATH 13.1, GSM8K 52.2), making "use a 7B and save the inference cost" a real option.
  2. The lever is inference architecture, not training: grouped-query attention shrinks the KV cache ~4×, sliding-window attention bounds per-layer cost while depth turns a 4096 window into a ~131K theoretical receptive field, and a rolling buffer cache fixes memory at the window size — together a 2× speedup at 16K tokens.
  3. The benchmark sheet is clean but unexplained: no training data, no contamination test, no long-context eval — so it's an excellent model release and a thin scientific claim about how small models compress so much.
