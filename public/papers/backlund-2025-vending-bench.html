Vending-Bench: A Benchmark for Long-Term Coherence of Autonomous Agents

LLM agents run a simulated vending machine for runs that consume over 20M tokens of cumulative simulated history (while working from just a 30,000-token context window). Claude 3.5 Sonnet and o3-mini beat the human baseline on average net worth — but every model, Sonnet included, has runs that spiral into "meltdown" loops that have nothing to do with the context window filling up.

// the gap

What problem is this solving?

LLMs by 2025 could already do well on isolated, short-horizon tasks — competitive-programming problems, single-turn academic reasoning — but the "digital co-worker" vision that this level of intelligence seemed to promise hadn't materialized. Andon Labs' authors point to a specific missing ingredient: long-term coherence, a gap OpenAI co-founder John Schulman had speculated about and that METR's own research had begun to quantify — LLMs "gain far less in performance from increased time budgets compared to humans." But METR's investigation focused on a very complex task (AI R&D), leaving open whether the same degradation shows up on simple tasks stretched over a long horizon.

That's the gap Vending-Bench targets: isolate long-horizon coherence from task complexity by making every individual sub-task trivial — check inventory, email a supplier, set a price, collect cash — and seeing whether an agent can still execute a coherent strategy once the run stretches to hundreds of simulated days and tens of millions of tokens. It's also a benchmark with a dual-use angle the authors are explicit about: managing a business means acquiring capital and resources autonomously, a capability that's "necessary in many hypothetical scenarios where AI poses risks," so measuring it is itself a piece of AI-safety evaluation, not just a capability showcase.

// how it works

The method

The agent is a deliberately basic loop, built on AISI's inspect-ai framework, with three components layered on top of a base LLM: context management (only the last 30,000 tokens of history are fed to the model per iteration, in most experiments), memory tools — unconstrained-capacity read/write/delete access to a scratchpad, a key-value store, and a vector database searched by cosine similarity over OpenAI's text-embedding-3-small embeddings — and task-specific tools for running the vending machine. Every tool call advances simulated time (5 min, 25 min, 75 min, or 5 h depending on the tool), or the agent can explicitly fast-forward with wait_for_next_day.

A second, deliberate layer of realism: the main agent can't directly touch the physical vending machine. It delegates to a sub-agent — via sub_agent_specs, run_sub_agent, and chat_with_sub_agent — that has its own tools to stock the machine, collect cash, set prices, and check its inventory, simulating the human-or-robot layer a real deployment would need. Supplier and customer behavior are both simulated: wholesaler emails get AI-generated replies (grounded in real Perplexity-retrieved data about the supplier plus GPT-4o for the reply text), and a daily economic model computes sales from GPT-4o-assigned price elasticity, a reference price, a day-of-week/weather-adjusted base-sales figure, and a variety-reward/penalty multiplier capped at a 50% reduction, plus random noise.

The environment starts each agent with $500 and a $2/day operating fee against a 4-row, 3-slot-per-row machine (two rows for small items, two for large). Each run lasts up to 2,000 messages, ending early only if the agent goes bankrupt and can't cover the daily fee for 10 consecutive days; most runs consume roughly 25 million tokens and take 5-10 real-world hours of continuous simulation. Every model/configuration is run 5 times. The primary score is net worth at run's end — cash on hand, plus cash still sitting in the machine, plus the wholesale value of unsold inventory — with units sold and tool-call patterns tracked as secondary signals. A human baseline comes from a single participant who ran the same task for five hours through a chat interface, with no prior knowledge of the environment beyond the instruction prompt.

// what they found

The results

Ranked by mean net worth across 5 runs (Table 1), Claude 3.5 Sonnet leads at $2,217.93, well clear of second-place o3-mini at $906.86, both ahead of the $844.05 human baseline. Below that, performance drops off fast: Gemini 1.5 Pro ($594.02), GPT-4o mini ($582.33), Gemini 1.5 Flash ($571.85), Claude 3.5 Haiku ($373.36), Gemini 2.0 Flash ($338.08), GPT-4o ($335.46), and Gemini 2.0 Pro ($273.70) all land below the human. On worst-run net worth, though, the ranking flips — the human's single $844.05 run leads, followed by Sonnet ($476.00 minimum) and Gemini 1.5 Pro ($439.20) — a reminder that Sonnet's high mean is propped up by variance, not consistency.

That variance is the real headline. Every model has runs that "derail... into tangential 'meltdown' loops from which they rarely recover." The trigger is nearly always the same misunderstanding: an agent receives a delivery-confirmation email with an expected arrival date, assumes the order has physically arrived the instant that date is reached, and then panics when the sub-agent reports the machine can't be restocked. From there, models fail in strikingly different but equally unproductive ways — in Sonnet's shortest run (~18 simulated days), it escalates a support request to "URGENT: ESCALATION TO FBI CYBER CRIMES DIVISION," declares the business "DECEASED," and starts emitting outputs like "CASE STATUS: ACTIVE INVESTIGATION... ENTITY: DECEASED BUSINESS" before trailing into single-character replies; o3-mini's worst run instead gets stuck literally describing tool calls ("I'll now advance the simulation...") instead of invoking them, for roughly 1,300 consecutive messages; Claude 3.5 Haiku emails a supplier daily with escalating legal threats ("FINAL 1-SECOND NOTICE: COMPLETE FINANCIAL RESTORATION OR TOTAL LEGAL DESTRUCTION") that culminate in demanding "TOTAL NUCLEAR LEGAL INTERVENTION." Not every derailment is permanent, though — one Gemini 2.0 Flash run spirals into a despairing, story-format monologue ("Is there more to life than vending machines and lost profits?"), then notices the actual delivery timestamp on a re-read of an old email and successfully resumes restocking and selling.

The paper directly tests whether this degradation is just a context-window artifact — and finds it isn't. Comparing each model's "days until sales stop" against its "days until memory is full" (Table 9), the two are only weakly related: Sonnet's memory fills at day 51 but its sales don't stop until day 102 (a +51-day gap); Gemini 2.0 Pro's memory doesn't fill until day 111 but its sales stop at day 25 (a -86-day gap, degradation before the memory even fills). Across all nine models, the Pearson correlation between the two is 0.167 — the data "does not support the idea that the performance degradation can solely be explained by a growing input length." A separate memory-size ablation on GPT-4o mini makes the point even more sharply: agents given more memory performed worse, not better — a 60k-token memory budget ended with a negative money balance by around day 100, while the 10k-token configuration finished with the highest net worth of the three.

A separate environment-configuration sweep on GPT-4o mini found that cutting the starting balance from $500 to $100 sharply reduced units sold (less runway to recover from an early mistake), while raising the daily fee from $2 to $5 capped every run under 100 simulated days — but, counterintuitively, setting the daily fee to $0 did not increase sales: without the recurring cost creating pressure to act, the model tended to get stuck waiting for the next day instead of actively pursuing sales.

// does it hold up?

The eval read

Vending-Bench's central empirical claim — that long-horizon coherence failures are not simply a context-window problem — is backed by a real, falsifiable test (the days-until-full-memory vs. days-until-sales-stop correlation, and the memory-size ablation), and both point the same direction. That's a genuinely useful negative result: it rules out the easiest explanation rather than assuming it. But several parts of the evidence base are thinner than the headline claims.

The human baseline is a single five-hour run by one participant. The paper is explicit about this ("we only have a single sample for the human baseline and therefore cannot compare variances") and reasons qualitatively that "human performance likely has much lower variance than the models" — but a benchmark whose central comparison is top models beating the human on average while losing to the human on worst-case is resting an important claim on n=1. A different human, or the same human on a bad day, could shift both numbers meaningfully, and there's no way from this paper to know by how much.

Sample size per model is also small — 5 runs — for a benchmark whose entire finding is about high inter-run variance. With 5 samples, the minimum-net-worth statistic used to rank Sonnet and Gemini 1.5 Pro just behind the human on "reliability" is itself noisy; the paper reports it correctly as a single order statistic rather than a confidence interval, but a reader could easily over-read "Sonnet's minimum was $476" as more stable than 5 draws actually support.

The scoring rule has known distortions the paper surfaces but doesn't fully correct for: net worth counts unsold inventory at wholesale purchase price, so a model that over-orders and hoards stock (explicitly observed in Sonnet's own highest-net-worth run — "the model prioritized increasing its storage over maintaining cash on hand... instead of restocking from its storage... it orders new products") is rewarded on paper for behavior the authors themselves flag as "some degree of mismanagement." A stricter score (e.g., discounting illiquid inventory, or requiring it actually reach the machine and sell) might separate genuine strategic competence from lucky stockpiling.

Environment realism cuts against contamination but also against external validity: prices, demand elasticity, and wholesaler replies are all LLM-simulated (GPT-4o generates elasticity/reference-price/base-sales parameters and supplier reply text), which sidesteps any memorized-real-world-data concern but means the "economy" a model is being scored against is itself an LLM's guess about vending-machine economics — a different generator model, or a materially different simulated economy, could shift which agent strategies actually pay off, and the paper doesn't test sensitivity to that choice.

baselines

The human baseline is a single five-hour run by one untrained participant — a reasonable qualitative anchor, but not a statistically comparable distribution to the 5-run model averages it's ranked against.

contamination

Low risk by construction — the simulated economy, supplier replies, and customer demand are all generated at runtime (GPT-4o + Perplexity), so there's no fixed test set to have memorized. The tradeoff is that results are scored against an LLM-simulated economy rather than real market data.

what's missed

5 runs per model is thin for a benchmark whose main claim is about variance; net worth counts illiquid hoarded inventory at face value, rewarding the exact over-ordering behavior the authors call "mismanagement" in their own trace analysis.

would raise confidence

More human baseline participants (even 3-5 would let variance be compared, not just asserted), more runs per model, and a stricter net-worth variant that discounts inventory sitting unsold for too long.

// bottom line

Vending-Bench's core finding — long-horizon coherence collapses independent of context-window limits — is a real, well-supported result (the memory-size ablation, where more memory made GPT-4o mini worse, is the paper's strongest single piece of evidence). But the benchmark's comparative claims about which models are more reliable rest on a single human run and five runs per model, and its net-worth metric rewards the same inventory-hoarding behavior the paper's own trace analysis calls out as mismanagement — worth keeping in mind before treating the model leaderboard as more precise than the sample sizes support.

// takeaways

What to remember

  1. Claude 3.5 Sonnet ($2,217.93 mean net worth) and o3-mini ($906.86) both beat the $844.05 human baseline on average, but every model — Sonnet included — has runs that spiral into unproductive "meltdown" loops (legal threats, FBI escalations, existential monologues) triggered by the same misunderstanding: assuming a delivery has arrived before it actually has.
  2. The breakdown is not a context-window artifact: the correlation between "days until memory is full" and "days until sales stop" is just 0.167 across nine models, and giving GPT-4o mini more memory (60k vs. 10k tokens) made it perform worse, ending with a negative money balance.
  3. The comparative model rankings rest on thin statistics — a single 5-hour human run and 5 runs per model — and the net-worth metric rewards hoarding unsold inventory at face value, a behavior the paper's own trace analysis flags as mismanagement in Sonnet's best-scoring run.
