2020 · ACL · Foundation Models

BART: Denoising Sequence-to-Sequence Pre-training for Natural Language Generation, Translation, and Comprehension

A sequence-to-sequence denoising autoencoder that generalizes BERT's bidirectional encoder and GPT's left-to-right decoder into a single architecture. Corrupt a document with an arbitrary noising function, then train the model to reconstruct it exactly — and unlike prior denoising schemes tied to one noise type, BART can mix and match, so the paper runs the noise functions themselves as an ablation and finds that shuffling sentences plus masking whole spans (not single tokens) wins.

// the gap

What problem is this solving?

By late 2019, NLP pretraining had split into two families that didn't talk to each other. BERT-style masked-language models encode text bidirectionally — every position sees both left and right context — which makes them excellent at classification and span-extraction tasks, but they predict the missing tokens independently, so they "cannot easily be used for generation." GPT-style left-to-right language models are natively generative, since each token conditions autoregressively on everything before it, but they can only ever look leftward, which caps how well they can learn bidirectional interactions needed for tasks like extractive QA. In between, methods like UniLM tried to have it both ways with ensembles of self-attention masks, and XLNet permuted the prediction order — but each of these approaches was tailored to a specific noise type (word masking, span masking, permutation) and inherited its own tradeoffs.

BART's question is architectural: is there one pretraining setup that subsumes BERT and GPT rather than compromising between them, and that is flexible enough to try many different corruption strategies within a single framework — including ones that change a document's length, which token-masking approaches structurally cannot do? The paper's answer is to attach a bidirectional encoder to an autoregressive decoder, corrupt the input with an arbitrary noising function, and train the pair end-to-end to reconstruct the original — turning "which noise function is best" from an architectural commitment into an empirical question the paper can actually run as a controlled ablation.

// how it works

The method

BART is a standard Transformer sequence-to-sequence model (Vaswani et al., 2017) with two small architectural changes relative to GPT: ReLU activations are swapped for GeLUs, and parameters are initialized from N(0, 0.02). The base model uses 6 encoder and 6 decoder layers; the large model uses 12 of each. Relative to BERT, BART's decoder adds cross-attention over the encoder's final hidden states (as in a standard NMT decoder) and drops BERT's extra feed-forward network before word prediction — net, BART has roughly 10% more parameters than an equivalently-sized BERT.

Pretraining optimizes the cross-entropy between the decoder's output and the original, uncorrupted document. In the degenerate case where all information about the source is destroyed, BART reduces exactly to a language model — which is what makes it a strict generalization rather than a third alternative. The paper evaluates five corruption transformations (and their compositions):

  • Token masking — BERT's scheme: random tokens replaced with [MASK].
  • Token deletion — tokens are deleted outright, so unlike masking, the model must also decide which positions are missing.
  • Text infilling — span lengths drawn from a Poisson(λ=3), each span (including 0-length spans) replaced with a single mask token. Inspired by SpanBERT, but SpanBERT uses a clamped geometric length distribution and replaces each span with a same-length run of masks; BART's single-token replacement forces the model to predict how many tokens are missing.
  • Sentence permutation — the document is split on full stops and the sentences shuffled.
  • Document rotation — the document is rotated to begin at a uniformly-chosen token, training the model to recognize the true start.

Fine-tuning reuses the same encoder-decoder pair in task-specific ways: for sequence classification, the uncorrupted input is fed to both encoder and decoder, and the final decoder token's hidden state (an extra token appended at the end, so it can attend to the complete input) feeds a new linear classifier — the BART analogue of BERT's [CLS]. For token classification (e.g. SQuAD span endpoints), every decoder position's top hidden state classifies that token. For generation, the encoder input is the source sequence and the decoder generates the output autoregressively — a direct match to the denoising pretraining objective. For machine translation, BART's own embedding layer is replaced by a new, randomly-initialized encoder trained to map a foreign vocabulary into the space BART already knows how to denoise into English; a two-stage schedule first freezes most of BART and trains only the new encoder plus BART's positional embeddings and first-layer input projections, then unfreezes everything for a short joint fine-tune.

// what they found

The results

Section 4 runs a controlled base-size (6+6 layer, hidden size 768) comparison of six pretraining objectives — a plain language model, a permuted LM (XLNet-style), a masked LM (BERT-style), a multitask masked LM (UniLM-style), a masked seq2seq (MASS-style), and BART with each of its five noise functions — all trained for the same 1M steps on the same books+Wikipedia data and fine-tuned identically. The clearest signal: token masking or deletion is load-bearing — objectives built on rotation or sentence permutation alone perform poorly, while BART with text infilling is the most consistently strong performer across tasks, and combining text infilling with sentence shuffling matches or beats infilling alone on 5 of 6 tasks. A pure left-to-right language model gets the worst SQuAD result (76.7 F1) of any method but the best ELI5 result — the one generation task where something beats BART — since ELI5's output is only loosely constrained by its input, favoring open-ended generation over reconstruction.

Scaled up to large models (12+12 layers, hidden 1024, batch 8,000, 500K steps, the same 160GB corpus and byte-pair vocabulary as RoBERTa, using 30%-token infilling plus full sentence permutation), BART on discriminative tasks essentially ties RoBERTa — SQuAD 1.1 EM/F1 of 88.8/94.6 vs RoBERTa's 88.9/94.6, MNLI-m 89.9 vs 90.2 — "suggesting that BART's uni-directional decoder layers do not reduce performance on discriminative tasks." The real separation shows up on generation: on XSum, BART posts 45.14/22.27/37.25 ROUGE-1/2/L, beating the strongest prior BERT-based system (BERTSUMEXTABS, 38.81/16.50/31.27) by roughly 6 points on every metric — the headline "up to 6 ROUGE" claim from the abstract. On CNN/DailyMail it also outperforms all prior work (44.16/21.28/40.90 vs UniLM's 43.33/20.21/40.51). On the abstractive-QA dataset ELI5, BART reaches 30.6/6.2/24.3 ROUGE-1/2/L, beating the best prior Seq2Seq-Multitask baseline's ROUGE-L by 1.2 points — though the paper is candid that ELI5 "remains a challenging" dataset where answers are only weakly specified by the question. On ConvAI2 dialogue generation, BART reaches 20.72 valid F1 / 11.85 perplexity versus the prior best system's 19.09/17.51. And on WMT16 Romanian→English translation (augmented with back-translation data), plugging a small randomly-initialized source encoder in front of BART improves BLEU from a 36.80 Transformer-large baseline to 37.96 — a +1.1 BLEU gain using only target-side (English) pretraining, with no Romanian pretraining data at all.

The paper also runs a qualitative check on XSum generations from WikiNews articles published after the pretraining corpus was collected, to rule out the summarized events being memorized. It finds the summaries fluent and mostly accurate, integrating background knowledge across the input (e.g. correctly inferring PG&E operates in California) — but flags one example where a generated summary states a finding was published "in the journal Science" — a claim the paper says "is not supported by the source" — an explicit, self-reported hallucination example that the paper includes rather than hides.

// does it hold up?

The eval read

BART's central empirical claim — that noising-function choice matters and text infilling is the best of the five studied — rests entirely on the base-size, single-run ablation in Table 1. That table reports one number per cell: no seeds, no variance, no confidence intervals, across six objectives × six tasks. Several deltas the paper leans on for its narrative (e.g. text-infilling's 90.8 vs token-masking's 90.4 SQuAD F1, or infilling-alone vs infilling+shuffling being a "wash" on most tasks except CNN/DM) are within a point or two — plausibly inside single-seed noise for a base-sized encoder-decoder, and the paper itself concedes elsewhere that "differences in training data, training resources, architectural differences... and fine-tuning procedures" make objective comparisons hard even when it is the one doing the controlling.

The large-scale headline comparison to RoBERTa is real but narrower than the abstract's framing suggests: BART "matches the performance of RoBERTa with comparable training resources" specifically on GLUE and SQuAD — tasks RoBERTa was built for — while its actual advantage is concentrated in generation, where RoBERTa (encoder-only) cannot compete at all, making that comparison structurally one-sided in BART's favor rather than a head-to-head on shared ground. On XSum specifically, the "6 ROUGE" gain is measured against BERTSUMEXTABS, a BERT-encoder-based abstractive system bolted onto a separately-trained decoder — not a fellow denoising sequence-to-sequence pretrained model, since none existed yet at BART's scale. That's a legitimate baseline gap in the field at the time, not a cherry-pick by the authors, but it means the "generation wins" story is partly "any properly pretrained decoder beats an ad-hoc one," not purely "this particular noise function is best."

baselines

Fair and current for October 2019 — RoBERTa, XLNet, UniLM, MASS are the right comparison set, and BART reports its own re-implementations of competing objectives under matched data/steps in Table 1 rather than only citing published numbers. The XSum comparison is the one asymmetric case: no other seq2seq-pretrained model existed yet to compare against.

contamination

Not formally audited (era-typical), but the paper takes one concrete step most contemporaries skipped: its qualitative XSum samples are drawn from WikiNews articles published after the pretraining corpus was collected, specifically to rule out the summarized events appearing in training data — a real, if narrow, contamination control.

what's missed

No variance/seeds anywhere, including the core noising-function ablation the paper's central claim depends on. The large-scale run's recipe (30% infilling + full sentence permutation) carries over a component the base-scale ablation itself found weak — the paper states sentence permutation "only shows significant additive gains on the CNN/DM summarization dataset" — and keeps it at scale purely on the hypothesis that "larger pre-trained models may be better able to learn from this task," not on further ablation evidence. And the one self-reported hallucination (misattributing a finding's publication venue) is presented as an isolated qualitative anecdote, not quantified across the summary sample.

would raise confidence

Multi-seed variance on the Table 1 ablation; a factual-consistency metric (e.g. entailment- or QA-based faithfulness scoring, which postdates this paper) run systematically over XSum/CNN-DM outputs rather than spot-checked by hand; and an ablation confirming the base-scale noise-function ranking holds at large scale rather than assuming it.

// bottom line

The architectural idea is the durable contribution: a Transformer denoising autoencoder that is provably a strict superset of BERT and GPT, evaluated as a genuine noise-function ablation rather than a single fixed recipe. The generation results are real state-of-the-art gains, not baseline-gaming — but they lean on being first to bring a properly-pretrained seq2seq model to summarization, and the fine-grained "which noise function wins" claim rests on single-run numbers that a careful reader should treat as suggestive rather than statistically settled.

// takeaways

What to remember

  1. One architecture, strictly generalizing two families. A bidirectional encoder feeding an autoregressive decoder reduces to BERT-style encoding in one limit and GPT-style generation in the other, and — because corruption is arbitrary — it can vary document length, which token-masking schemes structurally cannot.
  2. Text infilling (Poisson-length spans → single mask) beat token masking, deletion, permutation, and rotation in the paper's controlled base-scale ablation, and text infilling + sentence shuffling was the recipe carried to large scale.
  3. Matches RoBERTa on classification, leads decisively on generation. ~Tied on GLUE/SQuAD; +6 ROUGE over the prior best on XSum, new state of the art on CNN/DM and ELI5, and +1.1 BLEU on RO→EN translation using only English-side pretraining — the generation wins are real, but the SQuAD/GLUE "tie" and the XSum baseline gap both deserve the caveats above before treating this as an unqualified win on every axis.
