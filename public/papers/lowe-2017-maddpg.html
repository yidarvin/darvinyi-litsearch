Multi-Agent Actor-Critic for Mixed Cooperative-Competitive Environments

DDPG works for one agent learning against a fixed world. Put several learning agents in the same world and its replay buffer breaks: MADDPG fixes this by giving each agent a critic that sees everyone's actions during training, while each agent still acts on its own observations alone at test time.

Lowe, Wu, Tamar, Harb, Abbeel, Mordatch

// the gap

What problem is this solving?

By 2017, deep RL had strong single-agent recipes — DQN for discrete control, DDPG (built on Silver et al.'s Deterministic Policy Gradient) for continuous control — but almost all of them assume a stationary world: the environment's dynamics don't change while the agent learns. Multi-agent settings break that assumption twice over. First, Q-learning's experience replay buffer, the thing that stabilizes DQN and DDPG, becomes invalid: with N agents each updating their own policy, the transition probability P(s′|s, a₁,...,a_N, π₁,...,π_N) depends on everyone's current policies, so old transitions stored in the buffer no longer reflect the dynamics once any agent's policy has moved on. Second, policy gradient's variance gets worse as agents are added — the paper proves this formally (Proposition 1): for N agents with binary actions and a reward that only fires when all actions agree, the probability that a single-sample policy-gradient estimate even points in the correct direction falls off as (0.5)N, decaying exponentially in the number of agents. The obvious escape hatches don't fully work either: independent Q-learning per agent (Tan, 1993) is known to perform poorly in practice, and model-based approaches that back-propagate through a differentiable world model require assumptions about the environment and agent interactions that don't hold in adversarial games. Concurrent work (Foerster et al.'s COMA) had proposed a related centralized-critic idea but only for a single shared critic across all agents on a cooperative StarCraft micromanagement task — it doesn't handle agents with conflicting rewards. What was missing was a general-purpose multi-agent algorithm that (1) produces policies using only local observations at execution time, (2) needs no differentiable model of the environment or communication channel, and (3) works across the full spectrum from purely cooperative to purely competitive interaction, including the mixed cooperative-competitive games (predator chasing prey, a deceiver and a mark) that are exactly where independent single-agent RL degrades most visibly.

// how it works

The method

The fix is centralized training with decentralized execution: let each agent's critic see everything during training, but keep each agent's actor blind to everything except its own observation at test time. Concretely, for N agents with deterministic policies μθᵢ (following the DPG/DDPG recipe), agent i gets its own centralized action-value function Qiμ(x, a₁, …, a_N) that is conditioned on some state information x (in the simplest case, just the concatenation of every agent's observation) and every agent's action — not just agent i's own. Because Qi is learned per-agent and takes every agent's action as input, agents can have arbitrary — including directly conflicting — reward functions, which is what makes the method applicable to competitive and mixed games, not just cooperative ones. The actor μθᵢ, by contrast, is a function of agent i's own observation oi alone, so nothing changes about how it's deployed at execution time. The core argument for why this stabilizes training: if you condition on the actions of every agent, the environment is stationary again even while every policy is changing, since P(s′|s, a₁,...,a_N, π₁,...,π_N) = P(s′|s, a₁,...,a_N) regardless of what the π's are — the non-stationarity was only ever a symptom of not conditioning on the other agents' actions.

∇θᵢJ(μᵢ) = 𝔼x,a∼D[ ∇θᵢμᵢ(aᵢ|oᵢ) · ∇aᵢQiμ(x, a₁, …, a_N) |aᵢ=μᵢ(oᵢ) ] // MADDPG gradient — same replay buffer D now stores every agent's actions and observations, (x, x′, a₁,…,a_N, r₁,…,r_N), so the centralized critic's regression target is well-defined even as all N policies move

Two extensions round out the algorithm. First, Eq. 6's target update requires knowing the other agents' policies, which is a mild assumption in simulation but one the paper relaxes anyway: each agent can maintain an approximate model μ̂ⁱⱼ of every other agent j, fit by maximizing the log-probability of j's observed actions plus an entropy regularizer, and swap the true policies for these approximations when computing the critic's target. The paper shows this barely costs anything — agents trained against approximated policies match the success rate of agents trained with the true policies, and converge at about the same speed, even though the fitted listener policy has a fairly large KL divergence from the true one. Second, to keep policies from overfitting to one specific opponent, MADDPG can train an ensemble of K sub-policies per agent (K=3 for keep-away/cooperative-navigation, K=2 for predator-prey in the experiments), randomly selecting one sub-policy to execute each episode and maintaining a separate replay buffer per sub-policy — the idea being that a policy robust to a distribution of opponent behaviors generalizes better than one tuned against a single, momentarily-static adversary.

// what they found

The results

The paper builds on and modifies OpenAI's multi-agent particle environments (originally from Mordatch & Abbeel) — a 2D continuous-space, discrete-time world with agents and landmarks — into six tasks spanning the full cooperative-to-competitive spectrum, then trains MADDPG (two-layer, 64-unit ReLU MLPs throughout) against decentralized baselines: DDPG, REINFORCE, a first-order TRPO implementation, plain actor-critic, and DQN, each run independently per agent. The gap is starkest in cooperative communication: despite the task being simple in principle (a speaker just needs to relay a landmark's color to a listener that can't see it), every independently-trained baseline collapses to the listener ignoring the speaker and drifting to the centroid of all landmarks — MADDPG's listener instead navigates to the correct target 84.0% of the time after 25,000 episodes. In the competitive physical deception task, MADDPG-trained cooperating agents successfully deceive the adversary — covering both landmarks so it can't tell which is the real target — around 94% of the time when L=2, while an adversary trained with DDPG only reaches the correct landmark 16.4% of the time in that setting; DDPG-trained cooperators, by contrast, never successfully deceive a MADDPG adversary, and don't even reliably deceive other DDPG agents once L=4. In cooperative navigation MADDPG agents reach almost the same average per-landmark distance as DDPG but with roughly half the collisions per episode at N=2, and in predator-prey the asymmetry is stark: MADDPG predators chasing DDPG prey rack up 16.1 collisions/episode, versus only 10.3 collisions/episode for the converse (DDPG predators chasing MADDPG prey). In the adversarial covert communication environment (Alice/Bob/Eve, echoing the Abadi & Andersen cryptography setup), Bob's message-reconstruction success is 52.4% under MADDPG versus 25.1% under DDPG — and only the MADDPG-trained Alice learns to encode messages well enough to push Eve's reconstruction down to near-random, without any of the training tricks (loss-shaping, alternating optimization, a hybrid conv/feed-forward architecture) that the original cryptography paper needed.

// does it hold up?

The eval read

The comparison is fair on its own terms — every baseline (DDPG, REINFORCE, TRPO, actor-critic, DQN) is run with the same MLP architecture and the same independent-per-agent training regimen MADDPG is contrasted against, and the paper is explicit that it does not import any of the extra tricks (loss-shaping, alternating training, hybrid architectures) that made the closest prior adversarial-communication result work, which is the right way to isolate the centralized-critic contribution. But the baselines are also the weakest plausible opponents: independent DDPG with no awareness of other agents is close to the worst-case non-stationarity scenario the paper's own Proposition 1 warns about, so "MADDPG beats naively-applied single-agent RL" is a real result but a lower bar than "MADDPG beats the best known multi-agent alternative" — the paper's own related-work section flags a contemporaneous centralized-critic method (Foerster et al., tested on StarCraft micromanagement) that isn't benchmarked against here at all, only discussed qualitatively. The environments themselves are the second soft spot: six particle-world tasks with 2-4 agents, low-dimensional continuous observations, and short horizons are exactly the regime where the paper's own diagnosed failure mode (traditional RL working fine on short-horizon, fixed-position variants) is easiest to fix — the paper explicitly notes this risk about prior short-horizon multi-agent work, and the same caveat applies to how well its own results generalize to environments with many more agents, partial observability at scale, or long-horizon credit assignment. Most tellingly, the paper's own conclusion names the mechanism's central scaling problem without resolving it: the centralized critic Qi(x, a₁,...,a_N) takes every agent's action as input, so its parameter/input footprint grows linearly in N, and the paper explicitly defers a "modular Q function that only considers agents in a certain neighborhood" to future work rather than demonstrating it — meaning the headline results (N ≤ 4) tell you little about whether the centralized-critic recipe survives contact with dozens or hundreds of agents. There's also a training-time requirement worth flagging for anyone citing this as a decentralized method: the centralized critic needs every agent's observations and actions during training, which is free in simulation but is a real assumption to relax in any setting (multi-robot, real-world markets) where agents can't be jointly instrumented — the paper's policy-approximation extension (Section 4.2) addresses exactly this, but it's evaluated on only one of the six environments (cooperative communication), not the competitive ones where knowing an opponent's true policy might matter more.

baselines

Fair relative to what's compared (same architecture, no extra tricks for MADDPG) but all baselines are independent single-agent learners with no multi-agent awareness at all — the weakest class of opponent the paper's own non-stationarity argument predicts will fail; a contemporaneous centralized-critic competitor (COMA) is discussed but never benchmarked head-to-head.

contamination

Not applicable in the usual sense (no train/test leakage risk in an RL environment paper), but there is an environment-design risk: the six particle-world tasks were built by largely the same OpenAI/Berkeley group whose prior grounded-communication environment (Mordatch & Abbeel) this one extends, so the tasks may be tuned to showcase exactly the failure modes MADDPG fixes.

what's missed

Scaling behavior past N=4 agents (the centralized critic's input grows linearly in N and this is left as explicit future work, not measured); how much of the gain is "centralized critic" versus "any coordination signal at all"; and whether the policy-approximation relaxation (Section 4.2) holds up in competitive, not just cooperative, settings — it's tested on one environment only.

would raise confidence

A head-to-head against COMA or another centralized-critic baseline on a shared task; results on an environment with an order of magnitude more agents to probe the linear-input-growth concern the authors themselves raise; and a policy-approximation ablation on the competitive tasks (predator-prey, physical deception), where the incentive to misrepresent one's policy to an opponent is exactly where the approximation could break down.

// bottom line

A genuinely clean idea — condition the critic on everyone, keep the actor local — validated against the right control group (identical architectures, independently trained) but not against the strongest multi-agent alternative available at the time, and on toy environments explicitly chosen to expose non-stationarity failures. The scaling question the paper itself raises and defers (does a per-agent critic whose input grows with N survive beyond 4 agents?) is the one to watch before extrapolating this recipe to any system with real numbers of agents.

// takeaways

What to remember

  1. Centralized training, decentralized execution is the whole trick: give each agent's critic full visibility into every agent's observations and actions during training (restoring stationarity), but keep each agent's actor a function of its own observation only, so nothing changes about how it's deployed. This pattern outlived MADDPG itself and shows up across later multi-agent RL work.
  2. The gains are largest exactly where naive independent RL is diagnosed to fail — tasks needing sustained coordination or explicit adversarial modeling (84.0% vs. near-chance in cooperative communication; MADDPG deceiving DDPG adversaries in physical deception essentially always, and never the reverse) — rather than uniformly across every environment.
  3. The paper is honest about what it doesn't solve: the centralized critic's input grows linearly with the number of agents, tested only up to N≈4, and the fix (a modular, locality-restricted Q function) is named as future work, not delivered — the open question for anyone applying this beyond toy environments.
