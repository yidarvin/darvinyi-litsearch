Longformer: The Long-Document Transformer

Self-attention costs O(n²), so BERT-era models cap out near 512 tokens. Longformer swaps in an attention pattern that scales linearly — a sliding local window plus a handful of task-placed global tokens — and shows it can be dropped into a pretrained RoBERTa to read documents of 4,096 tokens and beyond.

Iz Beltagy, Matthew E. Peters, Arman Cohan · Allen Institute for AI
venue arXiv citations 5,560 topic Foundation Models source ↗

// the gap

What problem is this solving?

The Transformer's self-attention compares every token to every other token, so its time and memory both grow as O(n²) in the sequence length n. That quadratic wall is why BERT-style models cap their input at 512 tokens: past a few thousand tokens the attention matrix simply runs out of GPU memory. Yet plenty of real NLP — multi-hop QA over several Wikipedia articles, coreference across a full document, classifying a long news article, summarizing a scientific paper — lives at thousands of tokens.

The field's workarounds all leak information. You can truncate the document (and throw away everything after token 512), chunk it into 512-token windows and process each separately (losing any interaction that crosses a chunk boundary), or build a two-stage retrieve-then-read pipeline (which cascades errors from the retriever into the reader). Each forces a complex, task-specific architecture whose only job is to stitch fragments back together.

A parallel line of work — Transformer-XL, Adaptive Span, Sparse Transformer, Reformer — had attacked the quadratic cost, but almost entirely in the service of autoregressive (left-to-right) language modeling. As the authors put it, that focus "has led to the development of models with limited applicability": those models are left-to-right and so unsuitable for the bidirectional, pretrain-then-finetune transfer-learning recipe that was winning everything else in NLP. The open gap was a long-context attention that is (a) bidirectional, (b) a drop-in replacement for standard self-attention, and (c) actually validated on document-level downstream tasks, not just LM perplexity.

// how it works

The method

Longformer keeps the Transformer untouched and replaces only the attention pattern: instead of the dense n×n matrix, it computes a sparse, banded one. Two ingredients combine to do this while keeping cost linear.

1 · Sliding window (local) attention

Each token attends only to a fixed window of w neighbours — ½w on each side. That makes one layer cost O(n × w), linear in n. Crucially, stacking layers grows the effective receptive field the way CNNs do: with ℓ layers the top layer sees ℓ × w tokens, so information still propagates across the whole sequence even though no single layer looks far.

2 · Dilated window — reach without cost

To widen the receptive field without adding compute, the window can be dilated (gaps of size d between attended positions, exactly like dilated convolutions). The reach becomes ℓ × d × w — "tens of thousands of tokens even for small values of d." They use multi-headed dilation: some heads run with no dilation to hold onto local context, while a couple of heads dilate to reach far. Dilation is used for the character-LM experiments; for the pretrained model it is dropped because it does not match RoBERTa's learned weights.

3 · Global attention — a task-placed inductive bias

Local windows alone can't, say, let a [CLS] token summarize the whole document, or let every passage token see the question in QA. So Longformer adds global attention on a few hand-chosen positions. Global attention is symmetric: a global token attends to the entire sequence, and every token attends back to it. Because the number of global tokens is small and independent of n, the combined cost is still O(n). Where you place them is task-specific — the [CLS] token for classification, all question tokens for QA — making global attention a cheap, explicit place to inject task structure. Global tokens get their own set of query/key/value projections (Qg,Kg,Vg), initialized to copy the sliding-window projections.

full self-attention:  O(n2)   →   window + global:  O(n·w) + O(n) = O(n) // the same softmax(QKᵀ/√dk)·V attention — but QKᵀ is computed only on a fixed band of diagonals plus a few global rows/columns

Making it a drop-in for RoBERTa

The headline trick is engineering, not theory. The banded matrix multiply isn't supported by PyTorch/TensorFlow, so they ship three implementations: a loop (memory-efficient, supports dilation, "unusably slow", testing only), chunks (vectorized, non-dilated, used for finetuning), and a custom CUDA kernel written in TVM (used for language modeling). To extend a pretrained RoBERTa from 512 to 4,096 positions, they don't randomly initialize the new position embeddings — they copy RoBERTa's 512 learned embeddings repeatedly, preserving the model's strong learned bias toward local positions. After only a short continued-MLM pretrain, RoBERTa reads 8× longer documents with no architectural surgery. The same copy-and-extend recipe (BART's 1K positions copied 16× to 16K) produces LED, the Longformer-Encoder-Decoder for long seq2seq tasks like summarization.

// what they found

The results

Character language modeling. With the dilated window and staged training (5 phases growing sequence length from 2,048 up to 23,040, evaluated at length 32,256), the small model sets a new state of the art at 1.10 BPC on text8 and 1.00 BPC on enwik8 (41M params). A large 102M-param Longformer reaches 0.99 BPC on enwik8 — matching the Sparse Transformer (≈100M) and beating the 18-layer Transformer-XL (1.03), while only slightly trailing models with 2–3× the parameters (Adaptive 0.98, Compressive 0.97).

Pretrain–finetune (the point of the paper). Continuing MLM from the RoBERTa checkpoint, the controlled comparison is Longformer-base vs. the very RoBERTa it was initialized from, where the RoBERTa baseline chunks the document into 512-token segments. Longformer wins on all six tasks:

The gains track how much long context the task needs: huge on Hyperpartisan and WikiHop, near-noise on OntoNotes and IMDB (where most documents are short or where mentions sit close together). Scaling up, Longformer-large set new SOTA on WikiHop (81.9 F1, +3.6) and TriviaQA (77.3 F1, +4) at submission time (May 2020), and placed second on the published HotpotQA leaderboard at 73.2 joint F1 — a point behind the GNN-based HGN (74.2), notable because Longformer uses no graph machinery.

What actually drives the gains? The WikiHop ablation is the most revealing table in the paper. Stripping global attention and its separate projections costs 8.3 points — by far the largest single effect, much bigger than dropping MLM pretraining (−0.6) or shortening the sequence.

Long-document summarization (LED). On the arXiv dataset (90th-percentile length ≈ 14.5K tokens), LED-large reading 16,384 tokens reaches 46.63 / 19.62 / 41.83 ROUGE-1/2/L, edging out BigBird (46.63 / 19.02 / 41.77) — even though LED is merely initialized from BART with no summarization pretraining, while BigBird continues from Pegasus. The same model at 4,096 tokens only scores 44.40 / 17.94 / 39.76, and the ROUGE-1 curve climbs 35.21 → 44.48 → 46.23 as input grows 1K → 4K → 16K: the win is the context length itself.

// does it hold up?

The eval read

The central comparison is unusually fair: Longformer-base is benchmarked against the exact RoBERTa checkpoint it was warm-started from, so any improvement is attributable to attention-and-context rather than extra data or scale. The ablation that runs Longformer "as RoBERTa" (seqlen 512, full n² attention) and lands slightly below RoBERTa-base nails this shut — the gains are not a free lunch from extra pretraining.

But read the magnitudes honestly. Outside the two long-context standouts, the controlled wins are small — +0.2 on OntoNotes, +0.4 on IMDB, +0.9 on TriviaQA and HotpotQA — and these are single dev-set numbers with no variance reported. Only Hyperpartisan is averaged over five seeds, and that benchmark has just 645 documents, so the eye-catching +7.4 is also the noisiest point in the table. The paper's own ablation shows the workhorse is the hand-placed global attention (−8.3 without it), which is a manually specified, task-by-task inductive bias rather than something the model learns — so part of the "linear long-context attention" story is really "we engineered the right global tokens for each task."

On modeling, the char-LM "SOTA" holds only in the parameter-matched regime; bigger Sparse/Adaptive/Compressive models match or beat it. And the authors are commendably candid about timing: they note that BigBird later beat their WikiHop/TriviaQA leaderboard numbers — using ~16× more pretraining compute — and that ETC, BigBird and GMAT explored near-identical local+global ideas contemporaneously. Longformer's durable edge is less the idea than the packaging: a maintainable CUDA kernel, the copy-init position trick, and a genuine drop-in into pretrained encoders.

baselines

Strong and apples-to-apples: the same RoBERTa/BART it initializes from, plus the standard chunking pipeline. Char-LM compares to Transformer-XL, Sparse, Adaptive, Compressive. The "as-RoBERTa" ablation confirms gains aren't from extra pretraining.

contamination

Not analyzed, and largely orthogonal in 2020 — these are supervised-finetuning benchmarks, not few-shot prompts. The bigger reproducibility caveat is that all main numbers predate public BigBird/ETC, so cross-model comparisons mix concurrent, differently-resourced systems.

what's missed

No significance tests or error bars on the single-seed dev numbers; several wins (OntoNotes, IMDB, HotpotQA, TriviaQA) sit within plausible noise. Where to place global attention is hand-chosen per task — the −8.3 ablation shows it carries most of the lift, so it's a tuned knob, not a free property of linear attention.

would raise confidence

Multi-seed runs with variance on every task; a like-for-like compute match against BigBird/ETC; and an ablation that learns rather than hand-specifies global-token placement, to separate "long context helps" from "good task engineering helps."

// bottom line

A clean, practical answer to the 512-token wall: keep the Transformer, sparsify only the attention pattern, and bolt on a few global tokens for task structure. The controlled RoBERTa comparison is honest and the linear-scaling claim is real, but the strongest downstream gains come from long-context tasks and from a manually placed global-attention bias — so the takeaway is "long context, cheaply, as a finetuning detail," not "linear attention is free accuracy."

// takeaways

What to remember

  1. Sliding window + global = O(n). A fixed local window makes attention linear and, stacked over layers, still propagates information document-wide; a few symmetric global tokens add task structure without breaking the linear budget.
  2. Long context as a drop-in, not a rebuild. Copying RoBERTa's 512 position embeddings to 4,096 (and BART's 1K to 16K for LED) lets a pretrained encoder read 8–32× longer after only brief continued pretraining — the move that made long-context a finetuning option.
  3. The gain is context-shaped, and global attention does the heavy lifting. Wins are large where the task genuinely needs distant context (Hyperpartisan +7.4, WikiHop +2.6) and near-zero otherwise; ablating global attention costs 8.3 points, more than any other component.
