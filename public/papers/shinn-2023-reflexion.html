Reflexion: Language Agents with Verbal Reinforcement Learning

An LLM agent that learns from failure without a single gradient step: after each attempt it writes itself a natural-language post-mortem, files it in memory, and reads it back on the next try. The result is 91% pass@1 on HumanEval and 130/134 ALFWorld tasks solved — and a sharp lesson about how far self-evaluation can carry an agent.

Noah Shinn, Federico Cassano, Beck Labash, Ashwin Gopinath, Karthik Narasimhan, Shunyu Yao

// the gap

What problem is this solving?

Reinforcement learning is how you teach an agent to improve from trial and error — but classical RL pays for that improvement in gradient updates: thousands of rollouts and an expensive fine-tune to push a scalar reward into the policy's weights. For an LLM agent poking at a compiler, a website, or a household simulator, that loop is wildly impractical — you would have to retrain a frontier model after every botched episode.

And the teaching signal itself is thin. A scalar reward — "you failed" — says nothing about which of an agent's thirty actions was the mistake, or what it should have done instead. A model can fail the same task the same way forever and the reward never tells it why.

The contemporary alternatives sidestepped weight updates but hit their own ceilings. ReAct interleaves reasoning and acting within a single episode, but it carries no memory across attempts, so it repeats its mistakes. Refinement methods like Self-Refine edit a single output with self-feedback, but they don't persist lessons across trials and aren't grounded in environment feedback. The gap Reflexion targets: a way for a frozen LLM agent to genuinely learn across attempts — to convert sparse, end-of-episode feedback into something rich enough to change its next decision — with no gradient step at all.

// how it works

The method

Reflexion reframes the policy update as language, not gradients — what the authors call verbal reinforcement learning. Three LLM-driven roles form a loop around an environment:

  • Actor — a frozen LLM (prompted with Chain-of-Thought or ReAct) that generates a trajectory τ: the sequence of thoughts, actions, and observations. It also reads from a memory buffer.
  • Evaluator — scores the trajectory into a scalar reward. The instantiation depends on the domain: exact-match grading for QA, a hand-written heuristic for the household tasks, or self-generated unit tests for code.
  • Self-Reflection — the heart of the method. It reads the trajectory and its scalar reward and writes a specific, first-person verbal post-mortem ("I failed because I assumed both authors shared the same profession; I should have verified each separately").

That reflection is appended to an episodic memory — a sliding window of the last Ω reflections (set to 1–3 to stay inside the context limit; code uses just 1, reasoning and decision-making use 3) — which is fed back into the Actor's context on the next trial. The agent retries; if the Evaluator still says fail, it reflects again. No weights move. The entire "policy optimization" happens in the prompt.

rt = Me(τt)   →   srt = Msr(τt, rt)   →   mem ← mem ∪ {srt}   →   πθ, θ = {Ma, mem} // the Evaluator scores the trajectory; the Self-Reflection model turns that scalar into verbal feedback; the feedback is stored — and the "parameters" being optimized are the memory of reflections, not the network weights

For the programming domain — the most "deployable" setting, because correctness can be checked without a human — the Evaluator is a self-generated test suite: the agent uses CoT prompting to produce diverse candidate tests, filters them for syntactic validity by building an abstract syntax tree, and samples up to 6. To keep the comparison honest about contamination, the authors also introduce LeetcodeHardGym, 40 Leetcode hard-rated problems released after GPT-4's October 8, 2022 pre-training cutoff.

// what they found

The results

Reflexion improves over strong baselines by 22% absolute on ALFWorld, 20% on HotPotQA, and 11% on HumanEval. The shape of the gains matters as much as the magnitude.

Sequential decision-making (ALFWorld). ReAct + Reflexion solves 130 of 134 tasks using only a simple heuristic to flag hallucination and inefficient planning, and keeps learning across 12 consecutive trials — whereas ReAct-only plateaus after trial 6–7 and stalls at a 22% hallucination rate with no sign of recovery. Reflexion drives both failure modes toward zero.

Reasoning (HotPotQA, 100 questions). The baselines — CoT-only, ReAct-only, CoT(GT)-only — fail to probabilistically improve at all: no task they got wrong on trial 1 was later solved by retrying. Reflexion breaks that flatline. Even when given ground-truth context, CoT(GT) still misses 39% of questions, and Reflexion recovers 14% of accuracy without ever seeing the answer. An ablation isolating the verbal step shows self-reflection adds an 8% absolute boost over merely re-feeding the prior trajectory (episodic memory).

Programming. Reflexion sets new state-of-the-art pass@1 across most code benchmarks — but not all, and the exception is the most revealing result in the paper.

HumanEval Python jumps to 91.0 (from GPT-4's 80.1), Leetcode-hard doubles (7.5 → 15.0), and the Rust results show the loop is language-agnostic. But on MBPP Python, Reflexion loses to a single GPT-4 sample (77.1 vs 80.1). The cause is diagnostic: MBPP Python's self-generated tests have a 16.3% false-positive rate (tests pass on incorrect code) versus HumanEval Python's mere 1.4% — so the agent prematurely submits wrong solutions it believes are right. An ablation on the 50 hardest Rust problems confirms both halves are load-bearing: dropping test generation drops accuracy below baseline (0.60 → 0.52, because the agent can't tell when to stop and makes harmful edits), and dropping the verbal reflection erases the gain entirely (0.60), while the full method reaches 0.68.

// does it hold up?

The eval read

Reflexion is a genuinely influential idea, and the paper is unusually candid about its own failure mode. But the headline numbers deserve a careful read, because they lean on two things the method won't always have.

An oracle sits in the loop. Reflexion improves by retrying, and retrying needs a signal that says "right or wrong" between trials. In two of the three domains that signal is privileged: ALFWorld and HotPotQA use environment success / exact-match against the ground-truth answer as the binary that triggers reflection and stops the loop. The protocol lets the agent retry until 3 consecutive failures — effectively best-of-many under an oracle stopping rule the agent wouldn't have in deployment. Part of what looks like "learning" is "keep sampling until an oracle says stop." The honest, deployable version is the code setting, where the Evaluator is self-generated tests — and that is exactly where the method stumbles.

The evaluator is the ceiling. Reflexion is only as good as the reward it can compute for itself. The MBPP-Python regression (77.1 < 80.1) is the proof: a 16.3% false-positive test rate makes the same machinery that scores 91% on HumanEval actively hurt. The flagship HumanEval number rides on that benchmark's unusually clean 1.4% false-positive rate — a property of the benchmark, not a guarantee of the method.

Compute-unequal baselines. Reflexion is benchmarked against single-sample GPT-4 (one generation, pass@1). But Reflexion spends many model calls per task — generate, test, reflect, retry, across up to a dozen trials, plus separate Evaluator and Self-Reflection calls. There is no compute-normalized or pass@k-matched control, so it is unclear how much of the gain is the reflection mechanism versus simply spending 5–10× the inference budget.

baselines

Mostly single-sample GPT-4 at pass@1. No compute- or pass@k-matched control, so the gains conflate the reflection mechanism with a much larger inference budget; the natural missing baseline is best-of-n at equal token cost.

contamination

Handled well for the new LeetcodeHardGym (problems released after GPT-4's Oct-8-2022 cutoff). But HumanEval and MBPP predate GPT-4 and are plausibly in its pretraining, so the dramatic HumanEval figure should be read with that caveat.

what's missed

Two of three domains feed the agent a ground-truth success signal between trials — an oracle it lacks in the wild — and pass@1 hides the many-trial, oracle-stopped retry budget. The appendix's weak-model run (starchat-beta: 0.26 → 0.26) shows the method only fires on strong models.

would raise confidence

Equal-compute best-of-n baselines; results across multiple model families and larger task sets (100 HotPotQA / 134 ALFWorld / 40 Leetcode is thin); and an "evaluator-quality vs. accuracy" study, since the false-positive rate is what determines whether the method helps or hurts.

// bottom line

Replace the gradient with a written memo and a frozen model can iterate its way to large gains — that is the real, durable contribution, and the code setting (self-generated tests on a post-cutoff benchmark) is a legitimately deployable demonstration. But the flagship numbers lean on ground-truth success signals between trials and on compute-unmatched baselines, and the MBPP-Python regression is an honest preview of the method's true dependency: Reflexion is exactly as reliable as the evaluator you can give it.

// takeaways

What to remember

  1. Verbal reinforcement learning. A frozen LLM can "learn" across attempts by writing itself a natural-language post-mortem and reading it back — the optimized "parameters" are an episodic memory of reflections, not weights. No fine-tuning, no rollouts at scale.
  2. Big gains where feedback is grounded. 91% pass@1 on HumanEval, 130/134 ALFWorld tasks, a doubling on Leetcode-hard — but the gains invert where the self-evaluator is unreliable (MBPP Python 77.1 < GPT-4 80.1, traced to a 16.3% false-positive test rate).
  3. Both halves are required. Ablations show removing self-generated tests or the verbal reflection erases the benefit: grounding tells the agent whether it failed; reflection tells it why.
  4. The seed of the modern agent loop — and its caution. Memory + self-critique + retry is now standard in coding and web agents; Reflexion is also the clearest early demonstration that an agent's self-evaluation is its ceiling.
