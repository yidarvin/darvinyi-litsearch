FlashAttention: Fast and Memory-Efficient Exact Attention with IO-Awareness

Attention is slow not because it does too much math, but because it shuttles a giant N×N matrix in and out of GPU memory. FlashAttention computes the exact same attention while never writing that matrix to HBM — turning a memory-bound operation into a fused, on-chip kernel that runs up to 7.6× faster and uses linear memory.

What problem is this solving?

Self-attention has time and memory complexity quadratic in sequence length: for input length N and head dimension d, it forms an N×N scores matrix S = QKᵀ, softmaxes it, then multiplies by V. That N×N matrix is the bottleneck — it is what makes long context expensive in both runtime and GPU memory.

A whole literature of approximate attention — sparse (Reformer, Sparse Transformers), low-rank (Linformer, Performer), and combinations (Longformer, Big Bird) — chased this by cutting the asymptotic FLOP count to linear or near-linear. Yet most of them delivered little or no wall-clock speedup and never gained wide adoption. The paper's diagnosis is sharp: these methods optimize FLOPs, a quantity that doesn't track wall-clock time, and ignore the cost of memory access (IO).

The missing principle is IO-awareness — accounting for reads and writes between the levels of the GPU memory hierarchy. On an A100, on-chip SRAM runs at ~19 TB/s but holds only ~20 MB, while HBM runs at ~1.5 TB/s but holds 40–80 GB. Because compute has outpaced memory bandwidth, attention is memory-bound: most of its time is spent reading and writing the N×N matrix to slow HBM, not doing arithmetic. The real question isn't "can we do fewer FLOPs?" but "can we touch HBM fewer times?"

// how it works

The method

FlashAttention computes exact attention (no approximation) with far fewer HBM accesses, by refusing to ever materialize the full N×N matrices S and P = softmax(S) in HBM. It rests on two classic techniques — tiling and recomputation — fused into a single CUDA kernel.

Tiling with online softmax

Split Q, K, V into blocks. Loop over blocks of K, V (outer loop), loading each to SRAM; for each, loop over blocks of Q (inner loop), compute the block's scores on-chip, and accumulate the output. The obstacle is that softmax couples an entire row, so you seemingly need the whole row before normalizing. FlashAttention sidesteps this with the online-softmax trick: carry a running max m and running denominator ℓ per row, and rescale the partial output as each new block arrives. Concatenating two blocks x⁽¹⁾, x⁽²⁾:

m(x) = max( m(x(1)),  m(x(2)) )

ℓ(x) = e m(x⁽¹⁾)−m(x) ℓ(x(1))  +  e m(x⁽²⁾)−m(x) ℓ(x(2)) // softmax statistics combine across blocks — so the reduction is computed one tile at a time, never storing the full row

Recomputation instead of storing the matrix

The backward pass normally needs S and P to compute gradients — exactly the O(N²) matrices we refused to store. FlashAttention keeps only the output O and the softmax statistics (m, ℓ), then recomputes S and P on-chip from blocks in SRAM during the backward pass — a form of selective gradient checkpointing. This adds FLOPs, but because it slashes HBM traffic, the backward pass is still faster, not just smaller.

Why it's provably better, not just engineered better

The payoff is an IO-complexity result. Standard attention makes Θ(Nd + N²) HBM accesses; FlashAttention makes Θ(N²d²M⁻¹), where M is the SRAM size. For typical d (64–128) and M (~100 KB), d² is many times smaller than M — so FlashAttention touches HBM many times less (up to 9× fewer), even though Theorem 1 guarantees it returns the exact same softmax(QKᵀ)V in O(N²d) FLOPs and only O(N) extra memory.

standard  Θ( Nd + N2 )   ⟶   FlashAttention  Θ( N2d2 M−1 ) // HBM accesses. A matching lower bound (Prop. 3) proves no exact-attention algorithm can asymptotically beat this across all SRAM sizes

Finally, FlashAttention is a primitive the approximate methods can build on: block-sparse FlashAttention skips zero blocks of the mask, reducing IO to Θ(Nd + N²d²M⁻¹s) for sparsity fraction s — faster than any approximate attention method the authors tested.

// what they found

The results

The central measurement: for GPT-2 medium attention (seq 1024, head dim 64, 16 heads, batch 64, A100), FlashAttention does more GFLOPs (75.2 vs 66.6, from backward recomputation) yet cuts HBM traffic 9× (4.4 vs 40.3 GB) and runtime 5.7× (7.3 vs 41.7 ms). Memory accesses — not arithmetic — set the wall-clock time, exactly as the IO theory predicts.

Faster end-to-end training. BERT-large reaches the MLPerf 1.1 target (72.0% masked-LM accuracy) in 17.4 vs 20.0 minutes — 15% faster than Nvidia's training-speed record (8×A100, averaged over 10 runs). GPT-2 trains up to 3× faster than HuggingFace and ~1.7–1.8× faster than Megatron-LM at the same perplexity. The long-range arena benchmark runs 2.4× faster (block-sparse, 2.8×).

Higher quality from longer context. Because the kernel is fast and memory-linear, the authors push context length: GPT-2 small at 4K context with FlashAttention is 30% faster than Megatron at 1K, while scoring 0.7 better perplexity (17.5 vs 18.2). Longer sequences add 6.4 points of lift on long-document classification (MIMIC-III 52.8→57.1 going 512→16K; ECtHR 72.2→80.7 going 512→8K). And FlashAttention produces the first Transformer to beat chance on Path-X (seq 16K, 61.4%), with block-sparse reaching Path-256 (seq 64K, 63.1%) — capabilities that were previously out of memory.

// does it hold up?

The eval read

This is the rare paper where the headline metric is exactly the right one. The claim is "exact attention, measured wall-clock speedup" — and both halves are honest. Exactness means there's no quality/speed trade-off to hide: FlashAttention reproduces the baselines' training and validation curves to the same perplexity, so the numerical-equivalence claim is checkable, not asserted. Wall-clock time is the metric the approximate-attention literature dodged by reporting FLOPs — and this paper's whole thesis is that the dodge is why those methods didn't catch on.

The baselines are strong and contemporary for 2022: Nvidia's MLPerf 1.1 record for BERT, well-optimized HuggingFace and Megatron-LM for GPT-2, and the full LRA zoo (Linformer, Performer, Reformer, Linear/Local Attention, Smyrf) for approximate attention. The speedups are not asymptotic hand-waving — they come paired with an IO-complexity proof and a matching lower bound (Prop. 3) showing no exact algorithm can do asymptotically better across SRAM sizes. The honest soft spots are about generality and the quality claims, not the speed result.

baselines

Fair and current — the MLPerf BERT record, HuggingFace + Megatron-LM for GPT-2, and the LRA approximate-attention suite. Speedups are wall-clock, the metric prior work avoided; FlashAttention is exact, so the comparison isn't quality-for-speed.

contamination

Not a meaningful risk here — this is a systems result, scored by runtime, HBM bytes, and perplexity, none of which leak from a test set. The one caveat: LRA accuracy is famously tuning-dependent (the authors flag in footnote 3 that their reproduced baselines beat the originals), so the +0.5 LRA-average (59.3→59.8) is within tuning noise and shouldn't be read as a quality win.

what's missed

Numbers are hardware-specific (A100, fixed head dim / batch). IO-optimality holds "within constants" on a single GPU and leans on d²≪M, so the edge narrows for large head dimensions. The kernel is hand-written CUDA, re-coded per attention variant and not portable across GPU generations (a stated limitation). Path-X 61.4% is "better than chance," not solved. The quality gains come from longer context the speedup affords, not the kernel itself.

would raise confidence

Portability across GPU architectures and a high-level → IO-aware compiler (both flagged as future work, later partly addressed by FlashAttention-2/3); a multi-GPU IO analysis that accounts for inter-device transfer.

// bottom line

An unusually clean result: a measured systems speedup backed by an IO-complexity proof and a lower bound, with exactness removing the usual quality caveat. The honest limits are scope (one GPU, hand-tuned CUDA, A100-specific constants), not validity. It reframed efficient attention around memory movement rather than FLOPs and became the kernel under essentially every long-context model that followed.

// takeaways

What to remember

  1. Attention is memory-bound, not compute-bound: its cost is dominated by reading/writing the N×N matrix to slow HBM, so the right lever is fewer memory accesses — not fewer FLOPs.
  2. Tiling + online softmax + recomputation, fused into one kernel, compute exact attention while never materializing the N×N matrix in HBM — cutting HBM traffic 9× and using memory linear in N, even at a higher FLOP count.
  3. The result is grounded in an IO-complexity bound (Θ(N²d²M⁻¹) vs Θ(Nd+N²)) with a matching lower bound, and translates to real wins: 15% faster BERT, 3× faster GPT-2, and the first Transformers to clear Path-X / Path-256 — making FlashAttention the efficiency backbone of the long-context era.
