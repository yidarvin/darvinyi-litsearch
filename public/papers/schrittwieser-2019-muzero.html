2020 · Nature · Post-training & RL

Mastering Atari, Go, chess and shogi by planning with a learned model

MuZero learns a model of its environment purely from raw observations — never given the rules of the game or a simulator of the environment — and plans over that learned model with AlphaZero-style Monte Carlo Tree Search. The model is trained to predict only the three quantities planning actually needs (policy, value, reward), not to reconstruct what things look like. The result: superhuman play in Go, chess and shogi that matches AlphaZero despite never seeing the rules, and a new state of the art across all 57 Atari games, a domain where model-based planning had always lagged behind model-free methods.

Julian Schrittwieser, Ioannis Antonoglou, Thomas Hubert, Karen Simonyan, Laurent Sifre, Simon Schmitt, Arthur Guez, Edward Lockhart, Demis Hassabis, Thore Graepel, Timothy Lillicrap, David Silver — DeepMind, University College London

// the gap

What problem is this solving?

Tree-based planning is the reason machines beat humans at chess, Go, checkers and poker. But every one of those wins leans on the same crutch: a perfect simulator. AlphaZero can search a thousand hypothetical futures per move because it can ask the rules engine "what happens if I play this?" and get a ground-truth answer instantly, for free, with no error. That crutch simply doesn't exist for most problems anyone actually cares about — robotics, industrial control, or, as a convenient visually-rich stand-in the paper uses throughout, the 57 games of the Atari Learning Environment, where there is no accessible rules engine, only raw pixels and a reward counter.

Model-based RL is the obvious fix: learn a model of the dynamics first, then plan against the model instead of the real thing. In practice this has repeatedly failed to reach the state of the art in visually complex domains. The reason is that essentially all prior approaches point the model at the wrong target — they try to reconstruct the entire observation stream at the pixel level (or a latent space sufficient to reconstruct it), which "focuses the majority of the model capacity on potentially irrelevant detail": exact cloud shapes, sprite textures, background parallax — none of which changes which move to play. So the modeling problem is harder than it needs to be, prior methods "remain far from the state of the art" in Atari, and the field split in two: model-free algorithms (DQN, Rainbow, R2D2) win on visually complex domains but can't do explicit multi-step lookahead, while planning algorithms (AlphaZero) win on precision-lookahead domains but need the rules handed to them. MuZero's question is whether a single algorithm can do both — and the answer turns on what the model is asked to predict, not on giving up on models altogether.

// how it works

The method

MuZero's model has no obligation to look like the real environment at all. It is built from three learned functions, jointly trained end-to-end:

  • Representation hθ: encodes the past observations o1,…,ot (e.g. the last 32 Atari frames, or the last 8–100 board positions) into an initial hidden state s0. No pixels come back out — this is a one-way door.
  • Dynamics gθ: a recurrent function that, given a hidden state sk-1 and a hypothetical action ak, produces an immediate reward rk and the next hidden state sk — the internal, learned stand-in for "the rules."
  • Prediction fθ: from any hidden state sk, outputs a policy pk and a value vk, exactly like AlphaZero's dual-headed network.

The pivotal design choice is what these hidden states are for. "There is no direct constraint or requirement for the hidden state to capture all information necessary to reconstruct the original observation… nor is there any requirement for the hidden state to match the unknown, true state of the environment." The states are free to encode whatever is useful for predicting future policy, value and reward — nothing more. As the authors put it, "the agent can invent, internally, the rules or dynamics that lead to most accurate planning." That single sentence is the whole thesis: model only what planning needs, and let go of everything else.

Planning itself is the same Monte Carlo Tree Search machinery as AlphaZero — selection by a pUCT bound trading off the prior P(s,a) against the visited value Q(s,a), expansion via one call to gθ and fθ per simulation, backup of an n-step bootstrapped return — generalized to single-agent domains with discounted, arbitrary-magnitude intermediate rewards rather than only two-player, undiscounted, ±1 terminal outcomes. Because the search never needs the real simulator once it has s⁰, it is, structurally, the same cost per simulation as AlphaZero's search — the "environment" it searches inside is just the model's own hidden-state graph.

Compared directly against AlphaZero, MuZero removes the simulator from exactly the three places AlphaZero used it: state transitions inside the tree (now gθ), legal-action masking inside the tree (now not done at all beyond the root — the network learns not to propose illegal moves), and terminal-state detection (now just another hidden state the network learns to treat as absorbing). Training minimizes one joint loss, unrolled for K = 5 hypothetical steps per sampled trajectory, matching predicted policy, value and reward against search-derived and observed targets:

lt(θ) = Σk=0K [ lr(ut+k, rtk) + lv(zt+k, vtk) + lp(πt+k, ptk) ] + c‖θ‖² // reward + value + policy loss at every one of the K unrolled steps, plus L2 regularization; π and z come from the MCTS search itself, u from the real observed reward

Everything downstream is standard AlphaZero infrastructure: 16 residual blocks (vs. AlphaZero's 20) for representation and dynamics, 800 simulations per search in board games and 50 in Atari, 1 million training mini-batches, self-play data continuously refreshed from the latest network checkpoint. The dynamics function is deliberately deterministic — "the extension to stochastic transitions is left for future work" — a scope limitation worth keeping in mind (see below).

// what they found

The results

In Go, MuZero slightly exceeded AlphaZero's Elo despite using a smaller network per position evaluation (16 residual blocks vs. AlphaZero's 20) — the authors read this as MuZero "caching its computation in the search tree," using each additional dynamics-function application to deepen its understanding of the position rather than just re-evaluating a static board. In chess and shogi, MuZero's Elo curve converges to match AlphaZero's despite never being given the rules — no legal-move generator, no check/checkmate detector, no promotion logic; all of it reconstructed implicitly inside the learned dynamics function.

The headline result is Atari-57: MuZero achieved a new state of the art for both mean and median human-normalized score, outperforming the previous best model-free method, R2D2, in 42 of 57 games, and outperforming the previous best model-based method, SimPLe, in all evaluated games. Per Table 1, at 1M training steps and 20.0B environment frames over 12 hours of training, MuZero reaches 4999.2% mean / 2041.1% median human-normalized score, versus R2D2's 4024.9% mean / 1920.6% median over 37.5B frames and 5 days. Counting best-result-per-game across the full suite (supplementary Tables S1–S2), MuZero takes the top score in 37/57 games under standard no-op starts and 46/57 under human-expert starts, against R2D2's 13 and Ape-X's 5.

The data-efficient variant, MuZero Reanalyze, re-runs MCTS on old trajectories with the latest network parameters instead of collecting fresh experience, and reaches 731.1% median normalized score using only 200M frames — the same budget as IMPALA (191.8%), Rainbow (231.1%) and LASER (431%), all of which it clears by a wide margin. Ablations in Figure 3 show the learned model scales gracefully with search budget in Go (matching a perfect simulator even at 100× the thinking time seen during training) but far less so in Atari, where performance plateaus around 100 simulations per move — attributed to greater model inaccuracy — and that MCTS-based training beats an otherwise-identical Q-learning objective on Ms. Pac-Man, learning faster and converging to a much higher final score.

// does it hold up?

The eval read

The aggregate Atari-57 mean/median numbers are real and the comparisons are apples-to-apples on frames and games — but the paper's own supplementary tables quietly contain the caveat the abstract omits. MuZero scores 0.0% normalized on Montezuma's Revenge (a raw score of 0.0, identical to random play, versus R2D2's 2,061.3), 0.0%–0.3% on Venture, and single-digit-percent on Pitfall and Solaris — the classic hard-exploration, sparse-reward Atari titles. A mean-and-median-over-57-games headline is exactly the kind of metric that can hide a handful of total failures behind dozens of blowout wins (MuZero clears 10,000%+ normalized on alien, centipede and krull); the paper reports this honestly in Tables S1–S2, but the main text's framing leans entirely on the aggregate.

The "no knowledge of the rules" framing is accurate but worth pressure-testing: chess, Go and shogi remain fully observable, deterministic, discrete, two-player zero-sum games — the easiest class of environment for a learned model to internalize, precisely because there is no stochasticity or partial observability to get wrong. MuZero indeed never sees a legal-move generator, but it still gets to query a real emulator for the ground-truth outcome of every self-play game it trains on. The paper is explicit that the dynamics function "is represented deterministically; the extension to stochastic transitions is left for future work" — so the harder claim, that this recipe would work in a genuinely stochastic, partially-observed real-world setting (the robotics and industrial-control motivation from the introduction), is not actually tested here.

The scale is also easy to undersell by quoting "12 hours." Appendix G reports that each Atari game trained with 8 TPUs for training and 32 TPUs for self-play, and each board game with 16 TPUs for training and 1,000 TPUs for self-play — third-generation Google Cloud TPUs, not commodity hardware, and R2D2's "5 days" comparison runs on a different (GPU-based) infrastructure entirely, so the wall-clock numbers in Table 1 are not a controlled comparison of compute cost, only of elapsed time. Reviewers of later work (DreamerV3, for instance) have specifically flagged that "MuZero's authors did not release an implementation and the algorithm contains several complex components, making it challenging to reproduce" — a real, separately-verifiable reproducibility gap this paper doesn't address.

baselines

Fair and current for the time — R2D2, Ape-X, IMPALA, Rainbow, LASER and SimPLe were the genuine state of the art in their respective (model-free / model-based) categories in 2019, each compared at its own established frame budget.

contamination

Not applicable in the usual sense (self-play + a fixed simulator, no held-out test set to leak into) — but the "human normalized score" baseline itself inherits the well-known reproducibility caveats of fixed human-tester scores from the original DQN paper, a standard Atari-benchmark limitation this paper doesn't re-litigate.

what's missed

The abstract's aggregate framing; Montezuma's Revenge, Venture, Pitfall and Solaris are near-total failures (0.0%–3.4% normalized) buried in supplementary tables, and the dynamics function's deterministic-only design is a real scope limit for the paper's own stated motivation (real-world domains with unknown, likely stochastic dynamics).

would raise confidence

A released reference implementation (later addressed by community reimplementations and DeepMind's own MuZero-derived open-source work); a stochastic-dynamics ablation; and a compute-normalized (not just wall-clock) comparison against R2D2/Ape-X on identical hardware.

// bottom line

The central idea — train a model to predict only reward, value and policy, never to reconstruct the world — genuinely unifies precision board-game planning and visually complex Atari under one algorithm, and the Atari-57 numbers are a real, honest state of the art. But "no knowledge of the rules" is a narrower claim than it sounds against still-deterministic, fully-observable board games, the compute footprint is industrial-scale, and the sparse-reward failure modes (Montezuma's Revenge chief among them) are exactly the ones a from-first-principles planner would need to solve to reach the paper's own stated real-world targets. Later work — Sampled MuZero for large/continuous action spaces, EfficientZero for sample efficiency — reads as a direct response to gaps this paper leaves open rather than as an independent line of research.

// takeaways

What to remember

  1. Planning doesn't require a simulator — it requires a model that predicts value, policy and reward accurately, which is a strictly easier target than reconstructing full observations. That reframing is MuZero's real contribution, not any single number.
  2. One algorithm, one set of hyperparameter choices inherited almost wholesale from AlphaZero, matches superhuman board-game play and sets a new Atari-57 state of the art (4999.2% mean / 2041.1% median), beating the best model-free method in 42/57 games and the best model-based method in all of them.
  3. The aggregate hides the failure modes: MuZero scores 0.0% normalized on Montezuma's Revenge and near-zero on Venture, Pitfall and Solaris — sparse-reward, hard-exploration games where a deterministic learned model and vanilla MCTS still don't help. Combined with a thousand-TPU self-play footprint and a dynamics function that's explicitly deterministic-only, that's the honest boundary of what this specific recipe solves.
