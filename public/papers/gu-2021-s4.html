Efficiently Modeling Long Sequences with Structured State Spaces

One state space model — run as a parallel convolution for training and a stateful recurrence for 60× faster generation — that handles 16,000-step sequences and solves Long Range Arena's Path-X for the first time, while staying as fast as the efficient Transformers it beats by 20+ points.

Albert Gu, Karan Goel, Christopher Ré · Stanford
venue ICLR 2022 citations 3,747 topic Reasoning source ↗

// the gap

What problem is this solving?

The hardest thing in sequence modeling is the long-range dependency (LRD): reasoning across thousands or tens of thousands of time steps. Every standard model family struggles. RNNs lose the signal to vanishing and exploding gradients; CNNs have a bounded receptive field; Transformers pay an O(L²) cost in attention that makes very long sequences infeasible. The Long Range Arena (LRA) benchmark made the failure concrete — on its hardest task, Path-X (a 128×128 image flattened to a length-16,384 sequence), no prior model beat random guessing.

A promising alternative had just appeared: model a sequence by simulating a continuous state space model (SSM), x′(t) = Ax(t) + Bu(t). With a special class of HiPPO matrices for A, the predecessor model (the Linear State Space Layer, LSSL) could capture LRDs in principle, and unified the strengths of continuous-time, recurrent, and convolutional views. The catch was cost: the LSSL needed O(N²L) compute and O(NL) memory for state size N and length L — orders of magnitude more memory than a comparable RNN — and its supposedly fast algorithms were numerically unstable because the HiPPO matrix is highly non-normal. So the idea was powerful on paper and unusable in practice.

S4's goal: keep the SSM's theoretical strengths but make it computationally tractable — fast to train, cheap to run, numerically stable — and general enough to serve as a single architecture across images, audio, text, and time series with minimal task-specific tweaking.

// how it works

The method

An SSM maps a 1-D input signal u(t) through an N-dimensional latent state x(t) to a 1-D output y(t). To use it on a discrete sequence, you discretize the continuous system with a step size Δ (the bilinear transform), which produces an RNN-style recurrence. Because the SSM is linear and time-invariant, you can also unroll that recurrence into a single global convolution y = K̄ ∗ u with an explicit kernel. These are the same operator viewed two ways — and that duality is the whole point: the convolution is parallel and ideal for training, while the recurrence is stateful and ideal for fast autoregressive generation.

x′(t) = A x(t) + B u(t),  y(t) = C x(t)
xk = A xk−1 + B uk,  yk = C xk
y = K ∗ u,  K = ( CB, CAB, …, CAL−1B )
// the continuous SSM (top) → discrete recurrence (middle) → equivalent convolution kernel (bottom)

The bottleneck lives in that kernel. Computing K̄ = (C̄B̄, C̄ĀB̄, …, C̄ĀL−1B̄) naively means taking L successive powers of the matrix Ā — the source of the LSSL's O(N²L) cost. The obvious fix is to diagonalize A so the powers become cheap. But the HiPPO matrix is non-normal, and its diagonalizing change-of-basis has entries that grow like 24N/3 — astronomically large, so naive diagonalization blows up numerically.

S4's contribution is a parameterization that sidesteps this. The key structural fact: every HiPPO matrix can be written as Normal Plus Low-Rank (NPLR) — a unitarily-diagonalizable (normal) part minus a rank-1 or rank-2 correction.

A = V Λ V* − P Q*
// normal (V Λ V*) plus low-rank (− P Q*); conjugating gives a diagonal-plus-low-rank system

From there, three numerical-analysis ideas turn an intractable computation into a near-linear one:

  • Generating function in frequency space. Instead of computing the kernel's coefficients directly, S4 evaluates its truncated generating function at the roots of unity. This converts repeated matrix powers into a single matrix inverse (a resolvent), which can then be inverse-FFT'd back to the kernel.
  • Woodbury identity. The low-rank term −PQ* is removed with the Woodbury matrix-inversion lemma, reducing everything to the purely diagonal case.
  • Cauchy kernel. The diagonal case is exactly a Cauchy matrix computation — a classic problem in numerical analysis with fast, stable algorithms (Fast Multipole Method).

The result: the convolution kernel reduces to just 4 Cauchy multiplies, giving Õ(N+L) compute and O(N+L) memory — essentially tight against the Ω(L+N) lower bound, and stable because it never touches the ill-conditioned diagonalization. A deep S4 layer is then just an SSM initialized to the HiPPO matrix (only 5N trainable parameters — Λ, P, Q, B, C), with H independent copies mixed by a position-wise linear layer and nonlinearities between layers. It is, in effect, a depthwise-separable CNN whose convolution kernels are global and learned through the SSM.

// what they found

The results

On the Long Range Arena, S4 averaged 86.09% across the six tasks versus 59.37% for the best of eleven efficient-Transformer baselines — every baseline scored under 60%. Most strikingly, S4 became the first model to solve Path-X (length 16,384), reaching 96.35% where every prior model sat at ~50% random. And it did this while staying competitive on speed and memory with the fastest efficient Transformers it was benchmarked against (Table 3 in the paper).

To see that S4 was genuinely modeling the full 16,384-pixel context rather than exploiting a shortcut, the authors visualized its learned convolution kernels on Path-X (Figure 2): early layers learn local filters, but the deepest layers learn kernels that span the entire sequence at varying spatial frequencies — direct evidence of learned long-range structure.

The efficiency win over the LSSL it descends from is dramatic. At hidden dimension 512, S4 trains 29.6× faster while using 392× less memory — and unlike the LSSL, it does so stably:

Beyond LRA, the same architecture — applied with minimal changes — held up across modalities:

  • Raw speech (Speech Commands SC10, length-16,000 waveforms): 98.32%, versus 71.66% for the best specialized conv baseline (CKConv) and a WaveGAN-D CNN with 90× more parameters at 96.25%. Transformers and RNNs simply fail (≥70% error) on the raw signal. As a continuous-time model, S4 also generalizes to 0.5× sampling frequency without retraining (96.30%), just by changing its step size Δ.
  • Sequential CIFAR-10 (pixels fed one at a time): 91.13%, vs 84.65% for the LSSL and 62.2% for a Transformer — and competitive with a larger 2-D ResNet-18 (91.12% vs 89.46% without augmentation; 7.9M vs 11.0M parameters), despite using no 2-D inductive bias.
  • WikiText-103 language modeling: swapping S4 in for self-attention in a strong Transformer reaches 20.95 test perplexity — within 0.8 ppl of the 20.51 Transformer, and SoTA among attention-free models by over 2 ppl — while generating 60× faster (48K vs 0.8K tokens/sec) thanks to the recurrent mode.
  • CIFAR-10 density estimation: 2.85 bits/dim with no 2-D bias, competitive with PixelSNAIL (2.85) and Sparse Transformer (2.80). Time-series forecasting: beats the specialized Informer on 40 of 50 settings, cutting MSE by 37% on the longest (30-day weather) horizon.
// does it hold up?

The eval read

S4 is a rigorous, honest paper — and the most telling evidence is its own ablation. The authors strip the model down on sequential CIFAR-10 and ask which piece actually buys the accuracy: the fancy NPLR parameterization, or the HiPPO initialization. The answer (Figure 3) is unambiguous — a random NPLR matrix performs no better than a random dense one, while HiPPO initialization dominates regardless of parameterization, with a generalization gap "over 15%" between initializations even when all reach perfect training accuracy. In the authors' words, "S4's effectiveness primarily comes from the HiPPO initialization, not the NPLR parameterization." That reframes the contribution precisely: NPLR is not a new source of representational power — it is the trick that makes the already-powerful HiPPO SSM computationally feasible and stable. Valuable, but worth stating plainly rather than letting the benchmark sweep imply a new modeling primitive.

Two caveats temper the headline numbers. First, the LRA comparison is not quite apples-to-apples: the paper itself appends "Please read Appendix D.5 before citing this table" to both its LRA and speech tables, acknowledging preprocessing and protocol differences — the eleven efficient-Transformer baselines were not re-tuned for these tasks the way S4 was, so part of the 86-vs-59 gap is a tuning gap, not purely architecture. Second, there is real version drift inside this very PDF: the running text reports an LRA average of "80.48%" and a Path-X score of "88%," while Table 4 — updated in a later revision — reports 86.09% and 96.35%. A reader citing "the S4 numbers" can land on different values depending on whether they read a sentence or the table.

baselines

Strong and broad (11 LRA efficient Transformers; specialized speech CNNs, RNNs, CTMs; a tuned Transformer LM). But the paper flags ("read Appendix D.5") that LRA/speech baselines weren't re-tuned per task — the gap is partly protocol, not pure architecture.

contamination

Not a leakage story — LRA, Path-X, sequential CIFAR are synthetic / curated. The flip side: solving a 16,384-length connect-the-dots puzzle is a clean LRD stress test, but doesn't directly certify real long-document language quality.

what's missed

On WikiText-103, S4 is still behind the Transformer (20.95 vs 20.51) — "competitive attention-free," not better. And a footnote concedes the released parameterization had a latent numerical instability (eigenvalues on the right half-plane), fixed only by follow-up work (Λ−PQ* → Λ−PP*).

would raise confidence

An equally-tuned LRA protocol across all models; end-to-end results on a real long-context NLP or genomics task rather than synthetic LRA; and shipping the numerically-stable parameterization as the default — which the S4D / DSS successors went on to do.

// bottom line

S4's real achievement is engineering a known-powerful-but-infeasible idea (HiPPO state spaces) into something fast, stable, and general — and being candid that the accuracy comes from the HiPPO initialization, while the NPLR algorithm is what makes that initialization usable at scale. The caveats are about benchmark realism and numerical robustness, not about whether the core claim holds. Solving Path-X was a genuine first, and S4 became the architectural seed for the entire SSM / Mamba line of work.

// takeaways

What to remember

  1. One model, two computational modes. An SSM is simultaneously a parallel convolution (train fast) and a stateful recurrence (generate 60× faster) — S4's parameterization is what lets you cheaply switch between them.
  2. The contribution is efficiency, not a new primitive. Decomposing the HiPPO matrix as Normal-Plus-Low-Rank reduces the SSM kernel to a stable Cauchy-kernel computation: from O(N²L) to Õ(N+L), 29.6× faster and 392× less memory than the LSSL. The ablation shows accuracy comes from the HiPPO initialization.
  3. It cracked the benchmark nobody could. First model to solve Long Range Arena Path-X (length 16,384) at 96.35% vs ~50% random, with a 86.09% LRA average against sub-60% baselines — and it generalized across raw audio, pixels, language, and time series.
