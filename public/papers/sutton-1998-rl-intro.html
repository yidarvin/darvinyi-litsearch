Reinforcement Learning: An Introduction

The 1998 textbook that gave reinforcement learning its shared formalism — Markov decision processes, value functions, the Bellman equation, temporal-difference learning — the common language every RL paper on this map writes in.

// the gap

What problem is this solving?

By the late 1980s, "reinforcement learning" was less a field than a set of disconnected dialects. Dynamic-programming theorists (Bellman, Puterman) had exact algorithms for planning in known Markov decision processes (MDPs) but no story about learning from experience. Animal-learning psychologists had a century of conditioning theory but no computational formalism. A small adaptive-control lineage — Barto, Sutton, and Anderson's 1983 actor–critic pole-balancer, Watkins' 1989 Q-learning, Tesauro's TD-Gammon — had scattered, striking demonstrations with no textbook connecting them. As Sutton and Barto put it in the first edition's preface: "reinforcement learning had been thoroughly explored in the early days of cybernetics and artificial intelligence… On closer inspection, though, we found that it had been explored only slightly." Researchers coming from AI, control theory, neuroscience, and operations research were, in effect, reinventing each other's ideas under different names.

The 1998 first edition's contribution was not a new algorithm — it was a unification: showing that dynamic programming, Monte Carlo methods, and temporal-difference (TD) learning are three ways of estimating the same value functions defined over the same MDP formalism, differing only in whether they use a model, wait for full returns, or bootstrap from their own estimates. "We can now place component ideas, such as temporal-difference learning, dynamic programming, and function approximation, within a coherent perspective with respect to the overall problem," the preface says. The second edition (2018, twenty years and one long rewrite later) kept that same goal but folded in tabular innovations from the interim (UCB, Expected Sarsa, double learning, tree-backup) and, in its second part, extended the whole framework to function approximation — the bridge to the deep-RL papers that dominate this citation graph.

// how it works

The method

Everything in the book sits on top of one loop: an agent observes a state St, picks an action At, and receives a reward Rt+1 and a next state St+1 from the environment, repeated over discrete time steps. This is Figure 3.1, reproduced below exactly as it appears in the text — the diagram that appears, in some visual variant, at the top of essentially every RL paper, course, and blog post since.

The book states the objective as the reward hypothesis: "That all of what we mean by goals and purposes can be well thought of as the maximization of the expected value of the cumulative sum of a received scalar signal (called reward)." Everything the agent does is in service of one number. A policy π(a|s) maps states to action probabilities; a value function vπ(s) estimates expected future return under that policy; an optional model predicts state/reward transitions for planning. The book is explicit that the policy alone is sufficient to determine behavior — value functions and models exist only to make finding a good policy tractable.

The load-bearing identity is the Bellman equation for vπ — the recursive consistency between a state's value and the values of its successors:

vπ(s) = Σa π(a|s) Σs′,r p(s′,r|s,a) [ r + γ vπ(s′) ] // Eq. 3.14 — the value of a state equals the expected immediate reward plus the discounted value of whatever comes next

Every algorithm in the book is a different way to estimate that equation without knowing the environment's dynamics p(s′,r|s,a). Monte Carlo methods substitute an averaged sample return for the expectation. Dynamic programming substitutes the current value estimate for vπ(s′) but still needs the model. Temporal-difference learning does both at once — it bootstraps from an estimate and samples the transition — which is the book's central synthesis. The simplest case, TD(0):

V(St) ← V(St) + α [ Rt+1 + γ V(St+1) − V(St) ] // Eq. 6.2 — update toward a target formed one step ahead, not at the end of an episode

The book's own running example, Gridworld, makes the Bellman equation concrete: a 5×5 grid where two cells (A, B) teleport the agent to fixed destinations for rewards of +10 and +5, and running off the edge costs −1. // what they found

Where the framework actually landed

A textbook doesn't report benchmark numbers, so the honest way to evaluate this one is to check whether its formalism actually became the field's shared vocabulary — and by extension, whether it's the direct theoretical ancestor of the deep-RL papers already on this map. The book's own case-studies chapter (16) narrates the lineage explicitly: Tesauro's TD-Gammon (TD(λ) self-play backgammon, 1992–1995) is presented as proof that TD learning plus function approximation could reach master-level play; then, in the second edition, DQN's 49-game Atari sweep ("DQN achieved levels of play at or beyond human level on a large fraction of these games," using "the same raw input, network architecture, and parameter values… for all the games") and AlphaGo ("combining deep ANNs, supervised learning, Monte Carlo tree search, and reinforcement learning," which "defeated the European Go champion Fan Hui 5 games to 0" before beating Lee Sedol 4 of 5) are folded in as the formalism's most visible deep-learning-era validations.

The three misses are informative rather than a shrug: TRPO's only Sutton citation runs through the 1983 actor–critic paper instead, and PPO/DDPG cite neither — a sign that the pure policy-gradient lineage in this graph runs more through Williams' REINFORCE and Sutton et al.'s 2000 policy-gradient-theorem paper (neither yet on the map; see below) than through this general-purpose introduction.

// does it hold up?

The eval read

Judging a 1998/2018 textbook by 2026 standards is a different exercise than judging a benchmark paper, but the question is the same in spirit: does the framework still describe what the field actually does, and where does it visibly stop? The core formalism — MDPs, the Bellman equation, TD learning as bootstrapped-and-sampled value estimation — hasn't just "held up," it's invisible infrastructure: every deep-RL paper above still writes state, action, reward, and a discounted return the exact way chapter 3 defines them, which is precisely what the citation check above demonstrates in the concrete.

Where it visibly stops is the policy-optimization machinery that came to dominate large-scale RL after 2015. I searched the full 2018 second-edition text for "trust region," "proximal policy," "PPO," "RLHF," and "human feedback" — none of the five appear anywhere in 548 pages, despite TRPO (2015) and PPO (2017) predating the book's 2018 publication date. The book covers the policy-gradient theorem and basic actor–critic methods (chapter 13) but stops well short of the specific machinery — trust regions, clipped surrogate objectives, generalized advantage estimation — that made policy gradients usable at the scale of Atari-and-beyond, let alone the RLHF/PPO/GRPO stack that now trains frontier LLMs. Chapter 16's case studies describe DQN and AlphaGo narratively, as finished results, rather than teaching experience replay or target networks as first-class algorithms alongside TD(0) and Q-learning.

baselines

Not applicable in the benchmark sense — the book's job was synthesis, not new algorithms. Nearly every core method (Q-learning, actor–critic, TD-Gammon) predates it by 5–15 years; its contribution is the shared formalism connecting them, which is exactly what the citation check above confirms propagated.

contamination

Not a meaningful axis for a 28-year-old textbook — there's no leakage risk, only staleness risk. The 2nd edition (2018) is a hard cutoff: nothing published after 2017 in its bibliography, confirmed by the total absence of TRPO/PPO/RLHF despite both TRPO and PPO existing by then.

what's missed

Trust-region and clipped-objective policy optimization (TRPO/PPO), generalized advantage estimation, distributional RL (C51/QR-DQN), offline/batch RL, multi-agent RL, and — most consequentially in hindsight — anything resembling RLHF or reward modeling from human preferences, the specific RL recipe now used to align LLMs.

would raise confidence

A third edition (none has been announced) folding in the policy-optimization and preference-learning wave the way the second edition folded in TD-Gammon, DQN, and AlphaGo — the book has a template for absorbing a generation of algorithms, it just hasn't done a second pass since 2018.

// bottom line

The book's actual bet — that a shared formalism would outlive any particular generation of algorithms — has been vindicated about as concretely as a citation graph can show: six of nine deep-RL papers already on this map cite this exact text, with DQN's Nature paper listing it as reference #1. But the 2018 edition is now a snapshot of RL circa 2017, frozen just before trust-region policy optimization, PPO, and RLHF became the field's dominant algorithmic story. Its formalism generalized far better than its algorithm catalogue did — which is arguably the correct way around for a book whose job was to outlast its own examples.

// takeaways

What to remember

  1. Unifies dynamic programming, Monte Carlo methods, and temporal-difference learning as three ways of estimating the same Bellman-equation value functions over the same MDP formalism — the vocabulary every subsequent RL paper writes in.
  2. Direct citation-list verification (not assumption) confirms the lineage: DQN (Nature, reference #1), Double DQN, A3C, SAC, TD3, and the Arcade Learning Environment paper all cite this exact 1998 text; PPO and DDPG cite neither Sutton nor Barto at all, and TRPO's only Sutton citation runs through the 1983 actor–critic paper instead.
  3. The book's own case-studies chapter narrates the exact TD-Gammon → Atari DQN → AlphaGo lineage this citation graph encodes structurally.
  4. Even the 2018 second edition has zero mentions of trust regions, PPO, or RLHF — the algorithmic wave that carried policy-gradient RL from Atari into LLM alignment sits entirely outside its scope.
