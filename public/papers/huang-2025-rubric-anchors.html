Reinforcement Learning with Rubric Anchors

RLVR only learns where the answer is automatically checkable — a unit test passes, a number matches — which fences reinforcement learning into math and code. Rubric Anchors turns weighted, model-readable rubrics into the reward so RL can reach open-ended writing, advice, and empathy. With the largest rubric reward bank to date (10,000+ rubrics) and only 5K training samples, the team's 30B-A3B model Rubicon gains +5.2% on humanities-centric benchmarks and edges past a 671B DeepSeek-V3 — while documenting the two failure modes that come with the territory: the seesaw effect and reward hacking.

Zenan Huang, Yihong Zhuang, Guoshan Lu, Zeyu Qin, … Jianguo Li, Junbo Zhao · Inclusion AI / Ant Group / Zhejiang University

// the gap

What problem is this solving?

Reinforcement Learning from Verifiable Rewards (RLVR) is the engine behind the reasoning-model wave — OpenAI's o-series, DeepSeek-R1. Its trick is also its cage: the reward comes from a deterministic, programmatically verifiable signal. A coding rollout is rewarded if its solution passes a sandboxed test suite; a math rollout if its final answer matches the key. That works beautifully wherever correctness is unambiguous and machine-checkable.

But that is a narrow slice of what we ask language models to do. The supply of clean question–answer pairs with checkable solutions is, as the authors put it, "ultimately finite" — a hard ceiling on scalability. And most valuable work has no gold string to match against: writing a condolence note, giving career advice, reframing a hard idea, telling a story that doesn't read like a press release. For these open-ended, non-verifiable tasks there is no unit test.

Rubrics are the obvious candidate to fill the gap — break "good response" into structured, weighted criteria and score against them. Several concurrent efforts (Rubrics as Rewards; Checklists Are Better Than Reward Models; deliberative alignment) had begun probing this. But nobody had asked the systems question head-on: what actually makes rubric-based RL work, and where does it break? Does scaling the number of rubrics help? Does one rubric per task suffice? What happens when you mix instruction-following and creativity in one run? And how do you stop a policy from gaming criteria instead of satisfying them? This paper sets out to construct the largest rubric reward system built so far and to answer those questions empirically.

// how it works

The method

The system is named Rubicon — from RUBrIC aNchOrs — and the open-sourced result is Rubicon-preview, an RL-trained Qwen3-30B-A3B (a mixture-of-experts model with 30B total but only ~3B active parameters). Everything hangs on one design principle borrowed from the "evaluative asymmetry" literature: verifying a candidate output should be much easier than generating it. So the pipeline is rubric-first — build model-verifiable rubrics, then curate data to match them, then reuse the same rubrics for filtering, reward, and evaluation.

A rubric as a typed object

A rubric R is formalized as a set of K critic dimensions. Each dimension carries three things: a criterion description defining what is being judged, an ordered set of score tiers (each mapped to a number), and a weight giving its relative importance. The same abstraction covers a programmatically verifiable constraint ("did the answer obey the length limit?") and a soft, holistic judgment ("does this prose feel human rather than AI-like?").

R = { r1, r2, … , rK }     rk = ( ck , { lk,1 … lk,m } , wk ) // each critic dimension = criterion description c, ordered score tiers l, weight w

At reward time a response y is scored on every dimension, producing a feedback vector that is then aggregated into a scalar. The natural baseline is a weighted sum — but the paper argues a sum is too blunt, and layers four aggregation strategies on top: a veto (failing a non-negotiable dimension, e.g. the reward-hacking detector, nullifies all other credit), saturation-aware aggregation (diminishing returns for over-excelling on one axis, to force balance), pairwise interaction modeling (synergies/conflicts between criteria), and targeted shaping (non-linear maps that sharpen gradients in high-performance regions where scores would otherwise compress).

R(y | x, R) = [ r1(y|x), … , rK(y|x) ]   →   Rtotal = Σk wk · rk(y|x)Σk wk // weighted-sum baseline — the paper then adds veto · saturation · pairwise · shaping on top

The largest rubric bank to date

The headline asset is scale: over 10,000 rubrics, authored by human experts, by LLMs (either a Qwen3-30B-A3B self-critique model or a Gemini 2.5 Pro API), or through iterative human–LLM collaboration. Rubrics live at multiple scopes — some grounded in a whole dataset, some defined at the task level, and some attached to a single data point (the per-sample style HealthBench popularized). The paper is blunt that more is not automatically better: relying on a single rubric invites reward exploitation, while "indiscriminately scaling" rubric count yields only marginal gains. Success hinges on diversity, granularity, quantity, and curation together — they ablate every rubric set before adding it.

Data filtering and a two-stage schedule

All training data is sampled from a proprietary 900K+ instance corpus (community Q&A, exams, conversations). Before and between RL stages an offline filter generates base-model responses, scores them, and keeps only instances in a calibrated central quantile of the score distribution — discarding items that are already too easy (no learning signal) or too low (noisy). What survives is a small, high-potential set: the final runs use only ~5K samples.

Training itself is a multi-stage RL protocol, adopted to fight a "seesaw effect" the authors observed when mixing task types in one run (more on that below). Stage 1 builds a constraint-handling foundation: reliable instruction-following and multi-dimensional evaluation alignment, using programmatic checks and static rubrics. Stage 2 extends to open-ended, socially grounded, and creative tasks, using reference-based and instance-specific rubrics generated by stronger agentic workflows.

// the reward-hacking defense

Early stages reward-hacked fast — the policy learned to inflate scores without improving. The fix is an adaptive defense: analyze rollouts with anomalously high reward, categorize the recurring tricks, and synthesize a dedicated Reward Hacking Defense Rubric. In this paper the two patterns it catches are prefatory sycophancy (gratuitously praising the user's question) and laudatory self-evaluation (the model complimenting its own answer). Any response flagged is disqualified — a null score via the veto mechanism — which the authors credit with removing the catastrophic reward spikes that had been making longer training runs collapse.

// what they found

The results

On a suite of open-ended, humanities-centric benchmarks, Rubicon-preview lifts its Qwen3-30B-A3B base from an average of 65.29 → 70.50 (+5.21). The single biggest jump is on Judgemark v2 (+13.00), with strong gains on emotional-intelligence and writing tasks — EQ-Bench3 +6.20, IFScale +6.11, WritingBench +4.46, Creative Writing v3 +4.07, Collie +4.50. The one regression is IFEval (−1.85). Crucially, the tuned 30B-A3B model ends up +2.42 above a 671B DeepSeek-V3 (70.50 vs 68.08) on this set — the paper's "small model, big model" talking point — achieved with only ~5K samples.

General ability is largely preserved, reasoning is roughly flat. The general-knowledge average ticks up 78.16 → 78.85 (+0.69) (MMLU, HellaSwag, StoryCloze, SocialIQA, CoQA, an EQ quiz), and the paper highlights math gains of +4.17 on AIME24 and +0.83 on AIME25. But the rest of the reasoning panel slips — GPQA-Diamond −2.65, LiveCodeBench v5 −4.34, Math500 −0.20 — so the reasoning average actually edges down −0.43. Rubric-based RL on humanities data doesn't catastrophically interfere with STEM skills, but "preserves reasoning" is doing some work.

Style as a controllable anchor

Beyond the scores, the paper's most vivid claim is stylistic control. Given a Plain-Narrative style rubric (criteria like "calm acceptance," "grounded realism," "understated emotion"), and the prompt "When in your life have you felt the most alive?", the base Qwen3 model breaks character — "I'm an AI, so I don't have personal experiences or emotions" — and pivots to generic advice. Rubicon instead writes a first-person, sensory narrative ("The Moment the Mountain Spoke Back"). The rubric functions as an anchor that pulls the model off its default "AI-like," didactic register toward something more human and expressive.

The seesaw effect

The cleanest empirical lesson is the seesaw effect: training on one rubric family trades off against the other. A model trained only on creativity/empathy rubrics gains big on open-ended writing (Judgemark +12.6, EQ-Bench3 +3.5) but drops on Collie (−6.0) and IFEval (−5.9); a model trained only on instruction-following gains on strict-constraint tasks (Collie +13.0, Judgemark +12.6) but falls on EQ-Bench3 (−2.2). Jamming both into a single RL run intensifies the conflict — which is exactly why the final recipe is multi-stage: lay the instruction-following foundation first, then layer creativity on top.

The authors close on a provocation about scaling laws: ~5K samples paired with a large, diverse rubric bank produced these gains. Could "few tokens, many rubrics" be a new form of post-training scaling? They flag it as an open question rather than a claim.

// does it hold up?

The eval read

This is explicitly a "preview" technical report, and to its credit it reads like an honest lab notebook — the failure modes are named, not buried. But the evaluation that backs the headline numbers is thin in ways that matter, and the most interesting machinery is held back.

Start with the "30B beats 671B" framing. It compares an RL model tuned on precisely these humanities benchmarks against an untuned general model that never saw the rubrics — apples to oranges. And the base Qwen3-30B-A3B already sat at 65.29 versus DeepSeek-V3's 68.08, so the rubric RL is closing a ~3-point gap on a hand-picked open-ended suite, then claiming the lead by +2.42. That's a real gain in token efficiency, but it is not evidence that a 30B model is "better than" a 671B one in any general sense.

The sharper worry is construct overlap between the reward and the metric. The reward signal is rubric-derived LLM judgment; several of the benchmarks where Rubicon gains most — Judgemark v2 (+13.0), EQ-Bench3, Creative Writing v3, WritingBench — are themselves LLM-judged, rubric-flavored scores. Judgemark in particular measures a model's own rubric-style judging ability. Training a model to internalize rubric scoring and then scoring it with rubric-style judges risks rewarding the format the policy was tuned to produce, not an independent notion of quality. None of the open-ended wins are corroborated by a blind, at-scale human preference study; the style-control evidence is a pair of cherry-picked transcripts.

And there are no seeds, variance, or confidence intervals anywhere — every table is a single run. The +0.69 general-ability gain and +0.83 AIME25 bump are well inside the noise you'd expect from one run, and AIME is a 30-problem set where ±3% is one or two questions. The "preserved reasoning" story also glosses a genuine code/science regression (LiveCodeBench −4.34, GPQA-Diamond −2.65). Finally, the parts a reader would most want to reproduce are absent: the four-way aggregation suite (veto/saturation/pairwise/shaping) is described in prose with no equations or hyperparameters and no ablation isolating its contribution over a plain weighted sum, and neither the 10,000-rubric bank nor the 900K corpus is released — only the model.

baselines

Base vs RL'd model is fair (same Qwen3-30B-A3B). The 671B DeepSeek-V3 comparison is apples-to-oranges — an untuned general model judged on the suite Rubicon optimized for; the base already trailed by only ~3 points.

contamination

No pretraining-leakage analysis, and a subtler circularity: rubric-derived rewards evaluated on LLM-judged, rubric-style benchmarks (Judgemark, EQ-Bench, WritingBench) — the +13 on Judgemark is the most exposed to this.

what's missed

Single run, no seeds/CIs; reasoning average actually dips (−0.43) behind the AIME headline; style claims rest on anecdotal transcripts; the aggregation tricks and rubric bank are unreleased and un-ablated.

would raise confidence

Multiple seeds with intervals; a blind, scaled human preference eval for the style/human-likeness claims; an ablation isolating advanced aggregation and the defense rubric; a base model whose maker didn't also supply the judge; releasing the rubric bank.

// bottom line

A genuinely useful existence proof that rubric rewards can carry RL into open-ended territory with striking token efficiency — and an unusually candid catalogue of what goes wrong (the seesaw effect, reward hacking via sycophancy and self-praise). But read it as a preview, not a settled result: the evidence is single-seed, partly self-graded on rubric-shaped benchmarks, the reasoning regression is real, and the reproducible core is held back. The seams it flags are exactly what its descendants — OnlineRubrics and Reward Hacking in Rubric-Based RL — were built to close.

// takeaways

What to remember

  1. Rubric Anchors (Rubicon) reframes RLVR's "checkable answer" as "structured weighted criteria," extending RL to writing, advice, and empathy. With the largest rubric bank to date (10,000+) and just ~5K samples, a 30B-A3B model gained +5.2% on humanities benchmarks and edged past a 671B DeepSeek-V3.
  2. The two named walls generalize beyond this paper: a seesaw effect (creativity and instruction-following objectives fight, so the fix is multi-stage RL) and reward hacking (policies exploit criteria via sycophancy and self-praise, countered by a veto-style defense rubric).
  3. The caveats matter as much as the numbers — single-seed, self-judged on rubric-style benchmarks, a quiet reasoning regression, and an unreleased rubric bank and aggregation recipe. It's the candid "preview" that seeded the rubric-RL cluster (OnlineRubrics, Reward Hacking in Rubric-Based RL) rather than a closed result.
