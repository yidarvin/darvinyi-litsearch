DeepSeekMath: Pushing the Limits of Mathematical Reasoning in Open Language Models

A 7B model that scored 51.7% on competition-level MATH — without tools or voting, the first open model past 50%. But the lasting contribution is GRPO: a critic-free variant of PPO that drops the value model and estimates the advantage baseline from a group of sampled answers, halving the memory of RL fine-tuning. It is now the algorithm under most of the RLVR / rubric-reward stack — and the paper quietly shows that here RL mostly sharpens the output distribution (lifts Maj@K, not Pass@K).

Zhihong Shao, Peiyi Wang, Qihao Zhu, Runxin Xu, Junxiao Song, Xiao Bi, Haowei Zhang, Mingchuan Zhang, Y.K. Li, Y. Wu, Daya Guo · DeepSeek-AI

// the gap

What problem is this solving?

By early 2024 frontier closed models — GPT-4, Gemini-Ultra — could do real competition mathematics, but they were unavailable and unreproducible, and the open-source models trailed badly. The paper attacks two bottlenecks at once: a data problem and an RL problem.

The data bottleneck. Strong math models had been trained on small, hand-curated math corpora — Minerva's web math, OpenWebMath (13.6B tokens), Proof-Pile-2 (heavy on arXiv). The implicit assumption was that the open web is too noisy to be a primary math source. DeepSeekMath bets the opposite: that Common Crawl contains an enormous, untapped reservoir of high-quality mathematics if you can build a good enough filter to mine it.

The RL bottleneck. The standard recipe for RL fine-tuning of LLMs is PPO, an actor–critic algorithm. PPO trains a separate value model — typically the same size as the policy — to estimate a per-token baseline for variance reduction. In the LLM setting this is awkward and expensive: the value model roughly doubles the memory and compute of training, and it has to learn an accurate per-token value even though the reward model usually scores only the last token of a long solution. The question: can you keep PPO's stable, clipped policy-gradient updates while throwing the value model away?

// the two bets

(1) The open web, filtered well, beats curated corpora as a math pre-training source. (2) You don't need a learned critic to do LLM RL — a group of sampled answers to the same question is itself a baseline.

// how it works

The method

1 · Mining 120B math tokens from Common Crawl

The corpus is built by an iterative fastText bootstrap. Start with OpenWebMath as a seed; sample 500K positive pages from it and 500K random Common-Crawl pages as negatives; train a fastText classifier; rank the deduplicated 40B-page Common Crawl by the classifier and keep the top-scoring pages. Then expand the seed: group the whole crawl into domains (pages sharing a base URL), flag any domain where >10% of pages were already recalled as math-related (e.g. mathoverflow.net), have humans annotate the math-bearing URL paths within them, and feed the newly-found pages back to retrain a better classifier. After four iterations — at which point ~98% of the data was already found by iteration three — the result is the DeepSeekMath Corpus: 35.5M web pages, 120B tokens, roughly 7× the math web data behind Minerva and 9× OpenWebMath. To guard the benchmarks, any page containing a 10-gram (or, for short texts, a ≥3-gram exact) match to GSM8K / MATH / CMATH / AGIEval is filtered out.

The 7B base model (DeepSeekMath-Base) is continual-trained from DeepSeek-Coder-Base-v1.5 7B — starting from a code model, not a general LLM, was deliberately better — over 500B tokens, of which 120B are this math corpus, the rest natural-language and code. A small-scale ablation (DeepSeek-LLM 1.3B, 150B tokens each) shows the corpus quality is not just a size story: trained on it, the 1.3B model reaches the curves below far faster and higher than on MathPile, OpenWebMath, or Proof-Pile-2.

2 · GRPO: drop the critic, group the baseline

PPO maximizes a clipped surrogate objective in which each token's update is scaled by an advantage At computed by Generalized Advantage Estimation from a learned value function Vψ. GRPO removes Vψ entirely. For each question q it samples a group of G outputs {o1…oG} from the old policy, scores them with the reward model to get {r1…rG}, and uses the group's own mean and standard deviation as the baseline — a much cheaper, naturally comparative substitute for a critic.

For outcome supervision, every token in output oi gets the same normalized advantage:

Âi,t = ri − mean(r)std(r) // the group of G rewards is its own baseline — no value model needed

That advantage plugs into a PPO-style clipped objective, but with the KL term pulled out of the reward and added directly to the loss against a frozen reference model:

JGRPO(θ) = 𝔼 1G Σi 1|oi| Σt [ min( ρi,t Âi,t, clip(ρi,t, 1−ε, 1+ε) Âi,t ) − β·DKL(πθ‖πref) ] // Eq. 3 — ρi,t = πθ(oi,t|·)/πθ_old(oi,t|·) is the importance ratio; KL is regularized in the loss, not the reward

The KL itself uses an unbiased, guaranteed-positive estimator (Schulman, 2020) rather than the usual log-ratio. GRPO also supports process supervision (normalize a reward at the end of each reasoning step, and set each token's advantage to the sum of normalized step-rewards that follow it) and iterative RL (periodically regenerate the reward model's training data from the current policy and reset the reference model). DeepSeekMath-RL trains on ~144K CoT questions drawn only from GSM8K and MATH, sampling 64 outputs per question, KL coefficient 0.04, policy learning rate 1e-6.

3 · A unified view of SFT, RFT, DPO, PPO, GRPO

The paper then frames all of these as one template — data source × reward function × gradient coefficient — and shows the design axes empirically: online sampling (data from the live policy) beats offline (RFT, DPO), a model reward with magnitude-sensitive gradient coefficients (GRPO) beats a binary rule reward that reinforces all correct answers equally (Online RFT), and process- or iteration-aware variants add a little more. This taxonomy, more than the model, is what later RLVR work builds on.

// what they found

The results

The base model alone is the headline efficiency result: DeepSeekMath-Base 7B reaches 64.2% on GSM8K and 36.2% on MATH (CoT), competitive with the closed Minerva 540B — a ~77× smaller model matching it on quantitative reasoning, the proof that data quality, not parameter count, was the binding constraint. Math training also raised general reasoning (MMLU, BBH) rather than trading it away.

Instruction tuning then RL stack on top. DeepSeekMath-Instruct 7B hits 82.9% / 46.8% (GSM8K / MATH, CoT); DeepSeekMath-RL 7B pushes that to 88.2% / 51.7% — the first open model past 50% on MATH without tools, beating every 7B–70B open model and most closed ones, and reaching 60.9% on MATH with self-consistency over 64 samples. Crucially, RL used only GSM8K/MATH CoT data yet improved every benchmark, including out-of-domain Chinese ones (MGSM-zh 73.2→79.6, CMATH 84.6→88.8).

// does it hold up?

The eval read

This paper is unusually honest about its own limits, and the most important caveat is one the authors put in plain text. In Section 5.2.2 ("Why RL Works?") they evaluate Pass@K and Maj@K for the Instruct and RL models and find that RL improves Maj@K but not Pass@K. Read carefully, that means GRPO did not expand what the model can do: for large K the RL model and the SFT model solve the same set of problems. What RL did was re-weight the output distribution so the correct answer is more likely to be the top sample — "boosting the correct response from TopK rather than the enhancement of fundamental capabilities," in their words. The flashy 82.9→88.2 / 46.8→51.7 gains are, on this reading, sharpening, not new capability. This single figure became the opening question of the 2025 RLVR debate (Yue et al., "Does RL Really Incentivize Reasoning Beyond the Base Model?").

Baselines. The corpus comparison (MathPile / OpenWebMath / Proof-Pile-2) is methodologically tidy — same 1.3B model, same 150B-token budget — but it conflates quality with scale: the full DeepSeekMath Corpus is 120B tokens against OpenWebMath's 13.6B, so "ours wins" is partly "ours is 9× bigger," and the head-to-head is run at 1.3B, not the 7B that ships. The RL baselines (RFT, Online RFT, DPO, PPO) are the right comparisons and are likewise run on small/instruct proxies rather than the headline 7B.

Contamination. Mining 120B tokens straight from Common Crawl is exactly the regime where benchmark leakage is hardest to rule out. The decontamination is reasonable — 10-gram (and ≥3-gram exact) filtering against GSM8K/MATH/CMATH/AGIEval — but it is string-level, so paraphrased or reformatted problems can slip through, and the paper reports no post-hoc overlap audit or a held-out, freshly-collected test set to bound residual contamination. Given how large the GSM8K/MATH jumps are, an independent contamination probe would have meaningfully raised confidence.

What the headline misses. Beyond the Maj@K/Pass@K point: the model is narrow. The authors note it is weak on geometry and theorem proving ("cannot handle problems related to triangles and ellipses"), and — unlike GPT-4 — shows essentially no few-shot gain over zero-shot, a sign of limited in-context flexibility at 7B. RL was trained only on GSM8K/MATH CoT, so its out-of-domain lift, while real, is modest and may itself be distributional sharpening. None of this is hidden; it just means "approaches GPT-4 on MATH" is a statement about one metric on one benchmark family.

baselines

Right comparisons (rival math corpora; RFT/DPO/PPO/Online-RFT), but the corpus win conflates quality with 9× scale, and the key ablations run at 1.3B, not the shipped 7B.

contamination

10-gram + short-text exact-match filtering against the four benchmarks — sound but string-level, with no post-hoc overlap audit or fresh held-out math set to bound leakage from web-scale CC.

what's missed

By the paper's own Figure 7, RL lifts Maj@K but not Pass@K — sharpening, not new capability. Plus weak geometry/theorem-proving and no few-shot gain; RL trained only on GSM8K/MATH.

would raise confidence

Repeat the corpus ablation at 7B and token-matched; an independent contamination probe / post-2024 held-out benchmark; report Pass@K alongside every RL headline; and (later work did this) test GRPO outside math.

// bottom line

The model is now a footnote; GRPO is the legacy. Dropping the critic for a group-normalized baseline made LLM RL cheap and stable enough to become the default, and it now sits under most of the RLVR and rubric-reward stack (OnlineRubrics, POW3R, rubric reward-hacking, RLCF all build on it). The math results were strong and largely honest — and the paper's own Maj@K-not-Pass@K finding is one of the most consequential single figures in the post-training literature, because it framed the question of whether RL teaches anything new at all.

// takeaways

What to remember

  1. GRPO = PPO without the critic. Sample a group of answers to each question and use the group's mean/std as the advantage baseline. It roughly halves RL memory and became the default algorithm for RLVR and rubric-reward training.
  2. The open web is a math goldmine. An iterative fastText bootstrap mined 120B math tokens (35.5M pages) from Common Crawl — letting a 7B base model match Minerva 540B (64.2% GSM8K, 36.2% MATH) and a 7B RL model clear 50% on MATH (51.7%) for the first time in open source.
  3. Here, RL sharpened rather than taught. By the paper's own Figure 7, RL raised Maj@K but not Pass@K — it reranked answers already in the distribution, not the capability ceiling. That observation seeded the whole "does RL incentivize reasoning beyond the base model?" debate.
