ALBERT: A Lite BERT for Self-supervised Learning of Language Representations

Two parameter-reduction tricks — a factorized embedding and weights shared across every layer — cut BERT-large's parameter count by 18×. ALBERT then spends the freed budget on width, setting new state of the art on GLUE, SQuAD, and RACE. The fine print: "lite" counts parameters, not compute.

Zhenzhong Lan, Mingda Chen, Sebastian Goodman, Kevin Gimpel, Piyush Sharma, Radu Soricut · Google Research & TTI-Chicago

// the gap

What problem is this solving?

By late 2019 the recipe for natural-language understanding was blunt: scale the encoder. BERT had shown that across NLU tasks, more hidden units, more layers, and more attention heads kept helping — but it stopped at a hidden size of 1024. The obvious next move, just keep growing, runs into two walls that have nothing to do with whether bigger is smarter:

  • Memory. State-of-the-art models already had hundreds of millions of parameters; widening the network quickly exhausts GPU/TPU memory. Most of those parameters sit in the vocabulary embedding matrix (size V × H), which is updated only sparsely.
  • Communication. In distributed training, the per-step communication overhead is roughly proportional to the parameter count, so more parameters slow every worker down. Existing fixes — model parallelism, gradient checkpointing, reversible layers — buy memory back but pay for it in speed, and don't touch the communication bill at all.

Worse, naïve scaling can backfire. ALBERT's own ablations show that simply widening the model past a point hurts: a 3-layer ALBERT-large keeps improving as the hidden size grows to 4096 (avg 76.3), then declines to 74.0 at hidden size 6144 despite having half a billion parameters. So "is better NLP as easy as having larger models?" is a real, open question, not a rhetorical one.

There was also a second-order problem hiding in BERT's training objective. Alongside masked language modeling (MLM), BERT trained on next-sentence prediction (NSP) — a binary "do these two segments come from the same document?" task. RoBERTa and XLNet had already found NSP unreliable and dropped it. Lan et al. diagnose why: NSP's negatives are drawn from a different document, so the task collapses into easy topic detection, which MLM already learns. The signal that actually matters for multi-sentence reasoning — coherence — was never being trained.

// how it works

The method

ALBERT keeps BERT's Transformer-encoder backbone (GELU activations, feed-forward size 4H, H/64 heads) and changes three things. The first two cut parameters; the third changes what the model learns.

1 · Factorized embedding parameterization

In BERT the WordPiece embedding size E is tied to the hidden size H (E ≡ H). That conflates two different jobs: embeddings should hold context-independent token meaning, while hidden states hold context-dependent meaning — and the power of BERT comes from the latter, which argues for H ≫ E. Tying them also means the embedding matrix grows with H, so widening the model balloons a matrix that is mostly updated sparsely. ALBERT unties them: project one-hot tokens first into a small space of size E = 128, then up into H. With vocabulary V = 30,000, the embedding cost drops from

O(V × H)  ⟶  O(V × E + E × H) // e.g. at V=30k, H=4096: 30k·4096 ≈ 123M params collapses to 30k·128 + 128·4096 ≈ 4.4M

2 · Cross-layer parameter sharing

Instead of giving each of the L Transformer layers its own weights, ALBERT shares all parameters — attention and feed-forward — across every layer. The model still runs L layers of computation; it just reuses one set of weights. This decouples parameter count from depth entirely. It also acts as a regularizer: measuring the input/output embedding of each layer, ALBERT's layer-to-layer transitions are far smoother than BERT's, and — unlike the Deep Equilibrium Model conjecture — the embeddings oscillate rather than converge to a fixed point, even after 24 layers.

The two configurations that result: an ALBERT-large matches BERT-large's shape (24 layers, H=1024) with 18M parameters vs 334M — 18× fewer, while the headline ALBERT-xxlarge goes much wider (H=4096) yet still lands at 235M, about 70% of BERT-large.

3 · Sentence-order prediction (SOP)

ALBERT replaces NSP with a harder inter-sentence loss. Positives are two consecutive segments from the same document (as in BERT); negatives are the same two segments with their order swapped. Because both segments come from the same place, topic is no longer a giveaway — the model must reason about discourse coherence. As the ablation below shows, NSP cannot solve SOP (it sits at chance), but a model trained on SOP solves NSP reasonably well, confirming SOP subsumes the easier task.

Everything else follows BERT's setup: pretraining on BookCorpus + English Wikipedia (~16GB), a 30k SentencePiece vocabulary, n-gram masking up to trigrams, batch size 4096, the LAMB optimizer at learning rate 0.00176, and 125k steps by default on 64–512 Cloud TPU v3 chips.

// what they found

The results

Held to the same data and 125k steps, ALBERT-xxlarge beats BERT-large with ~30% fewer parameters: development-set gains of +1.9 on SQuAD v1.1, +3.1 on SQuAD v2.0, +1.4 on MNLI, +2.2 on SST-2, and a striking +8.4 on RACE, lifting the average from 85.2 to 88.7. The parameter savings are real and the accuracy is real — but note the speedup column: an ALBERT-large iterates ~1.7× faster than BERT-large, while the xxlarge model that wins is ~3× slower (it iterates at 0.3× BERT-large's speed) because it is so wide.

The SOP loss earns its place. Trained head-to-head at ALBERT-base scale, an NSP-trained model scores just 52.0% on the SOP task — essentially the 53.3% random baseline — proving NSP learns only topic shift. An SOP-trained model scores 86.5% on SOP and still 78.9% on NSP, and improves downstream tasks (+~1% SQuAD1.1, +~2% SQuAD2.0, +1.7 RACE; +1 on average).

An incidental but influential finding: ALBERT's largest models never overfit even after 1M steps, so the authors removed dropout entirely — and MLM accuracy rose (Figure 2b). They claim to be the first to show dropout can hurt large Transformers. Adding the extra data used by RoBERTa/XLNet also helps overall (avg 80.1 → 80.8, Figure 2a), though Wikipedia-based SQuAD slightly regresses on out-of-domain text.

Putting it together — ALBERT-xxlarge with MLM+SOP, no dropout, extra data, and 1.5M steps — sets state of the art on all three leaderboards: a GLUE score of 89.4, a SQuAD v2.0 test F1 of 92.2, and a RACE test accuracy of 89.4 (single-model RACE is 86.5). The RACE jump is the headline: from the task's original 44.1% in 2017 to ALBERT's 89.4%. A controlled wall-clock comparison is the cleanest evidence — given roughly the same ~32–34 hours of training, ALBERT-xxlarge beats BERT-large by +1.5 on average and +5.2 on RACE.

// does it hold up?

The eval read

ALBERT's results are solid and the SOP diagnosis is genuinely clever, but the framing rewards a close read. The central thing to keep straight is that "A Lite BERT" is about parameter storage, not compute. Cross-layer sharing collapses the parameter count and the memory/communication bills, but it does not reduce FLOPs — every shared layer still does the full forward and backward pass. So the model that delivers state of the art, ALBERT-xxlarge, has fewer parameters than BERT-large yet is ~3× slower to train and proportionally more expensive at inference because it is so wide (H=4096). The crowd-pleasing "18× fewer parameters, 1.7× faster" belongs to ALBERT-large (H=1024), a different and weaker model than the one in the SOTA tables. Reported together, they imply a free lunch the architecture doesn't serve.

The leaderboard numbers also lean on machinery the comparison rows don't always advertise. The headline 89.4 GLUE and 89.4 RACE are ensembles that average 6–17 fine-tuned checkpoints per task, selected on the dev set; the single-model RACE is 86.5. And the much-quoted "+45.3% on RACE" is measured from the 2017 task paper's 44.1% baseline — it bundles in all the progress made by BERT, XLNet, and RoBERTa. ALBERT's own contribution over the strongest prior single model (RoBERTa, 83.2 on RACE) is real but far smaller.

Finally, several wins are confounded or thin. ALBERT-xxlarge improves on BERT-large while changing many variables at once — width, the SOP loss, extra data, removed dropout, and far longer training — so the clean "fewer params + better" claim rests mainly on the controlled Table 2 / wall-clock comparisons, where the gain is solidly positive but achieved by a slower model. The SOP ablation, persuasive as it is, runs only at ALBERT-base scale with a ~+1 average downstream gain, and the dropout-hurts-Transformers claim is a +0.3 average effect shown on a single model family that the authors themselves flag as needing replication.

baselines

Strong and current for 2019 — BERT, XLNet, RoBERTa, with a controlled same-data and same-wall-clock comparison to BERT-large. But the SOTA tables mix single-model and 6–17-checkpoint ensembles, and the "1.7× faster" claim quietly switches to a different (smaller) ALBERT than the one that wins.

contamination

Not a 2019 pretraining concern, but dev-set early stopping, median-over-5-runs, and selecting ensemble checkpoints on the dev set are mild forms of adapting to public leaderboards.

what's missed

Parameter count is reported as if it were cost. No FLOPs or inference-latency numbers accompany the "lite" claim, yet the winning model is wider and slower than the baseline it beats. Param-efficiency ≠ compute-efficiency.

would raise confidence

Report wall-clock/FLOPs/latency beside every parameter count; hold width and data fixed in the SOTA comparison; scale the SOP ablation beyond base; and test the dropout-removal finding on non-ALBERT Transformers.

// bottom line

ALBERT is two clean, durable ideas — factorized embeddings and cross-layer sharing — plus a smart fix for a broken pretraining objective. Read it as a parameter-efficiency and regularization result, not a speed result: it makes encoders dramatically cheaper to store and lets you scale width without the embedding matrix exploding, but the model that tops the leaderboards pays for its accuracy in compute.

// takeaways

What to remember

  1. Two parameter cuts, not compute cuts. Factorizing the embedding (E≪H) and sharing all layers' weights shrink BERT-large from 334M to 18M parameters — saving memory and communication, but not FLOPs.
  2. Spend the savings on width. The headline ALBERT-xxlarge (H=4096, 235M params) tops GLUE (89.4), SQuAD v2.0 (92.2 F1), and RACE (89.4) — with ~70% of BERT-large's parameters but ~3× the training cost.
  3. SOP fixes NSP. Predicting the order of two same-document segments forces coherence learning that BERT's next-sentence task never did; NSP scores at chance on SOP, while SOP transfers to multi-sentence tasks.
