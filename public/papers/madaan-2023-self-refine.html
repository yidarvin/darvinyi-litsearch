Self-Refine: Iterative Refinement with Self-Feedback

A single frozen LLM plays generator, critic, and editor on itself — no extra model, no training data — for 5–40% absolute gains across seven generation tasks. The one place it barely moves the needle, math reasoning, turns out to be the most honest result in the paper: the headline curve for it depends on an oracle the method doesn't actually have.

Aman Madaan, Niket Tandon, Prakhar Gupta, Skyler Hallinan, Luyu Gao, Sarah Wiegreffe, Uri Alon, Nouha Dziri, Shrimai Prabhumoye, Yiming Yang, Shashank Gupta, Bodhisattwa Prasad Majumder, Katherine Hermann, Sean Welleck, Amir Yazdanbakhsh, Peter Clark

// the gap

What problem is this solving?

Ask an LLM to draft an email, optimize a function, or write a dialogue turn, and the first pass is rarely the best one — people revise their own writing, and there's no reason to expect a language model's zero-shot output to be an exception. The trouble is that every existing recipe for teaching a model to revise itself bakes in extra machinery. Learned refiners like PEER (Schick et al., 2022) or Self-Correct (Welleck et al., 2022) need a second model trained on domain-specific ⟨input, output, feedback⟩ triples. RLHF-style approaches (Stiennon et al.'s summarization work, Ouyang et al.'s InstructGPT) fold a reward signal into the weights via reinforcement learning, which means collecting large human-preference datasets and running an expensive training loop. Both routes tie the fix to one task and one round of costly supervision.

SELF-REFINE asks whether any of that machinery is actually load-bearing — or whether a single off-the-shelf LLM, prompted three different ways with nothing but a handful of few-shot examples, can generate an output, critique it, and rewrite it, using only the knowledge it already has. No new training data, no separate critic network, no gradient step.

// how it works

The method

SELF-REFINE is a closed loop around one frozen model M, alternating between two prompted roles applied to the same model: FEEDBACK, which asks M to critique its own most recent output, and REFINE, which asks the same M to rewrite that output given the critique. There is no separate generator, critic, or reward model — one LLM, three hats, driven entirely by few-shot prompting (in the style of Brown et al., 2020).

Concretely, the initial output is y₀ = M(pgen‖x), then feedback is fbt = M(pfb‖x‖yt), and refinement folds the entire history of drafts and feedback back into the prompt rather than just the last step:

yt+1 = M ( prefine ‖ x ‖ y0 ‖ fb0 ‖ … ‖ yt ‖ fbt ) // each refinement conditions on the input, every prior draft, and every prior piece of feedback — not just the most recent one

The authors are explicit that feedback quality is the mechanism: it must be actionable (contains a concrete fix) and specific (points at a concrete phrase or line to change) rather than generic praise or vague criticism — "This code is slow as it uses a for loop which is brute force. A better approach is to use the formula (n(n+1))/2" is the kind of feedback the prompts are built to elicit, not "improve the efficiency of the code."

The loop runs for up to 4 iterations per instance, with greedy decoding at temperature 0.7, and stops either at a fixed iteration budget or when the model's own feedback signals it is satisfied — the stop condition is task-specific and is itself generated by the same prompted model. SELF-REFINE is evaluated across three base LLMs — GPT-3.5 (text-davinci-003), ChatGPT (gpt-3.5-turbo), and GPT-4 — plus Codex (code-davinci-002) for the code tasks, on seven tasks spanning dialogue response generation, code optimization, code readability improvement, math reasoning (GSM8k), sentiment reversal, and two new tasks the paper introduces: acronym generation and a harder 20–30-keyword constrained generation task.

// what they found

The results

The headline claim: SELF-REFINE improves over direct one-shot generation on every one of the seven tasks, by 5–40% absolute, averaging roughly 20% absolute across tasks and models. But the size of the gain varies enormously by task — from a rounding error in math reasoning to the single largest jump in the paper, dialogue response generation with GPT-4.

The gains hold up, roughly, across all three base LLMs — GPT-3.5, ChatGPT, and GPT-4 all improve on every task with SELF-REFINE applied, and stronger base models generally see larger absolute jumps even where their un-refined starting point was already competitive. The preference-based tasks (dialogue, sentiment, acronym generation) lean on a GPT-4-based preference proxy that the authors report correlates with blind human A/B judgments at 82% (sentiment reversal), 68% (acronym generation), and 71% (dialogue response). Code Optimization and Constrained Generation use automatic task metrics (% programs optimized, keyword coverage), and Math Reasoning uses exact-match solve rate on GSM8k.

Math reasoning is the outlier the authors themselves flag: the modest gain traces to the model's own feedback frequently declaring victory prematurely — for ChatGPT, the feedback text is "everything looks good" on 94% of instances, even when the underlying answer is wrong. A qualitative failure analysis of 70 hand-reviewed cases in code optimization and math reasoning found that when SELF-REFINE failed to improve an output, 61% of the time the feedback suggested an inappropriate fix and 33% of the time it mis-located the error — only 6% of failures were the refiner mangling otherwise-correct feedback. In other words, most of SELF-REFINE's failures are feedback failures, not refinement failures.

A feedback-quality ablation (Table 2 in the paper) confirms the mechanism is doing real work: swapping actionable feedback for generic feedback ("improve the code") drops Sentiment Reversal from 43.2 to 31.2, and removing feedback entirely collapses it to 0 — the task fails outright without a critique to act on.

// does it hold up?

The eval read

SELF-REFINE is a clean, influential idea and the paper is unusually forthcoming about its own weak spots — but three of those admissions, read together, change how much weight the headline "5–40% absolute, ~20% average" claim should carry.

The rosiest math-reasoning result is oracle-assisted. Table 1's true zero-oracle math numbers are essentially flat — GPT-3.5 doesn't move at all (64.1 → 64.1), and ChatGPT/GPT-4 gain a mere +0.2. But the paper also reports a much more encouraging GSM8k curve (Appendix H.1, Figure 14) climbing from 71.34% to 76.19% over four iterations. The catch, stated directly in the appendix text: for that experiment "we use the correct label to decide when to go from one point in the loop to the next" — the stopping condition is fed the ground-truth answer, which a deployed system would not have. It is a legitimate ablation for isolating how good REFINE is if FEEDBACK were perfectly reliable, but it is not the no-oracle SELF-REFINE result, and pairing that curve with the flat Table 1 numbers is the most informative comparison in the paper, not the most flattering one.

GPT-4 grades GPT-4. For Sentiment Reversal, Dialogue Response, Acronym Generation, and Code Readability — four of the seven tasks, and the tasks with the biggest reported gains — the headline metric in Table 1 is a GPT-4-based preference proxy ("GPT-4-pref"), not a task-intrinsic score. When the model under test is GPT-4 itself, that means GPT-4 is judging outputs from GPT-4 (base) versus GPT-4 (+Self-Refine). The paper reports the proxy correlates with blind human judgment at only 68–82%, meaning up to roughly a third of individual preferences disagree with a human rater — and self-preference bias in LLM judges is exactly the failure mode later work in this graph (Panickssery et al., "LLM Evaluators Recognize and Favor Their Own Generations") went on to measure directly.

The independent human check is small and author-run. Appendix C's blind A/B evaluation — the paper's actual ground-truth human signal — was conducted on 150 examples per task, and states plainly that "the A/B evaluation in our study was conducted by the authors." Author-run evaluation isn't disqualifying, but it is a narrower and more bias-prone check than independent crowdworkers, and it covers only 3 of the 7 tasks.

baselines

Every comparison is the same base LLM with vs. without the refinement loop — there's no comparison to other test-time refinement methods (Self-Correct, PEER) or to a second, independent LLM as critic instead of self-critique.

contamination

Not discussed. GSM8k, CommonGen-derived constrained generation, and the dialogue/sentiment datasets all predate GPT-3.5/GPT-4's training cutoffs by a wide margin, and the paper offers no contamination analysis.

what's missed

Appendix J's Wilson confidence intervals show GPT-3.5's gains are statistically significant on only 3 of 7 tasks and ChatGPT on 4 of 7 (GPT-4 fares best, "nearly all" significant) — several of the reported deltas are within noise for the weaker base models.

would raise confidence

A non-oracle math result reported as the headline instead of buried in an appendix; independent (non-author) human annotation at the same scale as the GPT-4-pref proxy; and a critique-quality control where the critic is a different model than the one being refined.

// bottom line

The core finding is real and durable: a frozen LLM can meaningfully improve its own output by critiquing and rewriting itself, no extra training required, and the feedback-quality ablation proves the mechanism (not just extra sampling) is doing the work. But three of the seven tasks are graded by the same model family being tested, the one task graded by an external ground truth barely moves without an oracle stopping rule, and the fully independent human check is a 150-example, author-run sample. SELF-REFINE works reliably in exactly the regime the paper is most careful to separate out: preference-style tasks judged by an LLM, not tasks with a hard, checkable answer.

// takeaways

What to remember

  1. One model, three hats. SELF-REFINE needs no extra network, no reward model, and no training data — the same frozen LLM generates, critiques, and rewrites, driven entirely by few-shot prompts.
  2. Feedback quality is the mechanism, not a footnote. An ablation swapping actionable feedback for generic feedback collapses gains (Sentiment Reversal 43.2 → 31.2), and removing feedback entirely zeroes out the task — the loop's value lives in what the critique says, not just in getting extra attempts.
  3. The math-reasoning result is the tell. Where correctness is objective and there's no oracle to lean on, the true gain is ~0; the encouraging GSM8k curve in the appendix only appears once the ground-truth label is allowed to decide when to stop iterating.
  4. A foundation for the self-critique agent pattern — with a caveat baked in. Self-Refine's generate → critique → rewrite loop became a standard building block for later agents (cited by Reflexion, among many others); its clearest lesson is that self-critique is trustworthy in proportion to whether the critic has real, external grounding.
