Mind2Web: Towards a Generalist Agent for the Web

The first dataset for building and evaluating agents that follow natural-language instructions on any real website — 2,350 crowd-annotated tasks across 137 unsimplified sites and 31 domains — plus MindAct, a two-stage rank-then-choose pipeline that makes thousand-element HTML pages tractable for an LLM.

Xiang Deng, Yu Gu, Boyuan Zheng, Shijie Chen, Samuel Stevens, Boshi Wang, Huan Sun, Yu Su · The Ohio State University
venue NeurIPS 2023 D&B citations 1,163 topic Agents source ↗

// the gap

What problem is this solving?

The question Mind2Web poses is deceptively simple: can we build one agent that, handed any website and a high-level instruction in plain English, just does the task? The web hosts billions of sites. You can never collect training data for all of them, so the defining requirement is not raw competence on a known site — it's generalization to sites and entire domains the agent has never seen.

By 2023 there was a body of web- and mobile-agent work, but none met that bar. Existing datasets failed on one or more of three axes. (1) Simulated, simplified environments. MiniWoB++ used 100 hand-built mini-pages averaging 28 elements each; WebShop was a single simulated shopping site. Models tuned on these never touch the noise, dynamism, and sheer DOM size of the real web. (2) A narrow, fixed set of sites or domains, so there was no way to even measure cross-domain transfer. (3) Low-level, step-by-step instructions — "type New York in the location field, click search, choose the tomorrow tab" — which test instruction-following, not the planning-and-grounding an autonomous agent actually needs from a high-level goal like "what's the weather in New York tomorrow?"

Mind2Web's bet is to fix all three at once: real, un-simplified websites (averaging 1,135 elements per page versus MiniWoB++'s 28); 137 sites spanning 31 domains, deliberately split so you test on unseen sites and unseen domains; and only high-level goals, leaving the decomposition to the agent. The tasks are genuinely diverse — booking multi-leg flights, filtering Netflix comedies by release year, following a profile and liking its latest tweet — and they are long, averaging 7.3 actions and running up to 14 steps.

// how it works

The dataset, then the model

The paper has two halves: the Mind2Web dataset and MindAct, an exploratory model that shows the dataset is learnable.

Building the dataset

Collection runs in four stages. Website selection: start from 5 top-level domains (Travel, Shopping, Service, Entertainment, Information), break them into 31 secondary domains, and pick 3–5 of the most popular US sites per domain by similarweb ranking — 137 sites in all. Task proposal: Amazon Mechanical Turk annotators write open-ended, multi-step, high-level tasks; ChatGPT generates 50 seed tasks per site purely for inspiration (10 shown at a time), and proposals too close to a seed are rejected. Task demonstration: a Playwright-based tool records the worker performing the task, splitting every step into element selection then operation selection. Verification: the authors check every trace — they discarded 61 of 2,411 tasks, refined the description on 390 to match the demonstrated actions, and trimmed extraneous steps on 187, leaving 2,350 verified tasks.

Each instance is a task description, an action sequence of (target element, operation) pairs, and rich webpage snapshots — self-contained MHTML, a DOM snapshot with layout/style, a HAR network-traffic file, and the full interaction trace — so any task can be replayed entirely offline. Just three operations cover real usage: Click (folding in Hover and Press-Enter), Type, and Select Option; Type and Select also carry a value argument.

MindAct: rank, then choose

The core modeling obstacle is size. A real page can carry thousands of DOM elements — too long and too expensive to drop into an LLM context. MindAct splits the problem so a small model does the bulk filtering and the LLM only makes the final decision.

Stage 1 — candidate generation. A fine-tuned small LM (DeBERTa-base, 86M) acts as a cross-encoder ranker. The query is the task description concatenated with the previous actions; each DOM element is represented by its tag, text, salient attributes, and its parent/child context. The element is scored against the query, the score is passed through a sigmoid, and the model is trained with binary cross-entropy using the true target element as the positive and randomly sampled page elements as negatives. At inference the top-50 elements become the candidate pool.

se = σ( CrossEncoder( query ⊕ element ) ),    query = task ⊕ previous actions // per-element matching score; trained with BCE over 1 positive (the target) + sampled negatives, top-50 kept

Stage 2 — action prediction as multiple choice. Instead of asking the LLM to generate the target element (hard, error-prone), MindAct reframes selection as multi-choice QA — a discriminative formulation shown to be more sample-efficient and generalizable for grounding. The page is pruned to the candidates and their neighbours, then presented five options at a time plus a "None of the above" choice. The LLM picks one and emits the operation (and value). If several groups return a pick, the winners are re-grouped and the process repeats until a single element survives or every option is rejected. This works both by fine-tuning open models (Flan-T5) and by in-context prompting closed ones (GPT-3.5, GPT-4).

// what they found

The results

Mind2Web is graded on three generalization splits: Cross-Task (252 test tasks from 69 sites — same sites seen in training), Cross-Website (177 tasks from 10 held-out sites per domain), and Cross-Domain (912 tasks from 73 sites in two entirely held-out top-level domains, Information and Service). Training uses 1,009 tasks. Metrics: Element Accuracy, Operation F1, Step Success Rate (a step counts only if both element and operation are right), and the stringent whole-task Success Rate (every step correct). Cleaning the raw HTML cuts the average element count from 1,135 to 580 while keeping 94.7% target recall, and the DeBERTa ranker reaches 88.9 / 85.3 / 85.7% Recall@50 on the three splits.

The headline finding is that the multi-choice formulation matters enormously. Direct generation with Flan-T5B manages just 20.2% element accuracy / 17.5% step SR on Cross-Task; the same backbone inside MindAct jumps to 43.6% / 41.0%. The best model, MindAct with Flan-T5XL, reaches 52.0% step success rate on Cross-Task and 38.9% / 39.6% when generalizing to unseen websites and domains. GPT-4 (in-context, only 3 demos, scored on a 50-task budget subset with top-10 candidates) lands at 36.2 / 30.1 / 26.4% step SR — on par with the tuned Flan-T5 models for element selection in the cross-website/domain settings, which the authors read as real promise for prompting-based agents. GPT-3.5, by contrast, sits near the baselines (~20% element accuracy) and over-selects "None".

Two structural findings stand out. First, generalization is the hard part: every model is best on Cross-Task, with a >10-point absolute step-SR gap to the unseen settings. Second, Cross-Website ≈ Cross-Domain — there is no real penalty for moving to a brand-new domain. The bottleneck is the diversity of website design and interaction logic, not domain knowledge; pretrained LMs already decompose tasks at a high level, but grounding that plan into the right clicks on an unfamiliar layout is what breaks. Figure 6 shows the same thing at the per-site level: performance fans out from ~0.7 down toward 0.2 across individual websites within every split.

// the number that matters

Despite step success rates in the 40–52% range, the whole-task Success Rate is near zero everywhere: the best is 7.1% (Flan-T5L, Cross-Task), Cross-Domain tops out at 2.9%, and GPT-4 is 2.0% across all three settings. Because a task counts only if every step is correct, a single mistake sinks it — and in 2023, these agents almost always made at least one.

// does it hold up?

The eval read

Mind2Web is one of the most influential agent benchmarks of its era, and the dataset design — especially the three generalization splits — is genuinely strong. But the headline competence numbers need careful reading, and a few choices flatter the agents.

The step metric is teacher-forced. Every step is scored independently with the ground-truth action history supplied. So the 52% step success rate never has to compound its own errors — it answers "given a perfect prefix, is the next action right?", not "can the agent run the task end to end?". The honest end-to-end number is the whole-task Success Rate column, which is near zero (best 7.1%, GPT-4 2.0%). The gap between those two columns is the real story, and it is easy to miss if you quote only step SR.

A supervised ranker does the heavy lifting. The "LLM" never sees the raw web — it sees a 5-way multiple-choice question whose options were already selected by a DeBERTa model fine-tuned on Mind2Web. Element accuracy is therefore capped by the candidate generator: Recall@50 is 85–89%, so 11–15% of targets are simply absent from the options before the LLM speaks, an unfixable ceiling no amount of LLM quality can lift. Reframing element selection as multiple choice also hands the model a much easier problem than open-ended web action — the multi-choice gain (43.6 vs 20.2) is partly the formulation, not the agent.

The cross-model comparison isn't apples-to-apples, as the authors themselves flag: Flan-T5 is fine-tuned on the full 1,009-task training set while GPT-3.5/4 get only 3 in-context demos, and GPT-4 is evaluated on just 50 tasks per split with a top-10 (not top-50) candidate pool to save budget. So "GPT-4 matches tuned Flan-T5" is suggestive, not settled.

Offline replay creates false negatives. Tasks are graded against cached snapshots, so a correct action that wasn't the one recorded during annotation fails immediately. The authors normalize equivalent elements on a page (Appendix C.1), but real sites admit multiple valid paths to the same goal, and exact element+operation matching penalizes correct-but-different behaviour. There is also no live evaluation here — "systems should transfer to live websites out of the box" is asserted from snapshot fidelity, not measured.

baselines

Fair within the dataset — a classification head (DeBERTa) and direct generation (Flan-T5B), and the multi-choice gain over generation is real (43.6 vs 20.2 element acc). But GPT-3.5/4 get only 3 demos against Flan-T5's full fine-tune, and GPT-4 runs on a 50-task subset with top-10 candidates — the authors flag both as not directly comparable.

contamination

Snapshots are real, public 2023 websites released on HuggingFace; the HTML and tasks can leak into the pretraining of later LLMs. Static snapshots also drift from the live web over time, and the paper does no live-site evaluation.

what's missed

Step SR is teacher-forced (oracle action history) and so doesn't compound — the real end-to-end number is the near-zero whole-task Success Rate. Element accuracy is additionally capped by the ranker's ~85–89% Recall@50. Text-only modeling drops the rendered visual page entirely.

would raise confidence

Live online execution instead of cached replay; partial-credit / goal-completion metrics tolerant of alternate valid paths; a matched-budget LLM comparison (same demos and candidate pool); and a vision-grounded variant that uses the screenshots already in the dataset.

// bottom line

Mind2Web reframed web agents around real, unseen websites and high-level goals, and gave the field its first generalization-graded benchmark plus a pragmatic rank-then-choose recipe (MindAct) that made thousand-element pages tractable. Its step-success numbers look healthier than the near-zero whole-task success because they are scored with oracle history over a supervised candidate pool — the durable lesson is just how far 2023 LLMs were from reliable, end-to-end web autonomy. The dataset, more than the model, is the contribution that stuck.

// takeaways

What to remember

  1. The first generalization-first web-agent benchmark. 2,350 high-level tasks over 137 unsimplified real sites and 31 domains, with explicit Cross-Task / Cross-Website / Cross-Domain splits so you measure transfer to sites and domains the agent never trained on — not just competence on a memorized site.
  2. MindAct's rank-then-choose recipe. A small fine-tuned ranker (DeBERTa-base, Recall@50 ≈ 85–89%) prunes thousand-element pages to a handful of candidates, then the LLM answers a 5-way multiple-choice question. The discriminative framing beat direct generation by >20 points of element accuracy — a pattern echoed in later web and GUI agents.
  3. An honest 2023 baseline. Best step success ~52% (Cross-Task, teacher-forced) but whole-task success near zero (≤7.1%; GPT-4 2.0%). Reliable end-to-end web autonomy was far off, and the binding constraint was generalizing to unfamiliar website design and interaction logic, not domain knowledge.
