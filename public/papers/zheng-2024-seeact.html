GPT-4V(ision) is a Generalist Web Agent, if Grounded

SeeAct gives GPT-4V nothing but a rendered webpage screenshot and asks it to plan and execute open-ended tasks. It plans like a human — but converting that plan into one precise click is the bottleneck: even the best automatic grounding strategy trails a human-in-the-loop oracle by 20–30 points, and Set-of-Mark-style image annotation actually loses to a plain HTML-text multiple-choice.

// the gap

What problem is this solving?

By late 2023, "generalist web agent" work — most notably MindAct on the Mind2Web dataset — read a webpage the way a search-engine crawler does: as raw HTML fed to a language model. That's a bad match for how the web is actually built. The authors quantify it with a single example screenshot: it contains 423 HTML elements, which the GPT-2 tokenizer expands to 186,490 textual tokens — versus just 1,445 visual tokens under GPT-4V's own image tokenizer. HTML is also lossy: it misses the semantics baked into embedded images, icons, and layout that a person reads at a glance. Meanwhile GPT-4V and Gemini had just demonstrated strong general vision-language understanding, but nobody had rigorously asked whether that understanding transfers to the much messier, denser, more relational images that real websites render as.

So the paper poses two separable questions. First: can a large multimodal model, seeing only a screenshot, generate a correct step-by-step action plan for an arbitrary real-world task on an arbitrary real-world website? Second, and this is where the paper's actual contribution lives: even if it can plan correctly in natural language, can that plan be reliably converted into the exact executable action — the right HTML element, the right operation, the right value — that a browser needs? The paper's central finding is that these two questions have very different answers: GPT-4V is a strong planner, but grounding is the unsolved half of the problem.

// how it works

The method

SeeAct formalizes a web task as a sequence of browser events. At each step the agent observes an HTML document and a screenshot, and must generate an action a = (e, o, v): which element e to act on, which operation o to perform (Click, Type, Select), and what value v to supply. Because GPT-4V cannot directly output that structured triplet, SeeAct splits the problem into two stages.

at = π(st, T, {a1, …, at−1})   →   st+1 = {ht+1, it+1} // s = (HTML h, screenshot i); the model is grounded on i alone for action generation — h only enters at the grounding stage

Stage 1 — Action Generation (vision only)

GPT-4V is shown only the screenshot — no HTML — and instructed to imitate a human browsing the page: analyze the task, the visual layout, and the previous actions, then write a free-text action description, e.g. "Move the cursor over the 'Find Your Truck' button … and perform a click action." This is the step the paper shows GPT-4V is already good at.

Stage 2 — Action Grounding (turning words into a click)

The hard part is mapping that description onto one specific HTML element. SeeAct tests three grounding strategies, compared side by side in Figure 2: Element Attributes (ask the model to name the target's text and type, then heuristically search the DOM for a match — falling back to a follow-up multiple-choice question if several elements match); Textual Choices (a DeBERTa cross-encoder from MindAct first ranks the top-50 candidate elements, clustered into groups of 17, and the model picks by letter from their HTML text); and Image Annotation — the same idea as Set-of-Mark prompting — overlaying numbered bounding boxes on the ranked candidate elements and asking the model to output the matching label. A fourth, non-deployable reference point, Oracle grounding, has a human annotator identify the element the model's own description was clearly pointing to — an upper bound on what perfect grounding of GPT-4V's plans would achieve.

Everything is evaluated on Multimodal Mind2Web, a version of the Mind2Web dataset (2,000+ tasks across 137 sites, 31 domains) that the authors re-aligned with verified screenshots. Its three held-out test splits probe different kinds of generalization: Cross-Task (new tasks, same sites/domains as training), Cross-Website (10 new sites per domain), and Cross-Domain (two entire domains held out). Offline metrics — Element Accuracy, Operation F1, Step Success Rate — score against Mind2Web's single annotated reference trajectory. Because a real task can have more than one valid path to success, the authors also built a Playwright-based online evaluation tool that runs agents on live websites with a human monitoring for whole-task success, on a 90-task subset spanning all three splits.

// what they found

The results

With oracle grounding, GPT-4V is a genuinely strong generalist web agent: step success rates of 61.9% (Cross-Task), 65.0% (Cross-Website), 62.1% (Cross-Domain), beating the next-best method (supervised FLAN-T5-XL) by 8.4 points on Cross-Task and by a much larger 23.9 / 23.2 points on the harder Cross-Website / Cross-Domain splits — direct evidence that in-context GPT-4V generalizes to unseen sites far better than a fine-tuned smaller model does.

But swap oracle grounding for any automatic strategy and the picture changes. Textual-choice grounding is the best of the three, and it still trails oracle by 20–30 points at every split:

The image-annotation result is the paper's sharpest finding: a Set-of-Mark-style strategy that had already been shown to work well on natural, object-centric photos underperforms plain textual choices by up to ~19 points on webpage screenshots. A targeted error analysis of 100 correct-plan-wrong-grounding cases traces this to GPT-4V's own visual limits: 54% of errors are "visual illusion" — the model fabricates a bounding box and label that were never there — and another 46% come from failing to correctly link a real bounding box to its correct number label, i.e. a relative-position confusion. Webpage screenshots are denser and more spatially cluttered than the object- or scene-centric images Set-of-Mark was validated on, and GPT-4V's grounding breaks down accordingly.

On live sites, SeeAct with textual-choice grounding reaches 37.8% whole-task success, well ahead of GPT-4 (13.3%) and FLAN-T5-XL (8.9%); with oracle grounding it reaches 51.1%. // does it hold up?

The eval read

This is a paper that is unusually careful about not overclaiming its own headline — the abstract itself flags the grounding gap rather than burying it — but a few evaluation choices still deserve a skeptical read before quoting the "51.1%" number.

1. The paper's own headline comparisons quietly mix a 30-task subset against full test sets. Table 2 marks GPT-3.5*, GPT-4*, and GPT-4V-Oracle* with an asterisk meaning "evaluated on a 30-task subset per split" — and Table 3 (the entire grounding-strategy comparison: Attributes vs. Choices vs. Annotation vs. Oracle) is explicitly built on that same 90-task subset. But the SFT baselines it's compared against — FLAN-T5-XL, BLIP-2-T5-XL — are scored on the full test sets (177 / 142 / 694 tasks for Cross-Task / Cross-Website / Cross-Domain, per Table 1). So the paper's own headline claim, "[oracle] leads by 23.9% and 23.2% step success rates" over FLAN-T5-XL on Cross-Website/Cross-Domain, is a subset-evaluated number measured against a full-test-set baseline, not a like-for-like comparison. Confirming this matters: SeeAct-Choice's own Table 3 subset numbers (39.1/32.7/42.0) don't even match its Table 2 full-test-set numbers for the identical method (40.2/32.4/36.8) — the two tables are measuring genuinely different task samples.

2. The 51.1% headline needs its asterisk carried along too. That figure is oracle-grounded — a human annotator resolves which element GPT-4V's textual plan intended — on the same 90-task subset. It is a ceiling on what perfect grounding would buy, not a number any autonomous agent actually achieved. The number an unassisted SeeAct actually reaches online is 37.8%, and offline (Offline0, the setting most comparable to standard benchmarks) it's 3.3%. All three numbers are reported side by side in the paper's own Table 4 — but headline-quoting only the oracle figure, as is easy to do from the abstract alone, meaningfully overstates deployed capability.

3. The online evaluation is small and its judging process is under-specified. Whole-task online success is measured on 90 tasks total — just 30 per split — and the difficulty breakdown in Figure 3 further splits that into buckets as small as 18 (Hard) tasks. The paper describes pairing each run with "a human annotator" to judge success but reports no inter-annotator agreement, no annotator count, and no adjudication protocol — a thinner audit trail than, e.g., WebVoyager's later κ-scored human/GPT-4V judge comparison. At n=18–37 per cell, a few judgment calls can swing a difficulty-tier percentage several points.

4. Contamination is plausible and unaddressed. Mind2Web's HTML/task text was public before GPT-4V's training cutoff, and the underlying websites are public too; the paper does not discuss whether GPT-4V (or GPT-4) could have seen this benchmark's content during training. The cross-website/cross-domain splits somewhat mitigate this for the specific held-out sites, but not for general familiarity with sites like Booking, Zillow, or Home Depot supplying most of Mind2Web's coverage.

5. No cost or latency reporting. Running GPT-4V per action step, with a screenshot in the prompt every time, has meaningfully different cost/latency characteristics than a fine-tuned FLAN-T5-XL forward pass — a comparison the paper never surfaces, even though it's directly relevant to whether SeeAct's approach is practically deployable at the reported success rates.

baselines

Reasonable and contemporaneous — FLAN-T5-XL and BLIP-2-T5 (SFT, full test sets), GPT-3.5/GPT-4 (ICL), CogAgent (pixel-level grounding), LLaVA-1.5 and Gemini Pro Vision (other LMMs). GPT-3.5*, GPT-4*, and the GPT-4V-Oracle headline number all run on the same 30-task-per-split subset, not the full test splits the SFT baselines use.

contamination

Not discussed. Mind2Web and the websites it covers are public and likely within GPT-4V/GPT-4's training window; only the cross-website/domain splits offer partial protection.

what's missed

The paper's central "oracle beats SFT by 20+ points" claim compares a 30-task-subset number to a full-test-set baseline. The oracle-grounded 51.1% is easy to misquote as an achieved, autonomous number — it isn't (unassisted online success is 37.8%). Online judging has no reported inter-annotator agreement. No cost/latency accounting for the per-step GPT-4V calls.

would raise confidence

Oracle and grounding-strategy comparisons run on the full test sets (not a 30-task subset) so they're commensurable with the SFT baselines; a reported human-judgment agreement rate for online success; a contamination check against Mind2Web's public release date; per-task token/dollar cost alongside the success-rate table.

// bottom line

SeeAct's real contribution is diagnostic, not the headline number: it isolates grounding, not planning, as the bottleneck for vision-based web agents, and it shows — against the intuition that Set-of-Mark-style marking should transfer cleanly from natural images — that image annotation trails textual-choice grounding by up to ~19 points on webpage screenshots (it does still beat the weaker element-attributes strategy, but loses decisively to a plain HTML-text multiple-choice). That finding held up well: it's a big part of why WebVoyager's own Set-of-Mark implementation (cited by this paper's own successors) grounds marks directly from the DOM rather than from GPT-4V-interpreted bounding boxes. The 51.1% success figure, though, is best read as a ceiling for future grounding work, not as SeeAct's own delivered accuracy — and even the paper's offline "beats SFT by 20+ points" framing conflates a subset-evaluated oracle with full-test-set baselines.

// takeaways

What to remember

  1. Grounding, not planning, is the bottleneck. GPT-4V reliably writes a correct natural-language action plan from a screenshot alone. The unsolved problem is converting that plan into one exact, executable click — every automatic grounding strategy trails a human-grounded oracle by 20–30 points.
  2. Set-of-Mark-style image annotation is not automatically the right grounding tool for GUIs. A technique validated on natural, object-centric photos underperforms a plain HTML-text multiple-choice by up to ~19 points on dense, cluttered webpage screenshots — GPT-4V hallucinates or mislinks bounding-box labels 54%/46% of the time in the failure cases studied. This is a load-bearing counterpoint to reflexively porting visual-marking prompting into new domains.
  3. Offline benchmarks understate live-web capability, mostly. Every method scores much higher online than under the strict zero-error-tolerance offline metric, because real tasks admit multiple valid solution paths that a single reference trajectory can't credit — a methodological caution for any web-agent benchmark scored against one fixed gold path. (The one exception: FLAN-T5-XL's online score still trails its own error-tolerant offline number, a reminder the effect isn't perfectly uniform.)
