Learning Multiagent Communication with Backpropagation

A single differentiable module, shared across every agent, that talks by broadcasting the mean of everyone else's hidden state — turning "what to communicate" into just another thing backprop can learn, with no discrete messages or per-bit reinforcement-learning credit assignment required.

Sainbayar Sukhbaatar, Arthur Szlam, Rob Fergus

// the gap

What problem is this solving?

Plenty of real tasks — cars merging at a junction, a squad coordinating an attack, elevator banks dispatching cars, robots on a soccer pitch — need multiple agents to act as a team under only partial observability: no single agent sees the whole picture, so they have to tell each other things. The paper's framing is that almost all existing multi-agent systems solve this by fixing the communication protocol by hand ahead of time (e.g. broadcast your position and distance to the ball every step) and only learning the policy that consumes it. That is brittle and unambitious: it assumes a system designer already knows what's worth saying, in what format, and it hard-codes the answer rather than letting the agents discover it from the task itself.

The alternative — treat the message itself as something to be learned — had been tried, but mostly with tabular RL or evolutionary search on small, low-dimensional games, or (the paper's closest contemporary, Foerster et al.'s riddle-solving work, the direct predecessor of the DIAL paper published at the same NeurIPS) by making the message just another discrete action, picked by an action selector and trained with the same noisy, reward-only policy-gradient signal used for environment actions. That works, but every bit of the message then has to earn its keep through the same low-bandwidth RL credit-assignment channel as "move left" or "attack." The gap this paper targets is a fully differentiable alternative: let deep agents learn to communicate purely by backpropagating the downstream task loss through the message itself, with a design that works for a variable, dynamically-changing number of homogeneous agents rather than a fixed cast of two or three.

// how it works

The method

CommNet treats a whole team of J cooperating agents as one large feed-forward network, Φ, that maps every agent's state-view sj to every agent's action aj. Inside, Φ is built from a single module fi (weights shared across agents and across communication steps), applied K times. At communication step i, each agent's module takes its own running hidden state hij together with an incoming communication vector cij, and produces the next hidden state. The communication vector is deliberately the simplest possible aggregate: the mean of every other agent's hidden state from the previous step.

hi+1j = fi(hij, cij)    ci+1j = 1J − 1 Σj′≠j hi+1j′ // every agent's next hidden state depends on its own state and the average of everyone else's — no addressing, no discrete symbols, just a mean-pooled broadcast that any single differentiable module can consume

Because the aggregation is an unweighted mean, the whole system is permutation-invariant and dynamically sized: the same trained weights apply whether there are 3 agents or 30, and reordering the agents changes nothing. When fi is a single linear-plus-nonlinearity layer, the entire step can be written as one big matrix multiply hi+1 = σ(Tihi), where Ti is block-structured — an Hi block on the diagonal for each agent's own-state transform, and a shared, appropriately-rescaled C̄i block everywhere off-diagonal for the broadcast. A problem-specific encoder r(sj) produces the initial hidden state h0j, and a decoder q(hKj) — a linear layer plus softmax — turns the final hidden state into an action distribution. When supervision is available at every step the whole pipeline (encoder → K communication rounds → decoder) is trained by ordinary backprop; when only a sparse task reward is available, the same architecture is instead trained with policy gradient and a learned state-dependent baseline (Appendix A of the paper).

Three extensions widen the basic recipe. Local connectivity replaces the all-agents mean with a mean over only the agents currently within some range N(j) — the model then reads naturally as a dynamic graph, related to the belief-propagation view of graph neural networks (the paper explicitly draws the connection to Scarselli et al.'s GNN and Li et al.'s Gated Graph Sequence Neural Networks). Skip connections re-inject the original encoding h0j as an extra input at every communication step, which helps when the raw input signal would otherwise get diluted after several rounds of averaging. Temporal recurrence reuses the same module across environment time steps instead of (or in addition to) communication rounds, turning f into a plain RNN or, in the paper's strongest configuration, an LSTM.

The paper is explicit about how this differs from the DIAL/RIAL line of concurrent work from Foerster et al.: that approach has each agent emit a message as a discrete symbol, sampled from a softmax and optimized as if it were just another environment action under policy gradient — a message is only ever an action, never a hidden activation. CommNet's message is never sampled and never a discrete symbol at all; it is a raw, continuous hidden-state vector that flows straight into the next module's forward pass, so the "should I say X" decision is shaped by ordinary backprop through the whole communication stack rather than by a separate RL credit-assignment problem for every bit transmitted. The published DIAL paper later converges partway toward this by making its own channel continuous and gradient-carrying during centralized training — but DIAL still hard-thresholds to a bit at decentralized execution time and sends addressed, point-to-point messages between a handful of fixed agents, whereas CommNet stays continuous at both train and test time and broadcasts an unaddressed mean to a population whose size can change from one episode to the next.

To isolate what the learned continuous channel buys, the paper compares CommNet against three baselines built from the exact same encoder/decoder: an independent controller (communication zeroed out entirely), a fully-connected model (a single fixed-size network over the concatenation of every agent's state — no modularity, no invariance to agent count or order), and a discrete communication model (each agent emits a 1-hot symbol at every step, trained with policy gradient, and the bag of symbols from other agents is OR-ed together as the incoming message).

// what they found

The results

On a minimal lever-pulling game (5 levers, 500 agents in the pool, 5 drawn per round, reward for pulling distinct levers), CommNet reaches 0.99 of the maximum distinct-levers score under direct supervision and 0.94 under reinforcement learning, versus 0.59/0.59 for the independent, non-communicating controller — communication is the entire difference between agents that can coordinate and agents that can't.

On the traffic-junction task (cars entering a grid from four directions, must avoid collisions), CommNet cuts the failure rate to 2.2% (MLP module), 7.6% (RNN) and 1.6% (LSTM) — versus an independent controller's 20.6%/19.5%/9.4% and a discrete-communication controller's 15.8%/15.2%/8.4% on the same three module types. The fully-connected baseline is inconsistent: it beats independent agents with an LSTM module (4.8%) but does markedly worse with an RNN module (34.8%), underscoring that naive full connectivity is not a robust substitute for CommNet's structured, invariant broadcast. As the agents' own vision range shrinks from 7×7 down to a fully blind 1×1, every baseline's failure rate climbs toward 100%, while CommNet's stays close to 10% even with zero visibility — communication is doing most of the work that direct observation would otherwise provide. On harder variants — an easy two-road junction and a much harder four-junction grid — discrete communication is competitive on the easy case (1.1% vs. CommNet's 0.3%) but falls behind on the hard case (28.2% vs. CommNet's 22.5%, and 21.1% for CommNet with local rather than full-broadcast connectivity, the best result on that variant).

On the combat task (5-vs-5 grid battle against a hard-coded, shared-vision enemy team), CommNet's LSTM configuration wins 49.5% of games, versus 44.3% for the independent LSTM controller and 46.4% for discrete communication; with an MLP module the gap is larger — 44.5% for CommNet vs. 34.2% independent — and the fully-connected baseline actually degrades below the independent controller for every module type (as low as 2.9% with an RNN module). CommNet's advantage held up across variations in team size (51.0% vs. 29.2% at m=3 agents; 45.4% vs. 30.5% at m=10) and wider vision (73.0% vs. 60.5% at 5×5 visibility).

Finally, on the bAbI toy question-answering suite (each story sentence treated as its own agent), CommNet reaches 7.1% mean error and fails 3 of 20 tasks, versus a plain LSTM baseline's 36.4%/16 failed and an independent (no-communication) controller's 15.2%/9 failed — but it is still behind memory-network-style architectures purpose-built for this kind of multi-hop reasoning: MemN2N reaches 4.2%/3 failed and DMN+ reaches 2.8%/1 failed. Communication roughly halves independent agents' error and matches MemN2N's failed-task count, without any architecture designed specifically for reading comprehension.

// does it hold up?

The eval read

Every task here is either built by the authors themselves (lever pulling, the two MazeBase-based games) or a small, synthetic toy suite (bAbI) — this predates any shared, third-party multi-agent benchmark that later papers in this exact lineage would standardize on (e.g. the Multi-Agent Particle Environment used by MADDPG, or StarCraft micromanagement used by COMA's successors). That means the headline numbers are only comparable within this paper's own four baselines, not against any other published multi-agent communication method — a real limitation for a paper whose whole point is a head-to-head "does learned continuous communication beat the alternatives" claim.

The combat task's opponent is also a fixed, hand-coded rule-based bot rather than a learning adversary, and the paper states outright that "this shared vision gives an advantage to the bot team" — the enemy sees any agent visible to any of its members, while the model's own agents only see their individual 3×3 neighborhood. Win rates topping out around 44–50% may partly reflect that built-in asymmetry rather than a ceiling on what continuous communication can achieve; a symmetric matchup or a learning opponent would isolate the model's own contribution more cleanly. Reported standard deviations are also large relative to the gaps being claimed — CommNet's MLP combat win rate is 44.5±13.4% against an independent controller's 34.2±1.3%, a real and consistent-looking pattern across many task/module pairs, but no individual comparison comes with a significance test, so the paper leans on the breadth and consistency of the pattern (CommNet wins across four task families and three module types) rather than any one comparison being airtight on its own.

baselines

The independent, fully-connected, and discrete-communication baselines all share CommNet's exact encoder/decoder and training loop, differing only in how (or whether) agents talk — a genuinely fair like-for-like design. The discrete baseline in particular is a reasonable stand-in for the concurrent RIAL-style approach the paper contrasts itself against in its related-work section, though it is the authors' own re-implementation rather than a borrowed, independently-tuned method.

contamination

Not applicable in the usual leakage sense — every task (lever pulling, traffic junction, combat, bAbI) is procedurally generated or a fixed synthetic QA suite, not scraped web text, so there is no train/test overlap concern beyond bAbI's own standard 10K train/test split, which the paper follows as published.

what's missed

The paper's own communication analysis (Fig. 3) finds that most broadcast vectors on the traffic-junction task are near-zero — "silent" — with only a few distinct non-zero clusters correlating with specific braking decisions. That is presented as evidence of a sparse, efficient learned protocol, but it also hints at the unweighted-mean channel's core limitation: with more agents, any one agent's signal is diluted by 1/(J−1), and there is no mechanism for an agent to decide who is worth listening to. The paper never tests communication at a large agent count to see whether this dilution actually degrades performance.

would raise confidence

A shared, third-party multi-agent benchmark (rather than four self-designed testbeds) would make the "beats discrete and no-communication baselines" claim externally checkable; a learning or self-play opponent in combat would remove the fixed-bot asymmetry; and — the authors' own stated gap — testing at agent counts and message sizes well beyond this paper's handful of agents would show whether broadcast-mean communication scales or needs the "more sophisticated connectivity structures" the discussion section defers to future work.

// bottom line

The comparisons that exist are genuinely apples-to-apples — same backbone, same training budget, only the communication mechanism changes — and the pattern (continuous beats discrete beats none) repeats across four unrelated task families, which is real evidence. What's missing is anything to compare against outside the paper's own four baselines and its own four hand-built tasks, plus any test of the one scaling question the authors flag themselves and explicitly leave open: whether an unweighted broadcast mean still works once the number of communicating agents gets large.

// takeaways

What to remember

  1. CommNet's core idea is disarmingly simple: replace a hand-designed communication protocol with the mean of every other agent's hidden state, broadcast at each of K communication steps and consumed by one shared, ordinary feed-forward module — no discrete symbols, no separate communication-specific RL machinery, just backprop through the whole stack.
  2. Mean-pooling over a variable set of neighbors makes the model permutation-invariant and size-agnostic: the same trained weights handle a changing number of agents, which is exactly the property that connects this architecture to the graph-neural-network line of work the paper cites (Scarselli et al.'s GNN, Li et al.'s GGSNN).
  3. Continuous, backprop-trained communication beat both a no-communication baseline and a discrete, policy-gradient-trained communication baseline across four different task families — lever pulling, traffic junction, combat, and bAbI — but always in self-built toy environments against, in the one adversarial task, a fixed rule-based (not learning) opponent, and always at a small agent count the authors themselves flag as untested at scale.
  4. CommNet sits alongside DIAL as the two headline answers, from the same NeurIPS 2016 cycle, to "how should agents learn what to say": DIAL keeps a differentiable channel only during centralized training and discretizes to addressed, point-to-point bits at execution; CommNet stays continuous and broadcasts an unaddressed mean at both train and test time. Both COMA and MADDPG cite this paper directly as part of the same centralized-training lineage.
