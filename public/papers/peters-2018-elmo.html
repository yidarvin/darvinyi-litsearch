Deep Contextualized Word Representations

ELMo: stop assigning each word one fixed vector. Instead derive every token's representation from a pretrained bidirectional language model — a task-specific learned mix of all its internal layers — and bolt it onto any existing model to lift the state of the art on six NLP benchmarks at once.

// the gap

What problem is this solving?

By 2018, pretrained word vectors — word2vec (Mikolov et al., 2013) and GloVe (Pennington et al., 2014) — were a default ingredient in nearly every neural NLP model. But they share a crippling limitation: each word type gets exactly one vector. "Play" has a single embedding whether it's a verb, a theatrical noun, or a move in a game. That one vector is forced to average over every sense, and it cannot model two things at once: (1) the complex syntactic and semantic facts about how a word is used, and (2) how that usage shifts with context — polysemy.

The fix everyone wanted was to make a word's representation a function of the whole sentence it sits in. Two precursors had tried: CoVe (McCann et al., 2017) reused the top encoder layer of a supervised machine-translation model, and TagLM (Peters et al., 2017) reused the top layer of a language model. Both helped — but both took only the final layer of a pretrained network, on the implicit assumption that the most useful representation lives at the top.

This paper's bet is that that assumption is wrong. A deep network trained to predict the next word learns a stack of representations, and the layers are not redundant: lower layers capture syntax, higher layers capture meaning-in-context. Exposing only the top layer throws away the syntactic signal; exposing only a fixed layer forces every downstream task to use the same blend. The gap ELMo targets is a cheap, drop-in way to give existing supervised models access to the full depth of a large, purely unsupervised language model — and to let each task learn which layers it needs.

// how it works

The method

ELMo — "Embeddings from Language Models" — has two halves: pretrain a bidirectional language model, then collapse its layers into a task-specific vector you concatenate into any model.

1. Pretrain a bidirectional LM (biLM)

A forward LM predicts each token from its left context; a backward LM predicts each token from its right context. ELMo's biLM is trained to jointly maximize the sum of the forward and backward log-likelihoods, sharing the token-embedding and softmax parameters across the two directions while keeping separate LSTM weights per direction:

Σk [ log p(tk | t1…tk−1; Θx, ΘLSTM→, Θs) + log p(tk | tk+1…tN; Θx, ΘLSTM←, Θs) ] // joint forward + backward objective; token (Θx) and softmax (Θs) weights tied across directions

The input is purely character-based: 2048 character n-gram convolutional filters → two highway layers → a linear projection to 512 dimensions, so the model can embed any token (even out-of-vocabulary words) from its spelling. On top sit L = 2 biLSTM layers with 4096 units and 512-dimensional projections, plus a residual connection from the first layer to the second — a halved version of the CNN-BIG-LSTM from Józefowicz et al. (2016). Trained for 10 epochs on the 1B Word Benchmark, the model reaches an average forward/backward perplexity of 39.7 (vs. 30.0 for the larger forward-only CNN-BIG-LSTM).

2. Collapse all layers into ELMo

For each token, the 2-layer biLM produces three representations: the context-independent token layer plus the two biLSTM layers (each a concatenation of the forward and backward states). Rather than pick one, ELMo learns a task-specific linear combination of all of them — softmax-normalized weights sj across the three layers, times a single scalar γ that rescales the whole vector:

ELMoktask = γtask · Σj=0L sjtask · hk,jLM // stask: softmax-normalized per-layer weights · γtask: scalar that scales the whole vector · hk,j: layer j for token k

That is the entire learned apparatus added to a task: three weights and one scalar (optionally with layer-norm on each biLM layer before the weighted sum). To use it, you freeze the biLM, run it over the input sentence, and concatenate the resulting ELMo vector with the task model's usual word embedding — feeding it into the lowest layer of the task model and, for some tasks, into the output layer as well. The expensive language model is trained once; every downstream task reuses it as a fixed feature extractor and only learns its own tiny layer-mixing.

// what they found

The results

Adding ELMo to a strong task-specific baseline improved the single-model state of the art on all six benchmarks tested — question answering (SQuAD), textual entailment (SNLI), semantic role labeling (SRL), coreference (Coref), named-entity recognition (NER), and fine-grained sentiment (SST-5). The relative error reductions over the authors' own baselines range from roughly 6% to 25%:

The biggest single jump is SQuAD: 81.1 → 85.8 F1, a 24.9% relative error reduction that also set a new single-model record (+1.4 over the prior best); an 11-member ensemble reached 87.4. The same swap lifted SRL by 3.2 F1, coreference by 3.2 average F1, NER to 92.22 F1, and SST-5 by 3.3 points. The thinnest gain is SNLI (+0.7), where the baseline was already near the task's ceiling of ~88.

ELMo is also strikingly sample-efficient. With ELMo added, a model reaches the no-ELMo full-data performance using only a small fraction of the labeled set — for SRL, the ELMo model trained on just 1% of the data matches the baseline trained on 10%, and the gap is largest in the low-data regime:

Why mixing all layers matters

The paper's analysis explains the design. Probing the biLM's layers directly shows they specialize: the lower biLSTM layer is better at syntax (POS tagging on PTB: layer 1 = 97.3% vs. layer 2 = 96.8%), while the higher layer is better at word sense (fine-grained WSD F1: layer 2 = 69.0 vs. layer 1 = 67.4 — competitive with a dedicated supervised WSD system at 70.1). Because tasks need different blends, ELMo lets each one learn its own weighting, and using all layers beats using only the top layer:

// does it hold up?

The eval read

For 2018, this is an unusually honest and broad evaluation: six genuinely different tasks, each with a strong reimplemented baseline and the prior published single-model SOTA shown side by side, with mean±std over five seeds reported for the small-test-set tasks (NER, SST-5). The signature layer-wise probes (POS, WSD) aren't decoration — they justify the core design choice instead of just asserting it. But a few things are worth holding at arm's length.

First, every headline "increase" is measured over the authors' own baseline, not the prior SOTA. On SQuAD the baseline (81.1) actually trails the previous best (84.4); ELMo lifts it past to 85.8. The additive gain is real and the SOTA gap is shown transparently, but the framing — "+4.7" — is the delta over a model that started behind the field, not over the strongest available system. Second, the clean ablation that would isolate ELMo's central claim — that mixing all layers beats the top layer — is only partially run (Table 2 shows all-layers > top-layer on the dev sets). The repeated contrast with CoVe ("+4.7 vs. CoVe's +1.8") conflates two variables at once: ELMo exposes all layers and is pretrained on far more unlabeled text than CoVe's parallel MT corpus, so it doesn't cleanly separate "deep layer-mixing" from "more pretraining data."

Third, the gains are uneven in a way the "improves SOTA on all six tasks" headline smooths over. SNLI moves only +0.7 against a ±0.17 std — statistically fine, but a near-ceiling task where the practical lift is marginal. And each result reflects light per-task tuning: where to inject ELMo (input only, output only, or both — Table 3), the regularization strength λ, and the scalar γ are all chosen per task, so this is six tuned configurations, not one fixed recipe.

baselines

Strong, current (2017–18) task-specific baselines plus prior single-model SOTA shown per task. But the headline Δ is over the authors' own baseline, which on SQuAD/SST-5 starts behind the prior best — honestly disclosed, but not a delta over the strongest system.

contamination

biLM pretrained on the 1B Word Benchmark (news text), largely disjoint from the task corpora, so direct test leakage is unlikely. No formal leakage/overlap audit — a check almost nobody ran in 2018, and a non-issue here more by luck of domain than by design.

what's missed

Uneven gains hidden by the "all six tasks" framing (SNLI only +0.7 near ceiling); per-task tuning of injection point, λ and γ means six tuned setups rather than one recipe; the CoVe contrast entangles "deep layers" with "more pretraining data."

would raise confidence

A data/compute-matched ablation varying only top-layer vs. learned all-layer mix; significance tests on the deltas; a single fixed recipe applied across tasks without per-task search.

// bottom line

The numbers hold and the evaluation is honest for its moment. ELMo's lasting result isn't the leaderboard line — BERT erased these records within a year — but the demonstrated principle: a large unsupervised LM's internal layers carry complementary syntactic-to-semantic structure that transfers broadly, and exposing all of them beats picking one. What didn't survive is the paradigm the eval measured: a frozen biLM feeding features into bespoke per-task architectures. Months later, GPT and BERT showed that fine-tuning a single Transformer end-to-end beats feature extraction — so ELMo's evaluation was rigorous about an approach that was about to be replaced.

// takeaways

What to remember

  1. Context beats type. Swapping one-vector-per-word (word2vec/GloVe) for a function of the whole sentence — a learned mix of a pretrained biLM's layers — lifted the state of the art on all six tasks tested, with 6–25% relative error reductions, just by changing the input representation.
  2. The layers are not interchangeable. A deep LM's lower layers encode syntax (POS 97.3) and its higher layers encode word sense (WSD 69.0). ELMo's core trick is exposing all layers and letting each task learn its own softmax-weighted blend — which beats using the top layer alone.
  3. ELMo was the bridge into the pretraining era. It proved large-scale unsupervised LM pretraining transfers broadly and cheaply (freeze once, reuse everywhere). But it kept the biLM frozen and fed features into task-specific models — a paradigm GPT and BERT replaced within months by fine-tuning a single Transformer end-to-end.
