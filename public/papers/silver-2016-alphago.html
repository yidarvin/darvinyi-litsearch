2016 · Nature · Post-training & RL

Mastering the game of Go with deep neural networks and tree search

A pipeline of two convolutional networks — a policy network that picks moves, a value network that judges positions — fused into Monte Carlo tree search. AlphaGo turns a game whose ~250-move breadth and ~150-move depth made brute-force search hopeless into one where a trained model narrows both, and beats a human professional for the first time.

// the gap

What problem is this solving?

Every game of perfect information has an optimal value function that could, in principle, be found by exhaustively expanding the search tree — but only in principle. As the paper frames it, the tree contains roughly bd sequences of moves, where b is the breadth (legal moves per turn) and d is the depth (game length). Chess sits at b≈35, d≈80: large, but tractable enough that minimax search with a hand-built evaluation function beat the world champion two decades earlier. Go sits at b≈250, d≈150 — a search space so much larger that exhaustive or near-exhaustive search was, in the authors' words, "believed to be intractable in Go due to the complexity of the game." (The number of legal 19×19 board positions is commonly put at roughly 2×10170, a separate but related figure for just how large Go's state space is.)

The two standard tricks for shrinking a search tree both broke down for Go. Depth reduction — truncating the tree early and plugging in an approximate value function — had already delivered superhuman chess, checkers and Othello engines, but no one had a Go evaluation function good enough to truncate with. Breadth reduction via Monte Carlo rollouts — sampling random playouts to the end of the game and averaging the outcome — powered strong backgammon and Scrabble programs and the best existing Go engines, but even the strongest Monte Carlo tree search (MCTS) Go programs, enhanced with policies trained to imitate human expert moves, only reached "strong amateur play." Prior neural or learned components in that pipeline were themselves shallow — linear combinations of hand-designed input features — not the deep, many-layer networks already reshaping image classification and Atari game-playing. The gap, then, was whether a genuinely deep convolutional network could supply both a policy (to narrow breadth) and a value function (to narrow depth) good enough to make Go's tree tractable — something "previously believed to be at least a decade away."

// how it works

The method

AlphaGo's training pipeline (Fig. 1) has three stages, each producing a network the next stage builds on, plus one auxiliary network trained alongside the first stage. It treats the 19×19 board as an image and reuses a convolutional architecture for both policy and value prediction.

1 — supervised learning (SL) policy network

A 13-layer convolutional network pσ(a|s) is trained directly on 30 million board positions from the KGS Go Server, using stochastic gradient ascent to maximize the log-likelihood of the human move actually played in each position. It reaches 57.0% move-prediction accuracy on a held-out test set using all input features (55.7% using only raw board position and move history), against a prior state of the art of 44.4% from other research groups at the time of submission. Small gains in this accuracy translate into large gains in playing strength (Fig. 2a in the paper) — the first hint that the accuracy metric and downstream tournament strength are tightly coupled. Alongside it, a much cheaper linear-softmax "fast rollout" policy pπ is trained on small pattern features: only 24.2% accurate, but able to select a move in 2 μs versus 3 ms for the full policy network — cheap enough to run inside MCTS rollouts.

2 — reinforcement learning (RL) policy network

The RL policy network pρ starts with the SL network's weights (ρ = σ) and is then improved by policy-gradient self-play: it plays games against randomly-selected earlier versions of itself (to avoid overfitting to the current policy) with a terminal-only reward (+1 win, −1 loss), and weights are updated in the direction that increases the log-probability of moves taken in games that were won.

Δρ ∝ ∂ log pρ(at|st)∂ρ · zt // policy-gradient update — z_t is the ±1 game outcome from the current player's perspective

Head-to-head, the RL policy network won more than 80% of games against the SL policy network it started from. With no search at all, it beat Pachi — the strongest open-source Go program, running 100,000 simulations per move — 85% of the time; the prior state of the art among purely supervised conv nets managed only 11% against Pachi and 12% against Fuego. Self-play RL, in other words, converts a move-imitator into a much stronger player, purely by optimizing the correct objective (winning) instead of a proxy (matching human moves).

3 — value network

The final stage estimates a value network vθ(s) ≈ vpρ(s), a single-number regression target trained by minimizing mean-squared error between the predicted value and the actual outcome z of games played by the RL policy network against itself: Δθ ∝ ∂vθ(s)/∂θ · (z − vθ(s)). The direct approach — training on complete games from the KGS data set — overfits: because successive positions in one game are almost identical (differing by a single stone) but share one regression target, the network memorizes outcomes instead of generalizing, hitting an MSE of just 0.19 on the training set but 0.37 on the test set. The fix is to generate a fresh self-play data set of 30 million distinct positions, each sampled from a separate game, which collapses the gap to 0.226 train / 0.234 test MSE — evidence the memorization problem, not the architecture, was the bottleneck.

AlphaGo combines all three learned components inside MCTS (Fig. 3). Every edge (s, a) in the search tree stores a visit count N(s,a), an action value Q(s,a), and a prior probability P(s,a) copied from the SL policy network the first time that edge's child is expanded. Selection descends the tree by maximizing Q(s,a) + u(s,a), where the exploration bonus u(s,a) ∝ P(s,a)/(1+N(s,a)) decays with repeated visits. Expansion processes a new leaf once with pσ to set its children's priors. Evaluation scores the leaf two ways — a single forward pass of the value network vθ(sL), and the outcome zL of a fast rollout to the end of the game using pπ — mixed by a parameter λ: V(sL) = (1−λ)vθ(sL) + λzL. Backup propagates that evaluation up every traversed edge, averaging over all simulations through it. A mixed evaluation (λ = 0.5) beat either pure value-network (λ=0) or pure-rollout (λ=1) evaluation, winning ≥95% of games against those variants — the two evaluation signals are complementary, not redundant.

All of this runs on real hardware, not an abstraction: the match-strength single-machine configuration used 40 search threads, 48 CPUs and 8 GPUs; a distributed configuration scaled this to 40 search threads, 1,202 CPUs and 176 GPUs across multiple machines, executing simulations asynchronously on CPU while batching policy/value evaluation on GPU.

// what they found

The results

An internal round-robin tournament pitted AlphaGo against the strongest available Go software — commercial engines Crazy Stone and Zen, open-source engines Pachi and Fuego, and the pre-MCTS engine GnuGo — each given 5 seconds per move, with strength measured as a BayesElo rating. Single-machine AlphaGo won 494 of 495 games (99.8%) against every other program. Given a 4-stone handicap to make things harder, it still won 77% of games against Crazy Stone, 86% against Zen and 99% against Pachi. The distributed configuration was stronger still, winning 77% of games against single-machine AlphaGo and 100% against every other program.

Elo ratings translate the win rates into a single scale (a 230-point gap implies a 79% win probability, roughly one amateur-dan rank on KGS):

The headline result: over 5–9 October 2015, the distributed version of AlphaGo played a formal five-game match, with longer time controls, against Fan Hui — a professional 2-dan and winner of the 2013, 2014 and 2015 European Go championships. AlphaGo won 5 games to 0, the first time a computer program had beaten a human professional player without a handicap in the full 19×19 game — a result the paper notes "was previously believed to be at least a decade away."

// what happened next (not in this paper)

This Nature paper covers only the Fan Hui match. The result almost everyone associates with the name "AlphaGo" came two months later and is not reported here: in March 2016, an improved version of the system played Lee Sedol — an 18-time world-champion 9-dan, and by common consensus one of the strongest players in Go history — in Seoul, and won 4 games to 1. That match used networks and search retrained/refined beyond what this paper documents in detail; the gap between beating a strong European champion and beating a top handful of the game's best players is exactly the gap this paper's own critique should be read against.

// does it hold up?

The eval read

Judged purely on its own numbers, this is an unusually clean result: a 494/495 sweep of every existing Go program, replicated across handicap and distributed-vs-single-machine conditions, followed by a clean 5-0 sweep of a genuine, internationally-ranked professional under tournament time controls. But two specific evaluation questions matter for how far the headline claim ("defeated a human professional") should be allowed to travel.

baselines

The software baselines (Crazy Stone, Zen, Pachi, Fuego, GnuGo) were genuinely the strongest available engines in 2015 — this part is fair. The human baseline is a different story: Fan Hui, professional 2-dan and 3× European champion, is a legitimate professional, but European Go in 2015 trailed the Chinese/Korean/Japanese professional scenes by a wide margin — he was not among the world's top players. The paper is careful never to claim more than "defeated the human European Go champion"; the much stronger claim (beating a top-3-caliber player) is a separate, later result (Lee Sedol, 4-1) that this paper does not contain.

contamination

No obvious leakage: the 30M-position KGS training set and the self-play data used to train v_θ predate and are disjoint from the Fan Hui match games. But the paper's own admission is more interesting than a leakage check — the RL policy network, which is strictly stronger than the SL network head-to-head (>80% win rate), performs worse inside AlphaGo's MCTS than the value function derived from the weaker SL network. The authors' explanation ("humans select a diverse beam of promising moves, whereas RL optimizes for the single best move") is plausible but is offered without a controlled ablation isolating why diversity of the prior, rather than something else about the RL policy's training, is the causal factor.

what's missed

Compute cost is reported (48 CPUs/8 GPUs vs. 1,202 CPUs/176 GPUs) but never interrogated as a cost, accessibility, or reproducibility question — nor is the training-side compute (30M SL positions, self-play RL, 30M value-network positions) ever quantified in GPU-hours or wall-clock time. The tournament is a single run with 95% CIs on Elo, not multiple independent replications, so variance in the ranking itself is not separately characterized.

would raise confidence

A match (or several) against a top-10 world-ranked player at the time, rather than a strong regional champion; explicit training-compute accounting; a controlled study isolating why the "worse" SL-derived value function outperforms the "better" RL-derived one inside search — since that single result quietly complicates the paper's own "self-play RL improves on supervised imitation" narrative.

// bottom line

The internal tournament result (99.8% vs. every existing Go program) is airtight and the real headline. The Fan Hui match is a genuine, well-documented, contamination-free professional win — but it is the second-strongest evidence AlphaGo would go on to produce, not the strongest. Read this paper for the SL+RL+value+MCTS recipe; read the Lee Sedol match (not covered here) for the fully convincing human-vs-machine result.

// what to remember

What to remember

  1. Bootstrap from humans, then correct the objective: a supervised policy network trained to imitate 30M expert moves (57.0% test accuracy) gives fast, high-quality gradients; self-play policy-gradient RL then re-optimizes that same network for the true objective — winning — and beats its own starting point >80% of the time head-to-head.
  2. A learned value network, trained on 30M distinct self-play positions (to avoid the overfitting that plagued training on complete games, 0.226 vs. 0.37 test MSE), is a viable and complementary alternative to Monte Carlo rollouts for evaluating Go positions — the mixed evaluation (λ=0.5) beat either signal alone.
  3. AlphaGo is a starting point, not an end point: it still leans on 30M human expert positions to seed its policy network. Its own lineage removed that dependency one step at a time — AlphaGo Zero learned from self-play alone, AlphaZero generalized beyond Go, and MuZero dropped even the requirement of a known game model — each successive paper erasing one assumption this 2016 paper still needed.
