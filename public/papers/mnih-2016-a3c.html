2016 · ICML · Post-training & RL

Asynchronous Methods for Deep Reinforcement Learning

Run many actor-learners in parallel, each on its own copy of the environment, updating one shared network with lock-free asynchronous gradients — no replay buffer, no GPU, no parameter server. The best variant, A3C, trains on 16 CPU cores and beats the DQN family's state of the art on Atari in half the wall-clock time.

Volodymyr Mnih, Adrià Puigdomènech Badia, Mehdi Mirza, Alex Graves, Timothy P. Lillicrap, Tim Harley, David Silver, Koray Kavukcuoglu

// the gap

What problem is this solving?

By early 2016, deep RL had one reliable recipe for stability: DQN's experience replay. Naively combining online RL with a neural network was thought to be fundamentally unstable, because the stream of observations an online agent sees is non-stationary and strongly temporally correlated — consecutive frames of the same Atari episode look almost identical, so consecutive gradient steps push the network in correlated directions. DQN's fix was to store transitions in a replay memory and train on randomly sampled batches from it, which decorrelates updates and lets the same data be reused many times. It worked, but it came with two structural costs the paper's introduction calls out directly: replay uses more memory and computation per real environment interaction (every transition must be stored and later replayed), and it locks you into off-policy algorithms, since the data being replayed was generated by an older version of the policy. On-policy methods like Sarsa and actor-critic — which need data generated by the current policy — were effectively ruled out for stable deep RL. The other path to scale, DeepMind's own Gorila, threw distributed hardware at the problem: 100 actor-learner processes plus 30 parameter-server instances, 130 machines total, to significantly beat DQN and reach its scores up to 20× faster. That establishes that parallelism helps, but at a hardware cost far beyond what most labs could deploy. The open question this paper asks: is there a way to get the stabilizing effect of replay — and unlock on-policy and continuous-action methods along with it — using nothing more than the CPU cores already sitting in a single machine?

// how it works

The method

The paper's answer is to decorrelate updates in space instead of in time. Instead of one agent replaying old memories, run many actor-learner threads in parallel on a single multi-core CPU, each with its own copy of the environment and (optionally) its own exploration policy — one thread might be exploring greedily, another more randomly, since ε for ε-greedy exploration is periodically resampled per-thread from a distribution. At any instant the threads are almost certainly in different parts of state space, so the combined stream of gradients hitting one shared set of parameters θ is naturally decorrelated, the same job experience replay used to do — but without storing a single transition, and without being restricted to off-policy algorithms. All threads live on one machine and apply Hogwild!-style lock-free updates directly to the shared weights, which removes the network communication cost that Gorila's parameter-server design paid for.

The recipe is instantiated for four base algorithms: asynchronous one-step Q-learning, one-step Sarsa, n-step Q-learning, and advantage actor-critic. The value-based variants keep DQN's slowly-updated target network θ− for computing the bootstrap target, and accumulate gradients over several timesteps before applying a single asynchronous update — both tricks reduce the chance that concurrent threads overwrite each other's progress. The n-step variants use the forward view — explicitly rolling out up to tmax steps and computing n-step returns for every state visited, rather than the backward-view eligibility traces classically used for n-step TD — because the forward view composes more naturally with backprop-through-time and momentum-based optimizers.

The best-performing instantiation, asynchronous advantage actor-critic (A3C), maintains a policy π(at|st;θ) and a value estimate V(st;θv), sharing every layer except the two output heads (one softmax for π, one linear unit for V). Both are updated together every tmax actions or at a terminal state, using an n-step estimate of the advantage:

A(st,at;θ,θv)  =  k−1Σi=0 γirt+i + γkV(st+k;θv) − V(st;θv) // k varies per state, upper-bounded by t_max — the forward-view n-step advantage

The policy gradient is ∇θ′ log π(at|st;θ′)·A(st,at;θ,θv), and the paper adds an entropy bonus to the objective — β·∇θ′H(π(st;θ′)) — specifically to discourage premature collapse onto a deterministic sub-optimal policy; β sets its strength. For the optimizer, the paper compares plain SGD with momentum against RMSProp, and finds that a variant of RMSProp where the second-moment statistic g is shared across all asynchronous threads (rather than kept per-thread) is considerably more robust than either alternative.

// what they found

The results

On five Atari games used throughout the paper's analysis (Beamrider, Breakout, Pong, Q*bert, Space Invaders), all four asynchronous methods train successfully and tend to learn faster than DQN — despite DQN training on an Nvidia K40 GPU while the asynchronous methods use only 16 CPU cores. Among the four, the policy-based A3C significantly outperforms the three value-based variants (one-step Q, one-step Sarsa, n-step Q):

Scaled up to the full suite of 57 Atari games, following Van Hasselt et al.'s (2015) human-starts evaluation protocol, A3C's results compare directly against the published DQN-family state of the art of the time:

A3C-LSTM's 623.0% mean human-normalized score, after 4 days on 16 CPU cores with no GPU, tops every GPU-trained baseline that needed 8-10 days — and after just one day of training, the feedforward A3C already matches Dueling Double DQN's mean and nearly reaches Gorila's median, despite Gorila's 130-machine cluster. The paper is also the first to show one of these async algorithms working well beyond discrete Atari: on the MuJoCo physics simulator's continuous-control manipulation and locomotion tasks (from either physical state or raw pixels), A3C found good solutions in under 24 hours and typically within a few hours; on the TORCS 3D car-racing simulator, A3C reached roughly 75-90% of a human tester's score across four car/opponent configurations in about 12 hours; and on a new 3D maze-navigation task the paper introduces, called Labyrinth, an A3C-LSTM agent using only raw 84×84 RGB pixels learned to repeatedly find reward objects (apples worth 1 point, portals worth 10 and triggering a re-spawn) for a final average score of around 50.

The framework also scales the way it's supposed to. Averaged over seven Atari games, going from 1 to 16 parallel actor-learner threads gives a 24.1× speedup for one-step Q, 22.1× for one-step Sarsa, 17.2× for n-step Q, and 12.5× for A3C — all at least an order of magnitude, confirming the framework makes efficient use of added workers. The one-step methods' speedups are actually super-linear, which can't be explained by parallel compute alone; the paper attributes this to added threads reducing the bias of one-step bootstrap targets, so the methods also need less data, not just less wall-clock time, to reach a given score as thread count grows. Finally, sweeping 50 random learning rates and initializations for A3C across the same five games shows a wide band of learning rates that all reach good scores with virtually no collapsed runs inside that band — the paper's evidence that the method is robust rather than fragile.

// does it hold up?

The eval read

The headline claim — A3C beats DQN-family state of the art in half the training time using no GPU — is real, but the comparison it rests on is looser than it first appears. Table 1's DQN, Gorila, Double DQN, Dueling DQN, and Prioritized DQN numbers are not reproduced in-house under one shared harness; they're copied from each method's own paper. "8 days on GPU" vs. "4 days on 16 CPU cores" is also a wall-clock comparison across genuinely different 2015-era hardware, not a FLOP- or cost-normalized one, so "half the time" is a real practical win but not evidence that A3C is more compute- or sample-efficient in any hardware-neutral sense.

There's also a subtler overlap worth flagging: the six Atari games used to tune A3C's only real hyperparameters (learning rate and gradient-norm clipping) — Beamrider, Breakout, Pong, Q*bert, Seaquest, and Space Invaders — are almost exactly the five games used for every other analysis in the paper (Figure 1's learning curves, Figure 2's learning-rate sweep, Table 2's speedup numbers). That means the games where A3C is shown to be fast, stable, and robust to learning rate are largely the same games its one real hyperparameter search was fit to, and it's the games outside that curated six — the other 51 of the 57 — where the story is asserted (via Table 1's aggregate score) rather than shown.

The 57-game headline metric itself has a known blind spot: human-normalized mean score is dominated by the handful of games where any competent agent trivially multiplies the human reference score. A3C-LSTM's mean of 623.0% looks like a landslide over Dueling DQN's 343.8%, but the medians — 112.6% vs. 117.1% — are essentially tied, meaning on a typical game in the suite the two methods are close, and the mean's gap is coming from outlier games not shown in the main text (only a supplementary table breaks out per-game scores). Single-run reporting on that 57-game table, with no seed variance, is also a gap that later work (Henderson et al.'s "Deep RL that Matters," 2018) would go on to show matters a great deal for RL specifically.

baselines

DQN/Gorila/Double-DQN/Dueling-DQN/Prioritized-DQN numbers in Table 1 are each copied from their own paper, not re-run under a common harness; "days of training" also compares a K40 GPU to 16 CPU cores, not FLOPs or cost.

contamination

Not applicable in the LLM sense — Atari is a deterministic simulator with no train/test split — but the paper does use random "human starts" rather than the fixed reset state, the domain-appropriate defense against an agent just memorizing one trajectory.

what's missed

The 623.0% mean vs. 343.8% for Dueling DQN looks decisive, but the medians (112.6% vs. 117.1%) are nearly tied — the mean's gap comes from outlier games not broken out in the main text, and there's no seed variance on the 57-game numbers.

would raise confidence

Multi-seed variance on the full 57-game table (not just the 5-game speed plots), a hardware/FLOP-normalized compute comparison, per-game scores in the main text, and a learning-rate robustness check on games outside the six used for tuning.

// bottom line

The mechanism is genuinely well-supported: parallel asynchronous actor-learners really do decorrelate updates as well as a replay buffer, without its memory cost or its off-policy restriction, and the paper backs that up across four algorithms and four very different domains (Atari, TORCS, MuJoCo, Labyrinth). What deserves more skepticism is the specific "state of the art in half the time" framing in Table 1 — a wall-clock, cross-hardware, single-run, mean-dominated comparison that reads more decisively than the underlying evidence, most visibly once you notice the median scores are close and the tuning games double as the robustness-demo games.

// takeaways

What to remember

  1. Parallel actor-learners exploring different parts of the environment can decorrelate updates just as well as an experience-replay buffer — without storing transitions and without being restricted to off-policy algorithms.
  2. The best variant, A3C, is an n-step advantage actor-critic with an entropy bonus, trained with lock-free Hogwild!-style updates across 16 CPU cores — no GPU, no replay memory, no parameter server — and it reaches a 623.0% mean / 112.6% median human-normalized score on 57 Atari games in 4 days.
  3. Because it's on-policy and needs no replay buffer, A3C is the first of the paper's four methods shown to work directly on continuous-action MuJoCo control and a novel 3D visual maze task (Labyrinth), not just discrete-action Atari — the generality that later made it the baseline PPO, A2C, and IMPALA all measure themselves against.
